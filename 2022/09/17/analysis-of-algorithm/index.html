算法分析


引言

一般来说, 有4种算法:

  • Obvious (显示的): by problem definition

  • Methodical (有条不紊) : by a method good for a class of problems (有一些通用的方法适合解决某一类问题, 例如 dp)

  • Clever : 用于解决具体的一个问题
  • Miraculous (神奇): how did anyone think of this?

SCC (strong connected components) algorithm

Observation 1: Every di-graph is a DAG on SCCs

所有的有向图都可以看作是由强连通分量组成的 DAG


Observation 2: 假设我们能找到一个顶点 v inside a source SCC of G.

Then $SCC(v) = Reach(v, G^{reverse})$


思路: Remove $SCC(v)$ from $G$ and repeat

然而难点是:

  1. 如何找到 $v \in \text{source SCC}$

  2. Need the overall runtime $O(m+n)$


Intuition: Find $v$ by running a “kind of” Topo sort algo on G

然而例如下图, 可能所有顶点都有 incoming edges



A different algo for Topo Order (using DFS)

DFS algo

DFS(G)

  • init finish = $\text{<>}$

  • $\forall v \in V, D\text{[v]} =F \text{ and }\pi\text{[v]=Nil}$

  • for each vertex $v \in V$ if $D\text{[v]=F}$ then call $\text{DFS-visit}(v)$

DFS-visit(v):

D[v] 用来标记这个顶点是否被访问过, F 为没访问, T 为访问过


如果 G 是 DAG, 那么 finish 就是 topo order

深度优先后序遍历


SCC(G) 算法:

  1. 运行 DFS(G) 来取得 finish 的list
  2. 运行 $\text{DFS}(G^\text{reverse})$ 根据 finish 中 node 的顺序
  3. Output DFS trees $T_1,…T_k$ as SCCs

强连通分量


证明这个算法的正确性:

假设 $\text{finish=<} u,…,\text{>}$

结论: $\text{DFS-visit(u)}$ for $G^{reverse}$ will find $T_u=SCC(u)$

结论 1: $T_u\supe SCC(u)$

结论 2: $T_u$ 不包含任何 node $w$ 使得 $w \not \in SCC(u)$


贪心算法: 霍夫曼编码 (Huffman Coding)

Q: 对于一个使用 32 种 symbols 的文本(26 个英文字母, 空格, 以及一些标点符号), 我们如何用 bit 来 encode 这些文本

A: 我门可以使用 5 bits 来对每个 symbol 进行 encode. $2^5=32$, 这是 fixed length encoding


Q: 有一些符号 (e, t, a, o, i, n) 的使用频率更高, 根据这个信息我们如何缩短 encoding 长度

A: 我们可以用更少的bits 来 encode 这几个符号, 而对其他符号可以用更长的bit


Q: 我们如何知道什么时候下一个 symbol 开始?

A: Use a separation symbol (like the pause in Morse), or make sure that there is no ambiguity by ensuring that no code is a prefix of another one.


Greedy Analysis

Claim: Huffman code for $S$ achieves the minimum $ABL$ of any prefx code

证明: by induction, base on optimality of $T^{\prime}$ ($y$ and $z$ removed, $w$ added)


结论: $ABL(T^{\prime})=ABL(T)-f_w$

证明:

$\forall T, T^{\prime}$ obtained form $T$ by merging two least feq. leaves $x, y$ into $w$, with $f_w=f_x+f_y$


分治算法: Merge Sort

  • 把一个大问题分解为多个相同类型的小问题
  • 递归解决每个小问题
  • 把小问题的结果组合为大问题的结果

比较常用的方法是:

  • 把大问题 (size n) 分为两个小问题 (size n/2) 需要 O(n) 的时间
  • 递归解决两个子问题
  • 把两个子问题的结果组合为最后的结果 需要 O(n) 的时间


Merge Sort

  • 递归 sort 左半部分
  • 递归 sort 右半部分
  • 把已经 sort 后的两半部分组合成一个 sort 后的数组

Sequence Alignment

目标: 给定两个 string: $x_1x_2\cdots x_m$ 以及 $y_1y_2\cdots y_n$ , 寻找 min-cost alignment

定义: An alignment $M$ is a set of ordered pairs $xi-y_i$ such that each character appears in at most one pair and no crossings. ($x_i -y_i$ and $x{i’}-y_{j’}$ cross if $i < i’$ but $j>j’$)



Hirschberg’s algorithm

Edit distance graph

  • $f(i, j)$ 代表 从 $(0, 0)$ 到 $(i, j)$ 最短路径的长度

Lemma: $f(i, j)=OPT(i, j)$ for all $i$ and $j$

这里的问题就是只用 $O(mn)$ 的时间和 $O(m+n)$ 的空间来计算 $f(\cdot, j)$ ($\cdot$ 意味着”任何” , $f(\cdot, j)$ 就是计算第$j$ 列所有的 distance)

算法就是从 $(0,0) , (m,n)$ 向这一列进行普通 dp.

而这一次的计算中我们不需要存储路径, 因此我们不用创建 $O(mn)$ 的数组来存储, 而是可以复用空间, 因此只需要 $O(m+n)$ 的空间

而这个 dp 算法本身要 $O(mn)$ 的时间, 因此, 寻找 $f(\cdot, j)$ 只用 $O(mn)$ 的时间和 $O(m+n)$ 的空间

这里根据前一个分析 $T(m, n) \le O(mn\log n)$

因此 $T(m, 2) \le O(2m \log 2) \le cm$ for some constant $c$

我们是可以找到 $c$ 的

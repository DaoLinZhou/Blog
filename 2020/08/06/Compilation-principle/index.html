汇编指令入门(MIPS)

计算机通过指令指挥计算机工作.

CPU被时钟驱动, 不断地读取PC指针指向的指令, 并增加PC指针, 从内存中读取指令并执行 (如此周而复始)

不同的CPU架构使用不同指令. 目前使用最广泛的是RISC (Reduced instruction set computer, 精简指令集)

函数的返回

jal跳转会把函数返回之后继续执行的位置写入ra寄存器

但是ra只有一个, 不能复用, 所以进入函数时要把ra内的值也写入栈中. 同时栈指针指向下一位.

也就是说当调用一个函数时, 进行两次压栈操作

  1. 把参数压栈, 栈指针自增
  2. 把函数返回后继续执行的位置压栈, 栈指针自增.

这样返回时, 跳到t1就可以了

因此一进入函数就要跑这段程序


如何实现返回值

也可以通过栈实现. 进入函数体的时候创建一个返回值, 将他压栈(调用位置的下方)

在函数体中计算是通过偏移量设置返回值(覆盖)

当函数调用结束时程序取当前栈指针+4的位置把返回值读出来, 顺便把其他的都清掉.

中断和中断向量

中断 Interrupt

当外界发生变化时, 通过中断通知CPU应该去注意某个信号(事件). 这个时候, CPU当前执行的程序会被中断, 当前的执行状态会被保存, 中断相应程序会被执行


中断相应过程

中断向量表其实就是内存中的一块区域

每一个中断向量就是一个数字, 是一个指向要执行程序的地址

比如说机器中有255个中断, 中断的类型是不一样的

键盘相应是一种中断类型, 鼠标相应是一种中断类型…错误, 异常都会产生中断

OS接管中断后就是程序相应中断


中断请求 (Interrupt Request)

  • 硬件设备发给主板 (打印机, 键盘, 鼠标等)

  • 硬件中断: CPU异常(除以0), 时钟信号等

  • 软件中断: 发出 (异常, 切换到内核态等)


中断向量表(Interrupt Vector Tables)

(一般在内存中) 一块区域, 存储了中断类型和中断相应程序的对应关系. 每一行叫做一个中断向量


如果没有中断: (polling)

中断是为了提高效率 (异步IO)


中断的意义

  • 提高工作效率
  • 故障恢复 (异常处理, 紧急事件等)
  • 简化编程模型 (try/catch, 计时器等)

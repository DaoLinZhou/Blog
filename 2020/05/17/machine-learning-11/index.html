什么是集成学习

集成学习: ensemble learning

每一种机器学习算法考虑问题的方式都略微有所不同, 所以对于同一个问题不同的算法可能给出不同的结果

这个时候应该听那个算法的结果? 此时就可以把多个算法集中起来进行投票, 最后少数服从多数. 这就是集成学习


实现一个集成学习并不难, scikit-learn提供了更加方便的接口: Voting Classifier

就是基于不同算法最后进行投票的分类器

使用的数据:

if __name__ == '__main__':
    X, y = datasets.make_moons(n_samples=500, noise=0.3, random_state=42)

    plt.scatter(X[y==0, 0], X[y==0, 1])
    plt.scatter(X[y==1, 0], X[y==1, 1])
    plt.show()

    X_train, X_test, y_train, y_test = train_test_split(X, y, random_state=42)

    log_clf = LogisticRegression()
    log_clf.fit(X_train, y_train)
    print(log_clf.score(X_test, y_test))    # 0.864

    svm_clf = SVC()
    svm_clf.fit(X_train, y_train)
    print(svm_clf.score(X_test, y_test))    # 0.896

    dt_clf = DecisionTreeClassifier()
    dt_clf.fit(X_train, y_train)
    print(dt_clf.score(X_test, y_test))     # 0.84

    y_predict1 = log_clf.predict(X_test)
    y_predict2 = svm_clf.predict(X_test)
    y_predict3 = dt_clf.predict(X_test)

    # 对于二分类问题, 如果有两个预测值为1, 则预测为1
    y_predict = np.array((y_predict1 + y_predict2 + y_predict3) >= 2, dtype='int')
    print(accuracy_score(y_test, y_predict))    # 89.6%的准确率

Soft Voting

少数服从多数的投票方式叫做Hard Voting

还有一个更重要的方式: Soft Voting


背后的想法也比较简单.

很多情况下, 少数服从多数是不合理的.

社会学, 政治学中有这样一个概念: 民主暴政

就是如果一人一票的话, 使用完全民主的方式, 有些时候反而是一个非常不好的结果

更合理的投票, 应该有权值


假设有5个模型, 面对一个二分类问题, 把数据分为A类和B类

对于一些算法, 它把某一个数据分为某一类的依据其实是计算概率

这里就是5个模型把一个数据分成A类和B类的概率

对于这样一个结果, 如果我们只看最终的结果, A-两票, B-三票

根据Hard Voting, 最终结果为B


但是看这个数据好像有点问题, 虽然只有两个模型把这个数据分成A类. 但是这两个模型近乎是非常确定, 认为这个模型就应该分为A类; 相应有3个模型把这个数据分为B类, 但是他们分类为B的概率并不高.

所以虽然分为A类的票数少, 但是它的权值应该更高一些.

所谓Soft Voting就是这样的一个考虑

计算分为每类的概率的平均值

根据这样的数字来看的话, 其实我们应该把结果分为A类. 这就是Soft Voting

换句话说, 它不仅要看把多少票投给A, 把多少票投给B, 还要看投给A的票到底有多大的概率确认分为A类, 看投给B的票到底有多大的概率确认分为B类


因此Soft Voting要求集合的每一个模型都能估计概率

逻辑回归就是根据一个概率模型创建的, 它有一个predict_proba方法, 有这个函数就说明是可以求出概率的

借此回顾一下分类算法是不是都可以估计概率


逻辑回归, 本身就是基于概率模型的


kNN, 根据离他最近的几个点进行分类, 通过投票kNN会把这个点预测为是红色的类, 此时它属于红色类的概率就是66% = 2/3

kNN也可以考虑权值, 考虑权值的话就看加权值所占的比重就可以了. 因此kNN算法也是支持计算概率的


决策树算法计算概率的方式和kNN算法计算概率的方式非常一致, 只不过样本在决策树中从根节点走到叶子节点, 在叶子节点中通常是包含多个类别的数据的, 它的基尼系数或信息熵不为0, 此时哪个类数据量大就分给哪个类.

这个概率和kNN算法是一样的, 占比率最大的节点的数量除以总结点的数量


最复杂的是SVM算法

对于SVM / SVC算法, 它本身是没有考虑概率的, 而是寻找一个统计上margin的最大值. 虽然如此, SVM算法依然可以使用一些方式来计算出概率

这种方式代价就是更多的计算资源.

具体是怎么计算出概率的就不涉及了, 不过scikit-learn中SVC有一个probability参数, 默认为False, 但是如果改为True, 就可以估计样本分到各个类别的概率


他们都可以直接或间接的算出样本分到各个类别的概率, 有这个概率就可以使用Soft Voting这种方式

voting_clf2 = VotingClassifier(estimators=[
    ('log_clf', LogisticRegression()),
    ('svm_clf', SVC(probability=True)),
    ('dt_clf', DecisionTreeClassifier(random_state=666))
], voting='soft')
voting_clf2.fit(X_train, y_train)
print(voting_clf2.score(X_test, y_test))  # 0.904

很多时候Soft Voting Classifier的效果是比Hard Voting Classifier效果好


这只是集成学习入门的方式, 集成不同的算法.

可是这里有一个很大的问题, 但是算法的种类再多也是非常有限的, 怎样才能聚集更多的分类器, 创建出更好的分类结果?



Bagging 和 Pasting

集成学习就是集成多个算法, 让不同的算法对用一组数据进行分析, 投票来看具体那个结果是更好的结果.

虽然有很多机器学习方法, 但是从投票的角度看, 仍然不够多

如果想要有一个好的结果, 我们就希望有更多的投票者, 成百上千乃至上万, 才能保证结果更加可信. (概率论中的大数定理)


所以要创建更多子模型, 来集成更多子模型的意见.

更加重要的是, 子模型之间不能一致! 子模型之间要有差异性


例如: 一共有500个样本数据, 每个子模型只看100个样本数据. 而相应的, 每个子模型使用的算法可以是同一个算法

这样就做到了创建很多的子模型, 而且这些子模型之间存在差异

这么做每个模型的准确率不就变得很低么? 确实如此, 但同时这也是集成学习算法的威力

集成学习集成诸多子模型来投票最终的分类结果, 在这个过程中, 每个子模型不需要太高的准确率

举个例子: 如果每个子模型只有51%的准确率. (不需要太高的准确率, 但好歹也要比平均水平要高)

假设分类任务整体的数据是无偏的

如果我们只有一个子模型, 整体准确率: 51%

如果有3个子模型, 判断正确有两种情况(三个模型都判断正确, 或任意两个模型判断正确) 整体准确率:
$$
0.51^3+C_3^2\cdot 0.51^2\cdot0.49=51.5%
$$
如果有500个子模型, (只要有250个以上的模型判断正确) 整体准确率:
$$
\sum_{i=251}^{500}C_{500}^i\cdot 0.51^i\cdot0.49^{500-i}=65.6%
$$
随着子模型数量的增加, 集成学习的整体模型的准确率也是在不断上升的


假设准确率达到了60%, 如果有500个子模型, 整体准确率就达到
$$
\sum_{i=251}^{500}C_{500}^i\cdot 0.6^i\cdot0.4^{500-i}=99.999%
$$
当然这只是理论情况, 同时我们也不能保证每个模型都有60%的准确率, 通过这样的计算可以看到集成学习的威力. 每个模型不需要太高的准确率, 但合在一起, 整体显著率就会非常高


这就是统计显著 (Statistically Significant),

根据 中心极限定理 可以算出z值:
$$
\frac {0.5-p} {\sqrt {\frac {p(1-p)}{n}}} \le z
$$
z值服从正态分布, p为预测正确的概率, n为样本数量, 在Z table中就可以找到对应的概率. 当p>0.5的时候, n越大这个式子的值就越小, 因此z在正态分布下右侧的面积就越大, 概率就越大.


如何创建差异性?

每个子模型只看样本数据的一部分. 我们怎么看样本数据的一部分?

我们有两种方式只看样本数据的一部分: 放回取样(Bagging) 和 不放回取样 (Pasting)

  • 放回取样(with replacement): 在500个样本中只看100个样本, 一个子模型看了100个样本之后把这100个样本放回去, 下一个子模型继续在这500个样本中随机找100个样本进行学习
  • 不放回取样(without replacement): 一个子模型看100个样本, 下一个子模型只能从剩下的400个样本中再挑100个样本进行学习. 也就是说每个子模型看到的样本不重复

通常来讲放回取样(Bagging)更常用

如果使用不放回取样, 500个样本, 每次看100个样本, 只能训练5个模型, 数量太少. 而Bagging可以训练成千上万的子模型

Bagging的第二个优点就是: 没有那么强地依赖随机, 如果使用Pasting训练5个样本, 其实就相当于把样本分成5份, 每一份有100个样本. 怎么分就会非常强烈地影响最终的结果. Bagging则是每次从500个样本中取100个样本进行学习, 这个学习过程重复非常多次, 其实一定程度将随机带来的问题给取消了


Bagging是机器学习中的叫法. 统计学中, 放回取样叫做bootstrap

scikit-learn中, 选择放回取样还是不放回取样只要调节bootstrap这个参数即可


import numpy as np
import matplotlib.pyplot as plt
from sklearn import datasets
from sklearn.ensemble import BaggingClassifier
from sklearn.model_selection import train_test_split
from sklearn.tree import DecisionTreeClassifier

if __name__ == '__main__':

    X, y = datasets.make_moons(n_samples=500, noise=0.3, random_state=42)
    X_train, X_test, y_train, y_test = train_test_split(X, y, random_state=42)

    # Bagging
    # 集成500个模型, 每个模型看100个数据, 放回取样.
    bagging_clf = BaggingClassifier(DecisionTreeClassifier(),
                                    n_estimators=500, max_samples=100,
                                    bootstrap=True)
    bagging_clf.fit(X_train, y_train)
    print(bagging_clf.score(X_test, y_test))    # 0.904

让每一个集成的算法都是决策树模型, 因为决策树这种非参数学习的方式更能产生差异比较大的子模型

所以要集成成百上千个子模型, 首选决策树

scikit-learn中bagging和pasting统一使用BaggingClassifier, 只不过通过bootstrap传入True或False来改变

n_estimators=5000时, 准确率能达到 0.912

理论上说, 子模型数量越多, 最终准确率就会越高. 只不过不会向上面所说达到99.999%, 这是因为具体的学习中还是和样本数据相关的. 如果样本存在噪音, 甚至存在错误的样本, 我们依然会犯更大的错误

不管怎么说这种策略通常都是有很好的结果的



OOB (Out-of-Bag)

对于Bagging有一个非常重要的概念OOB. 什么是oob?

放回取样导致一部分样本很有可能没有取到

平均大约有37%的样本没有取到.

这37% 的样本就叫做 Out of Bag , 从来没有被取出来的样本.

如果有这样的情况的话, 其实在使用Bagging时就不用train-test-split了, 而是使用这部分没有取到的样本做测试 / 验证

scikit-learn中有一个属性 oob-score, 使用这个属性就能直接看对于Bagging, 这些Out-of-Bag样本作为测试数据集得到结果是怎样的


if __name__ == '__main__':
    X, y = datasets.make_moons(n_samples=500, noise=0.3, random_state=42)
    # 如果想使用oob, 就要把oob_score设置为True
    bagging_clf = BaggingClassifier(DecisionTreeClassifier(),
                                    n_estimators=500, max_samples=100,
                                    bootstrap=True, oob_score=True)
    bagging_clf.fit(X, y)
    print(bagging_clf.oob_score_)   # 0.918

并行处理

集成学习(Bagging)的思路极易进行并行化处理, 因为我们是独立训练若干个子模型, 每一个子模型都是从500个样本中取100个样本, 取样本的过程也是独立的

scikit-learn中, 对于可以并行处理的算法都是传入n_jobs这个参数, 传入几就用几个核, -1就用上所有核

Grid-SearchCV时使用过了

bagging_clf = BaggingClassifier(DecisionTreeClassifier(),

差异化

Bagging创建子模型要让子模型有一定差异化, 之前让子模型产生差异化的方式是让每个子模型都看更小的样本集, 其实还有其他产生差异化的方式


一个非常重要的方式就是针对特征进行随机取样 (随机子空间:Random Subspaces)

通常应用在特征非常多的情况, 例如图像识别: 每一个像素点都是一个特征, 每一个样本的特征量就比较大

随机子空间: 所有特征组成一个特征空间, 每一次取样的时候是在一个子空间进行取样


如果同时使用bootstrap对样本的数量进行取样, 又对样本特征进行取样(Random Subspaces)

这种采样方式又叫 Random Patches 随机补丁

这个名字本身很形象

数据本身就是一个矩阵, 每一行是一个样本, 每一列代表一个特征, Random Patches就是既在行的维度随机, 又在列的维度随机. 就如下图, 像是一块布上的很多补丁


在特征空间上的随机采样也可以叫 bootstrap features

if __name__ == '__main__':
    X, y = datasets.make_moons(n_samples=500, noise=0.3, random_state=42)
    random_subspaces_clf = BaggingClassifier(DecisionTreeClassifier(),
                                             # 总共有500个样本, 采用500就相当于没有随机
                                             n_estimators=500, max_samples=500,
                                             bootstrap=True, oob_score=True,
                                             # 最大特征数为1, 设置针对特征进行采样
                                             max_features=1, bootstrap_features=True)
    random_subspaces_clf.fit(X, y)
    print(random_subspaces_clf.oob_score_)  # 0.832

由于样本只有2个特征, 使用随机特征采样的方式是不合适的, 主要了解怎么使用的对样本进行随机采样的



随机森林

我们用BaggingClassifier都是集合了大量决策树, 其实这种集合方式还有一个非常形象的名字: 随机森林

BaggingClassifier中有500颗树, 它们就一起构成了一个森林, 对于每一棵树看到的样本都是不同的.


scikit-learn分装了一个随机森林类, 这个随机森林的模型还提供了更多的随机性: 决策点在节点划分上, 在随机的特征子集上寻找最优划分特征 (最好的维度/特征, 最优的阈值)

换句话说, 并不是在每一个节点上都在所有的特征上寻找最优划分, 这种划分方式增加了子模型的随机性

集成学习中, 子模型有成百上千个的时候, 每一个子模型和其他的子模型越有差异越好, 创建这种差异性的一个方式就是增加随机性

if __name__ == '__main__':
    X, y = datasets.make_moons(n_samples=500, noise=0.3, random_state=42)

    # 随机森林
    rf_clf = RandomForestClassifier(n_estimators=500, random_state=666, oob_score=True, n_jobs=-1)
    rf_clf.fit(X, y)
    print(rf_clf.oob_score_)    # 0.896

随机森林结合了BaggingClassifier和DecisionTreeClassifier, 它的参数也包含两者的大部分参数


Extra-Trees

和随机森林非常类似还有另外一种方法. 有时候我们也可以叫它随机森林, 不过有些时候它有另外一个名字 Extra-Trees (全称: Extremely Random Trees) 极其随机森林

极其随机森林的随机表现在: 在决策树的节点划分上, 使用随机的特征和随机的阈值.

此时每颗决策树的差异就会更加大, 因为更加随机

之前讲Bagging Classifier时说整体只要比大多数决策树的决策能力好一点就够了, 成百上千个决策树的决策结果也会是不错的

这就时 Extra Trees 有应用价值的基础

这种方式提供了额外的随机性, 抑制过拟合, 遏制了方差, 但增大了偏差bias

由于Extra-Tree划分时毫不费劲, 使用随机的阈值, 随机的特征. 所以这个模型有另外一个优点: 更快的训练速度


et_clf = ExtraTreesClassifier(n_estimators=500, bootstrap=True, oob_score=True, random_state=666)
et_clf.fit(X, y)
print(et_clf.oob_score_)    # 0.892

RandomForestClassifier() RandomForestRegressor()

至此就是Bagging的方式进行集成学习的方式, 通过随机取样的方式进行集成学习.

这种方式是非常常用的一种集成学习方式

但这不是唯一的一种集中学习的方式.


Boosting

boosting是增强推动的意思

这种集成学习的思路也是要集成多个模型, 不过和Bagging不同, 这些模型之间不是彼此独立的关系, 而是相互增强的关系

每个模型都在尝试增强(Boosting)整体的效果


Ada Boosting

首先有蓝色的原始数据集, 通过某一种学习的方法(假设解决回归问题, 当然也可以解决分类问题) (初始数据点的权值是一样的)


通过这样的算法学习完之后, 显然所有的算法都会犯错误, 那么我们就可以得到这样一个结果

此时数据点分成深色浅色, 浅色的点就是被第一个模型近乎毫无差别地被预测成功的点, 而深色的点和得到的模型差距比较远, 所以形成这样的样本点

在这组样本点中, 数据和数据之间产生了权重的差别

我们让上一次学习中没有很好被学习的点的权值增大一些, 而在上一次学习中已经很好学习的点的权值减小一些, 这样得到的新的样本数据再使用某一种学习算法进行学习.


经过第二次学习, 由得到一个模型, 同样也是有些样本点被准确预测, 有一些样本点没有被准确预测. 这样一来又形成一组新的数据

这组数据点中每一个点的权重又有差距, 再进行学习, 预测, 改变权值. 持续进行下去


再这个过程中, 每次生成的子模型都是在想办法弥补上一次没有成功预测的样本点, 换句话说每一个子模型都是在想办法推动 (Boosting), Boost上一个模型所犯的错误.

经过这个过程Ada Boosting也可以生成非常多的子模型, 对于每一个子模型它是基于同样的一组样本点只不过这些样本点中, 点和点之间的权重不一样, 子模型认为哪些点更重要, 哪些点不重要的程度不同, 所以每一个子模型是有差异的.

最终所有子模型进行投票作为Ada Boosting最终的学习结果


对于Ada Boosting来说, 怎样为数据点附上权值?

Gradient Boosting

另一种Boosting的算法是Gradient Boosting

思想是这样的:

训练一个模型m1, 产生错误 e1

针对e1训练第二个模型m2, 产生错误 e2

针对e2训练第三个模型m3, 产生错误 e3

最终预测结果是: m1 + m2 + m3 … …

左边绿色的线是每个模型的预测线, 右边红色的线是最终模型的样子(所有预测模型相加的结果)

当前模型(绿色的线) 总体模型(红色线)
m1 m1
m2 m1 + m2
m3 m1 + m2 + m3

m2 是根据 m1 犯的错误 (ypredict-ytrue) 得到的模型, m3 是根据 m2 犯的错误得到的模型 (ypredict-ytrue) 

# GradientBoostingClassifier就是以决策树作为基础的, 不是项AdaBoosting那样可以选择

Stacking

Voting Classifier 这种集成学习的方式就是: 假设有3个算法, 每个算法针对数据都进行预测, 最后综合这3个算法结果, 分类问题少数服从多数, 回归问题就是求均值


Stacking是另外一种思路:

首先也假设有3个算法, 对于这3个算法先求出对于数据的预测结果. 但是不使用这3个预测结果综合来得到最后的结果

而是以这3个预测结果进行输入, 再添加一层算法(以这三个算法为输入, 训练一个新的模型, 用这个新的模型的输出作为预测的结果)

多了一个模型, 落在三个模型之上. “落”这个动作就是Stacking表示的意义

这种思路同样可以解决分类问题, 解决的方法也非常简单:

下面这3个模型预测的结果是某一类的概率, 概率是一个具体的数值

将三个模型的概率值作为第四个模型的输入再得到一个总体的新的概率, 用这个概率进行分类

分类问题和回归问题可以以概率作为桥梁, 将分类问题转换为回归问题, 回归问题转换为分类问题


逻辑回归也介绍过, 为什么逻辑回归解决的是分类问题, 但这种方法却叫逻辑回归, 原因在于计算的是样本分为某一类的概率


怎么训练一个Stacking?

我们要做的是首先把训练数据集分成两份, 对于其中的一份用于训练这3个模型, 另外一份用于训练上面的那层模型


也就是说, 用第一个训练集把底下的三个模型训练好了, 在训练好了之后对于第二部分就可以直接扔进这些模型中得到一组输出.

这组输出结果和第二份样本数据真值的输出就形成了一个新的数据集

用这新的数据集训练第4个模型, 最终形成Stacking整体的模型


理解这一步之后, 就可以构建更加复杂的Stacking模型

落了3层模型, 第一层的三个模型就得到3个输出, 把这3个输出作为输入就可以分别再训练3个模型.

这三个模型(第二层)以第一层的输出作为输入再得到3个输出. 最终以这3个输出作为第三层(一个模型) 的输入, 第三层的输出结果作为整个集成学习Stacking的输出结果

训练这样一个Stacking就要将原始数据集分成3份, 第一份由于训练第一层的3个模型, 第二份用于训练第二层的3个模型, 第三份用于训练最后一层的一个模型


所以对于Stacking来说, 有几层也是一个超参数, 每一层使用多少个模型也是一个超参数

正因如此, 训练Stacking模型要复杂很多, 也因为这种复杂性, 它是非常容易过拟合的


这个图这样的方式已经非常像神经网络了, 只不过对于神经网络来说, 每一个神经元不是一个全新的算法, 而只是计算一个函数的值而已

对于神经网络来说, 如果层数增多就已经是深度学习的模型了, 对于神经网络如果层数增多会引发一些新的问题, 这些问题就是深度学习要解决的问题

神经网络正是因为这种灵活性, 非常容易发生过拟合

深度学习中很多话题就是在探讨对于这样一个模型, 我们要如何解决过拟合问题. 其中的很多技术其实适用于Stacking这个模型


Stacking这样的模型scikit-learn中根本没有提供任何接口来实现Stacking这个算法.



集成学习本身是一种非常强大的方法, 背后也有非常多的理论基础, 有统计学的知识支撑

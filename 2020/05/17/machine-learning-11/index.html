什么是集成学习

集成学习: ensemble learning

每一种机器学习算法考虑问题的方式都略微有所不同, 所以对于同一个问题不同的算法可能给出不同的结果

这个时候应该听那个算法的结果? 此时就可以把多个算法集中起来进行投票, 最后少数服从多数. 这就是集成学习


实现一个集成学习并不难, scikit-learn提供了更加方便的接口: Voting Classifier

就是基于不同算法最后进行投票的分类器

使用的数据:

if __name__ == '__main__':
    X, y = datasets.make_moons(n_samples=500, noise=0.3, random_state=42)

    plt.scatter(X[y==0, 0], X[y==0, 1])
    plt.scatter(X[y==1, 0], X[y==1, 1])
    plt.show()

    X_train, X_test, y_train, y_test = train_test_split(X, y, random_state=42)

    log_clf = LogisticRegression()
    log_clf.fit(X_train, y_train)
    print(log_clf.score(X_test, y_test))    # 0.864

    svm_clf = SVC()
    svm_clf.fit(X_train, y_train)
    print(svm_clf.score(X_test, y_test))    # 0.896

    dt_clf = DecisionTreeClassifier()
    dt_clf.fit(X_train, y_train)
    print(dt_clf.score(X_test, y_test))     # 0.84

    y_predict1 = log_clf.predict(X_test)
    y_predict2 = svm_clf.predict(X_test)
    y_predict3 = dt_clf.predict(X_test)

    # 对于二分类问题, 如果有两个预测值为1, 则预测为1
    y_predict = np.array((y_predict1 + y_predict2 + y_predict3) >= 2, dtype='int')
    print(accuracy_score(y_test, y_predict))    # 89.6%的准确率

手动地方式实现了一次集成学习


使用Voting Classifier

少数服从多数的方式就是 hard classifier

    voting_clf = VotingClassifier(estimators=[
        ('log_clf', LogisticRegression()),
        ('svm_clf', SVC()),
        ('dt_clf', DecisionTreeClassifier())
    ], voting='hard')

    voting_clf.fit(X_train, y_train)
    print(voting_clf.score(X_test, y_test))        # 89.6%的准确率

Soft Voting

少数服从多数的投票方式叫做Hard Voting

还有一个更重要的方式: Soft Voting


背后的想法也比较简单.

很多情况下, 少数服从多数是不合理的.

社会学, 政治学中有这样一个概念: 民主暴政

就是如果一人一票的话, 使用完全民主的方式, 有些时候反而是一个非常不好的结果

更合理的投票, 应该有权值


假设有5个模型, 面对一个二分类问题, 把数据分为A类和B类

对于一些算法, 它把某一个数据分为某一类的依据其实是计算概率

这里就是5个模型把一个数据分成A类和B类的概率

对于这样一个结果, 如果我们只看最终的结果, A-两票, B-三票

根据Hard Voting, 最终结果为B


但是看这个数据好像有点问题, 虽然只有两个模型把这个数据分成A类. 但是这两个模型近乎是非常确定, 认为这个模型就应该分为A类; 相应有3个模型把这个数据分为B类, 但是他们分类为B的概率并不高.

所以虽然分为A类的票数少, 但是它的权值应该更高一些.

所谓Soft Voting就是这样的一个考虑

计算分为每类的概率的平均值

根据这样的数字来看的话, 其实我们应该把结果分为A类. 这就是Soft Voting

换句话说, 它不仅要看把多少票投给A, 把多少票投给B, 还要看投给A的票到底有多大的概率确认分为A类, 看投给B的票到底有多大的概率确认分为B类


因此Soft Voting要求集合的每一个模型都能估计概率

逻辑回归就是根据一个概率模型创建的, 它有一个predict_proba方法, 有这个函数就说明是可以求出概率的

借此回顾一下分类算法是不是都可以估计概率


逻辑回归, 本身就是基于概率模型的


kNN, 根据离他最近的几个点进行分类, 通过投票kNN会把这个点预测为是红色的类, 此时它属于红色类的概率就是66% = 2/3

kNN也可以考虑权值, 考虑权值的话就看加权值所占的比重就可以了. 因此kNN算法也是支持计算概率的


决策树算法计算概率的方式和kNN算法计算概率的方式非常一致, 只不过样本在决策树中从根节点走到叶子节点, 在叶子节点中通常是包含多个类别的数据的, 它的基尼系数或信息熵不为0, 此时哪个类数据量大就分给哪个类.

这个概率和kNN算法是一样的, 占比率最大的节点的数量除以总结点的数量


最复杂的是SVM算法

对于SVM / SVC算法, 它本身是没有考虑概率的, 而是寻找一个统计上margin的最大值. 虽然如此, SVM算法依然可以使用一些方式来计算出概率

这种方式代价就是更多的计算资源.

具体是怎么计算出概率的就不涉及了, 不过scikit-learn中SVC有一个probability参数, 默认为False, 但是如果改为True, 就可以估计样本分到各个类别的概率


他们都可以直接或间接的算出样本分到各个类别的概率, 有这个概率就可以使用Soft Voting这种方式

voting_clf2 = VotingClassifier(estimators=[
    ('log_clf', LogisticRegression()),
    ('svm_clf', SVC(probability=True)),
    ('dt_clf', DecisionTreeClassifier(random_state=666))
], voting='soft')
voting_clf2.fit(X_train, y_train)
print(voting_clf2.score(X_test, y_test))  # 0.904

很多时候Soft Voting Classifier的效果是比Hard Voting Classifier效果好


这只是集成学习入门的方式, 集成不同的算法.

可是这里有一个很大的问题, 但是算法的种类再多也是非常有限的, 怎样才能聚集更多的分类器, 创建出更好的分类结果?



Bagging 和 Pasting

集成学习就是集成多个算法, 让不同的算法对用一组数据进行分析, 投票来看具体那个结果是更好的结果.

虽然有很多机器学习方法, 但是从投票的角度看, 仍然不够多

如果想要有一个好的结果, 我们就希望有更多的投票者, 成百上千乃至上万, 才能保证结果更加可信. (概率论中的大数定理)


所以要创建更多子模型, 来集成更多子模型的意见.

更加重要的是, 子模型之间不能一致! 子模型之间要有差异性


例如: 一共有500个样本数据, 每个子模型只看100个样本数据. 而相应的, 每个子模型使用的算法可以是同一个算法

这样就做到了创建很多的子模型, 而且这些子模型之间存在差异

这么做每个模型的准确率不就变得很低么? 确实如此, 但同时这也是集成学习算法的威力

集成学习集成诸多子模型来投票最终的分类结果, 在这个过程中, 每个子模型不需要太高的准确率

举个例子: 如果每个子模型只有51%的准确率. (不需要太高的准确率, 但好歹也要比平均水平要高)

假设分类任务整体的数据是无偏的

如果我们只有一个子模型, 整体准确率: 51%

如果有3个子模型, 判断正确有两种情况(三个模型都判断正确, 或任意两个模型判断正确) 整体准确率:
$$
0.51^3+C_3^2\cdot 0.51^2\cdot0.49=51.5%
$$
如果有500个子模型, (只要有250个以上的模型判断正确) 整体准确率:
$$
\sum_{i=251}^{500}C_{500}^i\cdot 0.51^i\cdot0.49^{500-i}=65.6%
$$
随着子模型数量的增加, 集成学习的整体模型的准确率也是在不断上升的


假设准确率达到了60%, 如果有500个子模型, 整体准确率就达到
$$
\sum_{i=251}^{500}C_{500}^i\cdot 0.6^i\cdot0.4^{500-i}=99.999%
$$
当然这只是理论情况, 同时我们也不能保证每个模型都有60%的准确率, 通过这样的计算可以看到集成学习的威力. 每个模型不需要太高的准确率, 但合在一起, 整体显著率就会非常高


这就是统计显著 (Statistically Significant),

根据 中心极限定理 可以算出z值:
$$
\frac {0.5-p} {\sqrt {\frac {p(1-p)}{n}}} \le z
$$
z值服从正态分布, p为预测正确的概率, n为样本数量, 在Z table中就可以找到对应的概率. 当p>0.5的时候, n越大这个式子的值就越小, 因此z在正态分布下右侧的面积就越大, 概率就越大.


如何创建差异性?

每个子模型只看样本数据的一部分. 我们怎么看样本数据的一部分?

我们有两种方式只看样本数据的一部分: 放回取样(Bagging) 和 不放回取样 (Pasting)

  • 放回取样(with replacement): 在500个样本中只看100个样本, 一个子模型看了100个样本之后把这100个样本放回去, 下一个子模型继续在这500个样本中随机找100个样本进行学习
  • 不放回取样(without replacement): 一个子模型看100个样本, 下一个子模型只能从剩下的400个样本中再挑100个样本进行学习. 也就是说每个子模型看到的样本不重复

通常来讲放回取样(Bagging)更常用

如果使用不放回取样, 500个样本, 每次看100个样本, 只能训练5个模型, 数量太少. 而Bagging可以训练成千上万的子模型

Bagging的第二个优点就是: 没有那么强地依赖随机, 如果使用Pasting训练5个样本, 其实就相当于把样本分成5份, 每一份有100个样本. 怎么分就会非常强烈地影响最终的结果. Bagging则是每次从500个样本中取100个样本进行学习, 这个学习过程重复非常多次, 其实一定程度将随机带来的问题给取消了


Bagging是机器学习中的叫法. 统计学中, 放回取样叫做bootstrap

scikit-learn中, 选择放回取样还是不放回取样只要调节bootstrap这个参数即可


import numpy as np
import matplotlib.pyplot as plt
from sklearn import datasets
from sklearn.ensemble import BaggingClassifier
from sklearn.model_selection import train_test_split
from sklearn.tree import DecisionTreeClassifier

if __name__ == '__main__':

    X, y = datasets.make_moons(n_samples=500, noise=0.3, random_state=42)
    X_train, X_test, y_train, y_test = train_test_split(X, y, random_state=42)

    # Bagging
    # 集成500个模型, 每个模型看100个数据, 放回取样.
    bagging_clf = BaggingClassifier(DecisionTreeClassifier(),
                                    n_estimators=500, max_samples=100,
                                    bootstrap=True)
    bagging_clf.fit(X_train, y_train)
    print(bagging_clf.score(X_test, y_test))    # 0.904

让每一个集成的算法都是决策树模型, 因为决策树这种非参数学习的方式更能产生差异比较大的子模型

所以要集成成百上千个子模型, 首选决策树

scikit-learn中bagging和pasting统一使用BaggingClassifier, 只不过通过bootstrap传入True或False来改变

n_estimators=5000时, 准确率能达到 0.912

理论上说, 子模型数量越多, 最终准确率就会越高. 只不过不会向上面所说达到99.999%, 这是因为具体的学习中还是和样本数据相关的. 如果样本存在噪音, 甚至存在错误的样本, 我们依然会犯更大的错误

不管怎么说这种策略通常都是有很好的结果的

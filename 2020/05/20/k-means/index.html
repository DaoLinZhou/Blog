kMeans算法以及其优化kMeans++算法

K Means

K-Means算法是无监督分类算法.

无监督学习: 机器学习的一种方法, 没有给定事先标记过的训练示例, 自动对输入数据进行分类或分群

无监督学习的优点:

  • 算法不受监督信息(偏见)的约束, 可能考虑到新的信息
  • 不需要标签数据, 极大程度扩大数据样本

主要应用: 聚类分析, 关联规则, 维度缩减


K Means 思想

把数据聚集到 k 个簇, 使得所有点到对应簇中心的距离和最小

因此损失函数就是
$$
E = \sum_{i=1}^k\sum_{x\in C_i}||x-\mu_i||_2^2
$$

损失函数是L2范数, 有空试试L1范数, 因为距离这个概念本身就是超参数

计算k个簇中, 每个簇(Ci) 中数据点(x) 到当前簇中心点(μi) 的距离和
$$
\mu_i = \frac 1 {|C_i|}\sum_{x\in C_i}x
$$
当前簇中心点的计算方式是: 当前簇所有数据点的和除以数据点的个数


然而要找到以上问题的最优解是一个NP难问题, 需要遍历所有可能的簇划分, K-Means算法采用了贪心策略, 通过迭代优化来求近似解.

算法流程如下:

  1. 在样本中随机取k个样本点 μ1,μ2,…μk 充当初始各个簇的中心点.
  2. 计算所有样本点与各个簇中心之间的距离, 把样本点划入最近的簇中.
  3. 根据簇中已有的样本点, 从新计算中心点 μ1,μ2,…μk
  4. 重复2, 3步, 直到收敛

代码实现

class KMeans:

    def __init__(self, k=1, distance='l2'):
        self.k = k
        self.center_points = None
        self.distance = distance

    def _distance(self, X1, X2):
        if self.distance == 'l1':
            return np.sum(np.abs(X1-X2), axis=1)
        elif self.distance == 'l2':
            return np.sum((X1-X2) ** 2, axis=1)

    # 随机选k个点作为初始点
    def _random_init_point(self, X):
        self.center_points = X[np.random.permutation(X.shape[0])[:self.k]]

    # 获取X数据集中的中心点, X中至少有一个值
    def _get_center(self, X):
        return np.sum(X, axis=0) / X.shape[0]

    # 根据旧的中心点进行聚类, 返回新的中心点
    def _clustering(self, X):
        # 根据旧的中心点进行分类
        y = self.predict(X)
        new_center_points = np.empty(self.center_points.shape)
        for i in range(self.k):
            # 确保簇中至少有一个元素才求中心点
            if np.sum(y==i) != 0:
                new_center_points[i, :] = self._get_center(X[y == i, :])
        return new_center_points

    def fit(self, X, y=None, n_iters=int(1e9), epsilon=1e-7):
        self._random_init_point(X)
        dist_change = 1     # 初始化随便选, 只要比epsilon大即可
        for _ in range(n_iters):
            if dist_change < epsilon:
                return
            # 获取新的中心点
            new_center_points = self._clustering(X)
            # 计算新的中心点的移动距离
            dist_change = np.sum(self._distance(self.center_points, new_center_points))
            # 更新中心点
            self.center_points = new_center_points

    # 根据中心点把X分为k类, 0, 1, 2, ... k-1类
    def predict(self, X):
        # k列, 第j行第i列代表: 第j个元素距离第i个中心点的距离
        dist = np.zeros((X.shape[0], self.k))
        for i in range(0, self.k):
            dist[:, i] = self._distance(X, self.center_points[i])
        # 选择所有距离中最小的那个节点作为输出标记
        return np.argmin(dist, axis=1)

计算距离时不开根也没有关系, 因为我们并不是要用距离计算某个具体的值
而是使用相对的距离来判断分类的结果, 所以x^2大, |x|自然就大
只要距离能表示点与点之间的远近关系就行, 不用太精确

使用欧拉距离(l2)得到分类结果

使用曼哈顿距离(l1)得到的分类结果


但是K Means有一个非常重要的缺点: 初始随机的中心点会严重影响分类结果


KMeams的优缺点

K-Means算法的优点:

  1. 速度快
  2. 对大数据集有较高的效率

缺点:

  1. k要事先指定, 大多数时候我们是不知道要指定多少
  2. 初始中心点严重影响分类结果
  3. 算法计算分类中心时要对所有数据进行k次求模运算, 当样本量非常大的时候, 时间开销较大

KMeans ++

为什么会有这样的问题?

很大一个原因就是两个初始的簇中心点太近了. 一个改进的思想就是: 尽可能远的选取聚类中心

距离当前已有的聚类中心更远的点更有概率被选为下一个聚类中心


因此只要改进初始中心点的选取即可.

  1. 从数据集中随机选一个初始的样本点作为第一个簇中心 C1
  2. 计算剩余样本点作为新的簇中心的概率
  3. 选取根据这个概率抽取一个点作为下一个簇中心点
  4. 重复2, 3直到找到k个中心点

问题就变成了怎么求剩余样本点作为新的簇中心的概率.

当这个点距离已有的任何一个中心点越近, 它就越不可能是新的中心点;

因此可以用这个点到离它最近的中心点的距离除以总距离表示概率

点到离它最近的中心点的距离
$$
D(x^{(i)})=min(dist(x^{(i)}, C_1), …dist(x^{(i)}, C_n))
$$
点作为簇中心的概率
$$
P(x^{(i)})=\frac {D(x^{(i)})^2} {\sum D(x^{(j)})^2}
$$

新版本的KMeans

class KMeans:

    # init_center_points 为mean-shift做准备, 因为如果已知中心点大体位置就可以更准确地进行分类
    def __init__(self, k=1, init="k-means++", distance='l2', init_center_points=None):
        assert init in ["k-means", "k-means++"], \
            "init must be k-means or k-means++"
        assert distance in ['l1', 'l2'], \
            "distance must be l1 or l2"
        assert k >= 1, "k must an integer and >= 1"
        if init_center_points is not None:
            assert k == init_center_points.shape[0], \
                "number of init_center_points must equal to k"
        self.k = k
        self.init = init
        self.center_points_ = init_center_points
        self.distance = distance

    def _distance(self, X1, X2):
        if self.distance == 'l1':
            return np.sum(np.abs(X1 - X2), axis=1)
        elif self.distance == 'l2':
            return np.sum((X1 - X2) ** 2, axis=1)

    # 选k个点作为初始点
    def _random_init_point(self, X):
        if self.init == "k-means":
            self.center_points_ = X[np.random.permutation(X.shape[0])[:self.k]]
        elif self.init == "k-means++":
            n_sample, n_feature = X.shape
            self.center_points_ = np.empty((self.k, n_feature))
            self.center_points_[0, :] = np.array(X[np.random.randint(0, n_feature), :])

            for init_point_i in range(1, self.k):
                prop = np.empty(n_sample)
                for i in range(n_sample):
                    D2 = np.min(self._distance(self.center_points_[:init_point_i, :], X[i, :]))
                    prop[i] = D2
                prop = prop / sum(prop)
                # 通过计算概率随机抽取下一个中心点
                next_center_index = np.random.choice(n_sample, p=prop)
                self.center_points_[init_point_i, :] = X[next_center_index, :]

    # 获取X数据集中的中心点
    def _get_center(self, X):
        return np.sum(X, axis=0) / X.shape[0]

    # 根据旧的中心点进行聚类, 更新新的中心点
    def _clustering(self, X):
        # 根据旧的中心点进行分类
        y = self.predict(X)
        new_center_points = np.empty(self.center_points_.shape)
        for i in range(self.k):
            if np.sum(y == i) != 0:
                new_center_points[i, :] = self._get_center(X[y == i, :])
        return new_center_points

    def fit(self, X, y=None, n_iters=int(1e9), epsilon=1e-7):
        if self.center_points_ is None:
            self._random_init_point(X)
        dist_change = 1  # 初始化随便选, 只要比epsilon大即可
        for _ in range(n_iters):
            if dist_change < epsilon:
                return
            new_center_points = self._clustering(X)
            dist_change = np.sum(self._distance(self.center_points_, new_center_points))
            # 更新中心点
            self.center_points_ = new_center_points

    # 根据中心点把X分为k类, 0, 1, 2, ... k-1类
    def predict(self, X):
        # k列, 第j行第i列代表: 第j个元素距离第i个中心点的距离
        dist = np.zeros((X.shape[0], self.k))
        for i in range(0, self.k):
            dist[:, i] = self._distance(X, self.center_points_[i])
        return np.argmin(dist, axis=1)

为什么大多数实现要用概率随机抽取的方式选下一个中心点, 而不是直接取最远的那个点(也就是概率最大的点)作为中心点?

因为概率最大的点也就是距离最远的点很可能是outlier, 使用抽取的方式更有可能选到簇群中的点


K means 还有一种改进思路是ISODATA, 它支持动态合并/分裂中心点.

但是它所需要更多的超参数, 所以在实际的使用中并没有k means++受欢迎

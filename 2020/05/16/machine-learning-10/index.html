决策树算法的特点

总结一下决策树:

  • 非参数学习算法
  • 可以解决分类问题
  • 天然可以解决多分类问题
  • 也可以解决回归问题 (依然是创建这样一颗树, 经过每个节点的判断原则就会落到叶子节点, 可以用落在叶子节点中所有的样本数据的平均值当作回归问题的预测值)
  • 有非常好的可解释性

既然了解了什么是决策树, 问题就变成怎么构建决策树

每一个节点都首先找到一个维度, 其次找到这个维度中的一个阈值, 以这个维度的这个阈值为依据判断继续进行划分.

关键的问题其实就是:

  1. 每个节点到底在哪个维度做划分?

  2. 某个维度的哪个值上做划分?



信息熵

信息熵是信息论的一个基础概念

熵在信息论中代表: 随机变量不确定度的度量

  • 熵越大, 数据的不确定性越高, 数据越随机

  • 熵越小, 数据的不确定性越低

其实熵这个概念是从物理-热力学中引申出的概念

也可以这样理解, 熵越大, 粒子无规则地运动越剧烈, 不确定性越高; 熵越小, 粒子越倾向与静止的状态, 不确定性越低.


信息熵的计算公式: 香农提出的
$$
H=-\sum_{i=1}^k\ p_i\ log(p_i)
$$
一个系统信息熵的总和是这样计算的, pi是什么意思?

就是对于一个系统中可能有k类的信息, 每一类信息所占比例就叫pi

例如鸢尾花数据集中有3种鸢尾花, 每种鸢尾花所占比例可能为1/3, 那么p1, p2, p3就分别等于1/3

为什么有一个负号? 因为pi一定是一个小于1的数, 由于log函数中如果传入的参数小于1, log(x) 的结果就会是负数, 再加上这个负号就变成大于0的数.

所以最终计算出来的熵(H)肯定是大于0的


$$ H=-\sum_{i=1}^k\ p_i\ log(p_i) $$ 但是这个公式还是太抽象, 我们举几个例子:

显然右边的数据的信息熵比左边的信息熵小.

熵代表不确定性的度量, 熵越小, 不确定度就越低, 确定性越强

如果熵越大, 系统就越不确定, 越随机.


这么看就是说: 右边的数据比左边的数据更确定

由于右边的数据很多数据都在第三类里, 占70%之多, 所以右边数据是更确定的. 而左边数据每个类别各占1/3, 所以数据不确定性越强.

按照设个思路在强一点: 如果第一类数据占100%, {1, 0, 0}
$$
H=-1\cdot log(1)=0
$$
这个信息熵就是这个公式的最低值. 放进信息熵的物理定义中就是: 此时数据的不确定性是最低的, 它是最确定的 (所有数据都在一类中没有任何不确定性)


$$ H=-\sum_{i=1}^k\ p_i\ log(p_i) $$ 以二分类问题为例, 感性地看一下信息熵的公式是什么样子的.

如果数据类别是两类, 一类是x, 另一类占得比例就是1-x, 此时信息熵的公式就是:
$$
H=-xlog(x)-(1-x)log(1-x)
$$

import numpy as np
import matplotlib.pyplot as plt

def entropy(p):
    return -p * np.log(p) - (1-p) * np.log(1-p)

if __name__ == '__main__':
    x = np.linspace(0.01, 0.99, 200)
    y = entropy(x)

    plt.plot(x, y)
    plt.show()

这样一根曲线像是抛物线的样子, 且这个曲线的对称轴是0.5

换句话说, 当x=0.5, 曲线取到了最大值, 相当于对于信息熵来说, 两个类别都是0.5时, 整个信息熵是最大的

此时的数据是最不稳定的, 确定性最低. 此时数据到底是第一类还是第二类各有50%的可能性, 所以是最不确定的.

无论是x值更小还是更大, 整个数据的信息熵都在降低. 无论更小还是更大我们的数据都更偏向某一类, 数据整体确定性更高了, 信息熵就变低了

如果系统中有3类, 绘制的就是立体的曲面


如果系统偏向某一类, 就相当于一定程度有了确定性


  1. 每个节点到底在哪个维度做划分?

  2. 某个维度的哪个值上做划分?

当有信息熵的概念, 这两个问题就好解决了.

我们希望在每一个节点上都希望在某一个维度上基于某一个阈值划分

我们在划分以后, 要做到事情, 就是要让划分为两部分之后相应的系统的整体信息熵降低, 换句话说让整个系统变得更加确定

极端情况下, ABC三个叶子节点每个叶子节点中的数据只属于A类, B类, C类的话, 此时整个系统的信息熵就达到了0.

我们要找在每一个节点上某一个维度的取值, 根据这个维度的这个取值进行划分使得总体信息熵比其他划分方式的信息熵都小. 我们称这样的划分就是当前最好的划分


怎么找到这样的划分? 对所有划分可能性进行一次搜索即可

使用信息熵寻找最优划分

def entropy(y):
    counter = Counter(y)
    res = 0.0
    for num in counter.values():
        p = num / len(y)
        res += -p * log(p)
    return res


# 基于维度d的value值进行划分
def split(X, y, d, value):
    # 维度d的value <= value
    index_a = (X[:, d] <= value)
    index_b = (X[:, d] > value)
    return X[index_a], X[index_b], y[index_a], y[index_b]


# 尝试划分
def try_split(X, y):
    best_entropy = float('inf')
    best_d, best_v = -1, -1     # 初始随便取值
    for d in range(X.shape[1]):
        # 阈值可选的值就是每两个样本点在d这个维度上中间的值
        sorted_index = np.argsort(X[:, d])
        # 每次寻找i-1和i在d这个维度的中间的值
        for i in range(1, len(X)):
            if X[sorted_index[i-1], d] != X[sorted_index[i], d]:
                v = (X[sorted_index[i-1], d] + X[sorted_index[i], d]) / 2
                X_l, X_r, y_l, y_r = split(X, y, d, v)
                # 计算两个区域的信息熵的和
                e = entropy(y_l) + entropy(y_r)
                if e < best_entropy:
                    best_entropy, best_d, best_v = e, d, v
    return best_entropy, best_d, best_v


if __name__ == '__main__':
    iris = datasets.load_iris()
    # 保留后两个特征
    X = iris.data[:, 2:]
    y = iris.target

    best_entropy, best_d, best_v = try_split(X, y)
    print(best_entropy)  # 熵: 0.6931
    print(best_d)        # 维度: 0
    print(best_v)        # 阈值: 2.45
    X1_l, X1_r, y1_l, y1_r = split(X, y, best_d, best_v)
    print(entropy(y1_l))       # 信息熵为0, 把蓝色数据全都化为左边
    print(entropy(y1_r))       # 信息熵为0.69

    best_entropy2, best_d2, best_v2 = try_split(X1_r, y1_r)
    print(best_entropy2)  # 熵: 0.4132
    print(best_d2)        # 维度: 1
    print(best_v2)        # 阈值: 1.75
    X2_l, X2_r, y2_l, y2_r = split(X1_r, y1_r, best_d2, best_v2)
    print(entropy(y2_l))       # 信息熵为0.3084
    print(entropy(y2_r))       # 信息熵为0.1047

可以看到

  • 第一次决策节点为: 维度=0, 阈值=2.45
  • 第二次决策节点为: 维度=1, 阈值=1.75

尝试实现一个简单的基于熵的决策树

from math import log
from collections import Counter

import numpy as np
import matplotlib.pyplot as plt
from matplotlib.colors import ListedColormap
from sklearn import datasets
from sklearn.tree import DecisionTreeClassifier


def plot_decision_boundary(model, axis):
    x0, x1 = np.meshgrid(
        np.linspace(axis[0], axis[1], int((axis[1] - axis[0]) * 100)).reshape(-1, 1),
        np.linspace(axis[2], axis[3], int((axis[3] - axis[2]) * 100)).reshape(-1, 1),
    )
    X_new = np.c_[x0.ravel(), x1.ravel()]
    y_predict = model.predict(X_new)

    zz = y_predict.reshape(x0.shape)
    custom_cmap = ListedColormap(['#EF9A9A', '#FFF59D', '#90CAF9'])

    plt.contourf(x0, x1, zz, cmap=custom_cmap)


class Node:
    # res 表示判断完成后的类别, next是子节点用于继续判断
    def __init__(self, d, v, res, left_is_small):
        self.d = d
        self.v = v
        self.res = res
        # Ture, 则判断<=v; False, 则判断>v
        self.left_is_small = left_is_small
        self.next = None


class DecisionTree:

    def __init__(self, max_depth):
        self.max_depth = max_depth
        self.root = None

    def fit(self, X_train, y_train):
        X_r = X_train
        y_r = y_train

        self.root = Node(-1, -1, -1, True)
        cur_node = self.root

        for _ in range(self.max_depth):
            e, d, v = self.try_split(X_r, y_r)
            X_l, X_r, y_l, y_r = self.split(X_r, y_r, d, v)
            left_is_small = True
            if self.entropy(y_l) > self.entropy(y_r):
                X_l, X_r, y_l, y_r = X_r, X_l, y_r, y_l
                left_is_small = False
            cur_node.next = Node(d, v, Counter(y_l).most_common(1)[0][0], left_is_small)
            cur_node = cur_node.next
        cur_node.next = Node(d, v, Counter(y_r).most_common(1)[0][0], not left_is_small)
        self.root = self.root.next

    def predict(self, X):
        return np.array([self._predict(x) for x in X], dtype='int')

    def _predict(self, x):
        cur_node = self.root
        while cur_node:
            if cur_node.left_is_small:
                if x[cur_node.d] <= cur_node.v:
                    return cur_node.res
            else:
                if x[cur_node.d] > cur_node.v:
                    return cur_node.res
            cur_node = cur_node.next
        return -1


    def entropy(self, y):
        counter = Counter(y)
        res = 0.0
        for num in counter.values():
            p = num / len(y)
            res += -p * log(p)
        return res

    # 基于维度d的value值进行划分
    def split(self, X, y, d, value):
        # 维度d的value <= value
        index_a = (X[:, d] <= value)
        index_b = (X[:, d] > value)
        return X[index_a], X[index_b], y[index_a], y[index_b]

    # 尝试划分
    def try_split(self, X, y):
        best_entropy = float('inf')
        best_d, best_v = -1, -1  # 初始随便取值
        for d in range(X.shape[1]):
            # 阈值可选的值就是每两个样本点在d这个维度上中间的值
            sorted_index = np.argsort(X[:, d])
            # 每次寻找i-1和i在d这个维度的中间的值
            for i in range(1, len(X)):
                if X[sorted_index[i - 1], d] != X[sorted_index[i], d]:
                    v = (X[sorted_index[i - 1], d] + X[sorted_index[i], d]) / 2
                    X_l, X_r, y_l, y_r = self.split(X, y, d, v)
                    # 计算两个区域的信息熵的和
                    e = self.entropy(y_l) + self.entropy(y_r)
                    if e < best_entropy:
                        best_entropy, best_d, best_v = e, d, v
        return best_entropy, best_d, best_v

    # 保留后两个特征

    y = iris.target

    dt_clf = DecisionTree(max_depth=2)

    plt.scatter(X[y == 2, 0], X[y == 2, 1])
    plt.show()

基尼系数

之前注意介绍使用信息熵来对节点进行划分,其实还可以使用另外一个指标来划分: 基尼系数
$$
G=1-\sum_{i=1}^k p_i^2
$$
这个式子和信息熵是有同样的性质的


同样举几个例子来看

用样, 基尼系数越高意味着数据整体随机性越强; 基尼系数越低意味着数据整体不确定性越弱


同样可以用一根曲线来看一下是什么样子, 依然是使用二分类的方式, 此时基尼系数就是:
$$
G=1-x^2-(1-x)^2
$$

$$
= -2x^2+2x \ \ \ \ \ \ \
$$

此时基尼系数就是抛物线, 开口向下, 最大值就是在抛物线对称轴的地方 x=0.5

dt_clf = DecisionTreeClassifier(max_depth=2, criterion="gini")

所做的只是把criterion改为gini

而得到的图和使用信息熵是差不多的. 通常二者结果区分不大


模拟使用基尼系数划分

def gini(y):
    counter = Counter(y)
    res = 1.0
    for num in counter.values():
        p = num / len(y)
        res -= p**2
    return res


def split(X, y, d, value):
    # 维度d的value <= value
    index_a = (X[:, d] <= value)
    index_b = (X[:, d] > value)
    return X[index_a], X[index_b], y[index_a], y[index_b]


# 尝试划分
def try_split(X, y):
    best_g = float('inf')
    best_d, best_v = -1, -1     # 初始随便取值
    for d in range(X.shape[1]):
        sorted_index = np.argsort(X[:, d])
        for i in range(1, len(X)):
            if X[sorted_index[i-1], d] != X[sorted_index[i], d]:
                v = (X[sorted_index[i-1], d] + X[sorted_index[i], d]) / 2
                X_l, X_r, y_l, y_r = split(X, y, d, v)
                g = gini(y_l) + gini(y_r)
                if g < best_g:
                    best_g, best_d, best_v = g, d, v
    return best_g, best_d, best_v

    plt.scatter(X[y == 2, 0], X[y == 2, 1])
    plt.show()

    best_g, best_d, best_v = try_split(X, y)
    print(best_g)  # 基尼系数: 0.5
    print(best_d)  # 维度: 0
    print(best_v)  # 阈值: 2.45
    X1_l, X1_r, y1_l, y1_r = split(X, y, best_d, best_v)
    print(gini(y1_l))  # 基尼系数=0, 把蓝色数据全都化为左边

    best_g2, best_d2, best_v2 = try_split(X1_r, y1_r)
    print(best_g2)  # 基尼系数: 0.2105
    print(best_d2)  # 维度: 1
    print(best_v2)  # 阈值: 1.75

    print(gini(y2_r))  # 基尼系数为0.0425

信息熵 VS 基尼系数

信息熵的计算比基尼系数稍慢, 因为信息熵要计算log

因此scikit-learn中默认为基尼系数

大多数时候, 二者没有特别的效果优劣

的确我们可以把”选择使用信息熵”或”选择使用基尼系数”作为超参数, 但很少有情况是其中一个效果非常好, 而另一个效果较差

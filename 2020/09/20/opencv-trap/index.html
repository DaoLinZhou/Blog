imshow和imwrite结果不同

imwrite always expects [0,255], whereas imshow expects [0,1] for floating point and [0,255] for unsigned chars.

def test():
    img = cv2.imread("../resources/cards.jpg")
    img = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)

    lapkern = np.array([[0, -1, 0],
                        [-1, 4, -1],
                        [0, -1, 0]])
    lap = conv2(img, lapkern, mode="same")    # 在图像学的那篇文章中我实现了conv2
    cv2.imshow("test1", lap.astype(float))
    cv2.imwrite("test2.png", lap)

期望显示的图片

实际显示的图片:


然而实际保存的图片却是期望的图片.


这是因为imshow和imwrite的期望值不同.

要解决这个问题, 有两种方法

  1. 由于imshow和imwrite都支持[0, 255]的整型, 所以我们就把它们转化为整型进行存储和展示
# 一定要进行一次过滤, 不然.astype(np.unit8)会有整型溢出, -1 会变为 255, 256会变为-1
lap[lap < 0] = 0
lap[lap > 255] = 255

cv2.imshow("test1", lap.astype(np.uint8))
cv2.imwrite("test1.png", lap.astype(np.uint8))

  1. 第二种方法就是imshow的参数用[0, 1]之间的小数, imwrite的参数用[0, 255]

此时就不需要过滤了, imshow和imwrite会自动过滤

(第一种方法不能自动过滤是因为我们在过滤前就溢出了)

cv2.imshow("test2", lap/255)
cv2.imwrite("test2.png", lap)

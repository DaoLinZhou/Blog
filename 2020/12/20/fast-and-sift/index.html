特征检测算法 FAST 和 SIFT

FAST

OpenCV Tutorial

算法思想比较简单

就是对一个像素与它周围16个点进行对比, 如果周围有连续12(有时候也可以选择9)个像素一定程度大于或小于这个像素, 就说明这个点是一个特征点

import cv2
import numpy as np
from scipy.signal import convolve2d

# neighbors' offset from p
# fellow this order: https://opencv-python-tutroals.readthedocs.io/en/latest/_images/fast_speedtest.jpg
d = [[-3, 0], [-3, 1],  # 1, 2
     [-2, 2],  # 3
     [-1, 3], [0, 3], [1, 3],  # 4, 5, 6
     [2, 2],  # 7
     [3, 1], [3, 0], [3, -1],  # 8, 9, 10
     [2, -2],  # 11
     [1, -3], [0, -3], [-1, -3],  # 12, 13, 14
     [-2, -2],  # 15
     [-3, -1]]  # 16

sobel = np.array([[-1, 0, 1],
                  [-2, 0, 2],
                  [-1, 0, 1]])


# 获取一个点的周围16个neighbors
def get_neighbors(img: np.ndarray, x: int, y: int) -> np.ndarray:
    """
    :param img: image
    :param x: row number
    :param y: column number
    :return: the 16 neighbors ordered from 1-16
    """
    return np.array([img[x + offset[0], y + offset[1]] for offset in d], dtype=int)


# 快速判断一个点不是 feature point
def high_speed_test(img: np.ndarray, x: int, y: int, t):
    """
    :param img:
    :param x: row number
    :param y: column number
    :param t: threshold
    :return:  0 if test fail
              1 if 3 of them are too bright
             -1 if 3 of them are too dark
    """
    intensity = img[x, y]
    # points contains neighbors[1, 5, 9, 13]
    points = np.array([img[x + d[i << 2][0], y + d[i << 2][1]] for i in range(4)])
    if np.sum(points >= (intensity + t)) >= 3:
        return 1
    elif np.sum(points <= (intensity - t)) >= 3:
        return -1
    else:
        return 0


# 通过 neighbors 和 当前 pixel 的 intensity 以及给定的 threshold 判断一个点是否是特征点
def FAST(neighbors: np.ndarray, intensity, t, darker: bool):
    if darker:
        tmp = neighbors <= (intensity - t)
    else:
        tmp = neighbors >= (intensity + t)
    if np.sum(tmp) < 12:
        return False
    tmp2 = np.concatenate([tmp, tmp]).tostring()
    return tmp2.find(np.ones(12, dtype=bool).tostring()) != -1


def my_fast_detector(img: np.ndarray, t=10):
    r, c = img.shape
    indexes = np.zeros((r, c), dtype=int)
    kps = []
    for i in range(r):
        for j in range(c):

            if i < 3 or i + 3 >= r or j < 3 or j + 3 >= c:
                # if (i, j) doesn't have all neighbors
                continue

            test_res = high_speed_test(img, i, j, t)
            if test_res == 0:
                # high speed test fails
                continue

            neighbors = get_neighbors(img, i, j)
            if FAST(neighbors, img[i, j], t, test_res == -1):
                indexes[i, j] = 1
                kps.append([j, i])
    return indexes, kps


def robust_by_harris_cornerness_metric(img: np.ndarray, indexes: np.ndarray):
    gauss = cv2.getGaussianKernel(5, 1)
    gauss = gauss.dot(gauss.T)
    dog = convolve2d(gauss, sobel)

    ix = cv2.filter2D(img, -1, dog)
    iy = cv2.filter2D(img, -1, dog.T)

    ix2g = cv2.filter2D(ix**2, -1, gauss)
    iy2g = cv2.filter2D(iy**2, -1, gauss)
    ixiyg = cv2.filter2D(ix*iy, -1, gauss)

    harcor = ix2g * iy2g - ixiyg**2 - 0.05 * ((ix2g + iy2g) ** 2)
    return np.bitwise_and(harcor > 0.001, indexes)


if __name__ == '__main__':
    img = cv2.imread("./S2-im1.png")
    img_gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)

    indices, kps = my_fast_detector(img_gray)
    robust_indexes = robust_by_harris_cornerness_metric(img_gray, indices)

    cv2.imwrite("kps.png", robust_indexes*255)
    cv2.waitKey(0)



SIFT

SIFT(Scale Invariant Feature Transform)

这个算法为了实现Scale Invariant, 使用高斯金字塔模拟摄像头的远近来scale image. 在scale后的image中寻找特征点.

因为原图中的一个特征点可能会因为缩放变为一条边

每个 octave 是一组.

octave 1 是对原图进行不同高斯过滤(使用不同$\sigma$ 的高斯核)后的结果

octave 2 的第一张图是对 octave 1 进行 subsampleing 后的结果. 之后重复使用高斯核来得到octave中其他的图片.


那么一共要几个octave? David 的论文中使用 $O=[\log_2(min(M,N))]-3$

为什么?

因为我们进行subsampling就是把图片长和宽都缩小2倍, 因此O从原图的长和宽的最小值中取$\log_2$是合理的结果, 然而考虑到如果图片太小就得不到有意义的信息. 因此舍去最小的三个octave.

对高斯金字塔中同一个octave中所有层求差值, 得到高斯差分金字塔.

高斯差分金字塔比原本高斯金字塔每个octave少一层. 而我们要对高斯差分金字塔的中间层求极值. 因此要对比一个像素的周围26个像素.(如图所示)

因此最上层和最下层无法求极值, 又有2层无法使用. 因此如果说高斯金字塔每个octave的层数是S, 我们希望每个octave中有n层用于计算极值, 那么 $S=n+3$


sigma的取值:

sigma0 为什么等于 1.52?

当相机进行拍照时会进行一次blur. David 认为此时blur的sigma为0.5. 如果我们想要让第一个octave的第一张图blur的sigma为1.6的话, 就要让sigma0位1.52, 因为高斯核的sigma满足$a^2+b^2=c^2$的性质

octave2取octave1的倒数第3个image进行subsampling, 因为这一层的sigma 是$2\sigma_0, k^n=2$

octave中的每个image都是对原图的blur. 不同的sigma对应不同程度的blur.

而不同程度的模糊对一些极值点来说影响较大, 对普通的点影响较小, 因此求差分才显得有意义(个人理解)


在差分金字塔中求得的极值点再反向计算它在原图中的位置. 我们就得到了临时的特征点.

此时还可以用泰勒展开求精确的极值点. (需要矩阵求导,这块我还没太理解)

最后用 Harris 矩阵来舍弃一些不好的极值点(舍弃在边上的极值点)

这样就得到最终的特征点了.



匹配

通过 FAST 和 SIFT都求得了特征点

之后就可以通过这些特征点和原图求他们的特征描述符, 在通过匹配特征描述符来匹配两张图片

机器学习的主要任务

机器学习关注的任务主要有两类: 分类任务, 回归任务


分类任务

例如, 给一张图片, 区分里面的动物是猫还是狗

例如, 识别手写数字


对于分类任务可以再进行细分, 主要分为这样几类任务

二分类任务

二分类任务, 二选一的任务, 例如判断是猫是狗

生活中很多任务使用二分类就可以了: 判断邮件是不是垃圾邮件, 判断发给客户信用卡是否有风险, 判断肿瘤是良性还是恶性


多分类任务

把结果锁定多个中的一个而不是两个中的一个, 手写数字识别就是从10个种类中选择一个

生活中的应用: 数字识别, 图像识别

一些看似复杂的任务, 也可以转换成分类任务, 例如2048

最终机器要做到的就是: 给出现在盘面的状况机器判断上移下移左移右移即可(4个中选一个)

同理对于更加复杂的游戏也是如此, 例如下围棋. 毕竟棋盘上落子的地方有限的, 所以每一步机器要做的事情就是根据现在的棋盘信息选择一个地方进行落子

甚至无人车领域也可以转化为分类任务, 根据车周围环形的图像信息机器进行选择方向盘向左还是向右, 以及旋转多少角度

即使配合上踩下油门和刹车力度的不同. 方向盘, 油门, 刹车他们的组合也是有限的, 在这些组合中选择一个, 因此我们也可以把它转换成一个多分类问题

注意: 只是说这些问题可以转换为多分类问题, 但不是说用多分类解决这些问题是最好的方案

有一些算法只支持完成二分类的任务

但是一个多分类的任务我们其实是可以转换成二分类的任务的, 这种转化的方式还不止一个

有一些算法天然可以完成多分类任务


多标签分类

不过更高级的分类任务可以让我们把一张图片分到多个类别中

这一类任务被称为回归任务

回归任务

结果是一个连续数字的值, 而非一个类别

例如: 预测房屋价格, 预测学生乘积, 市场分析, 股票价格

有一些算法只能解决回归问题, 有一些算法只能解决分类问题, 有一些算法的思路既能解决回归问题, 又能解决分类问题

某些情况下, 回归任务是可以简化为分类任务的

依然用无人驾驶举例, 之前说它是分类问题, 但是方向盘打的角度其实是连续的数值, 只是我们将他类别化(每一度看作是一个类别), 这样一来我们就将一个回归任务简化成一个分类任务

比如说预测学生的成绩, 其实是回归任务, 但有时候我们并不需要预测学生的具体成绩, 只需要看成绩是A+, A, B, C-等等, 这样也把一个回归的任务变成了分类的任务

监督学习, 非监督学习, 半监督学习 和 增强学习

分类和回归这两类任务其实是从机器学习可以解决的问题的角度分类的, 而不是从机器学习算法本身进行分类的

如果我们将目光放在机器学习算法上, 整体上我们可以将机器学习算法分成 监督学习, 非监督学习, 半监督学习 和 增强学习 这四个大类


监督学习

给机器的训练数据拥有 “标记” 或者 “答案”

例如上面提到的鸢尾花和房产的数据集, 都拥有”标记”列

比如分辨照片是猫还是狗只给出照片是不够的, 还要分辨这个照片到底是猫还是狗

同样对于mnist数据集, 不仅要有图片信息, 还要有标记信息


监督的意思就在于我们人类已经针对给机器的数据进行了正确答案的划分. 这种正确答案的划分本身就被称为是一种监督的信息

实际思考一下生活中很多学习的过程都是监督学习的过程

文章中大部分算法都属于监督学习算法


非监督学习

给机器的训练数据没有任何”标记”或者”答案”

每一个点并没有表示类别, 非监督学习就是基于相应的算法可以为这些点进行分类

通常管这样的工作叫做聚类分析

其实电商网站对每一个人都是同等对待的, 初始的时候是没有标记的, 但是随着在电商网站上留下越来越多的足迹, 电商网站就可以用非监督学习的方式对用户进行分类


非监督学习的另一个重要应用就是对数据进行降维处理

对数据进行降维处理主要分为两部分内容:

  • 特征提取: 信用卡的信用评级和人的胖瘦无关?

    数据通常有很多特征, 有一些特征对我们将要预测/计算的结果没有贡献

  • 特征压缩: PCA

特征压缩并不丢弃任何特征

特征压缩做的事情就是: 尽量少的损失信息的情况下, 将高维的特征向量压缩成低维的特征向量


对数据进行降维还有一个重要的意义: 方便可视化

还可以进行异常检测

非监督学习通常用作辅助监督学习


半监督学习

一部分数据有”标记”或者”答案”, 另一部分数据没有

这种情况更为常见, 因为生活中会因为各种原因产生标记缺失

通常都先使用无监督学习手段对数据做处理, 之后使用监督学习手段做数据的训练和预测

半监督学习就是监督学习和非监督学习的结合, 相对是容易的


增强学习

根据周围环境的情况, 采取行动, 根据采取行动的结果, 学习行动方式

算法通常叫做一个Agent, 要根据周围的环境来采取行动, 行动之后算法会收到一些反馈, 这些反馈有的时候会使用 reward(奖赏) 的机制, 有的时候是使用惩罚的机制进行反馈.

当Agent得到反馈之后, 它就会相应地改变自己的行为模式, 之后根据该机的算法在做出一个Action(动作), 环境再反馈Agent以此类推

在循环中增强自己的智能, 所以叫增强学习

增强学习是非常适合机器人的, AlphaGo实际上是增强学习的一个应用, 它在和人或者自己博弈中每一次落子其实都是和环境进行了一次交互, 此时AlphaGo就会思考刚才的落子是好还是不好

无论是无人驾驶还是机器人其实都需要增强学习

同样增强学习是比较前沿的学习领域, 监督学习和半监督学习是基础


然而监督学习, 非监督学习, 半监督学习 和 增强学习这四个大类并不是对机器学习分类的唯一标准

批量学习

之前所介绍的算法如果没有特殊说明都可以使用批量学习的方式来进行机器学习

所谓批量学习就是首先收集一定量的样本资料, 然后将这些样本资料送给机器学习算法, 算法就会训练出一个模型

当我们训练出这个模型之后就可以直接将模型投入生产环境中了

在具体应用的时候可能有新的样例送进来, 让模型预测出相应的结果, 这个过程中我们的模型不会再发生任何变化

例如: 手写识别系统, 在训练好模型后投入实践, 虽然在使用的过程中依然会投入大量新的样例, 但这些样例不会作为训练集来优化模型了


批量学习的特点

优点: 简单, 只需要学习机器学习算法本身就好了, 不需要考虑当算法投入生产环境中以后, 面对新的数据我们怎样来逐步优化算法

问题: 如何适应环境的变化? 例如在不同年代, 垃圾邮件的标签不同

解决方案: 定时重新批量学习, 让新来的样本数据和原来的样本融合在一起, 形成新的数据集, 然后在新的数据集上重新执行算法, 重新训练模型

缺点: 每次重新批量学习, 运算量巨大. 在某些环境变化非常快的情况下, 甚至不可能


在线学习

整体流程不变, 不过每次输入样例进入模型得到预期的结果之后同时我们输入样例并不浪费掉, 我们将输入样例和正确结果迭代进机器学习算法来改进模型

在线学习相当不不仅仅使用模型得到想要的结果, 与此同时我们还在不断地训练算法让算法进行学习, 所以叫做在线学习


在线学习的特点

优点: 及时反映新的环境变化

问题: 新的数据带来不好的变化, 例如错误地使用系统产生的异常数据

解决方案: 需要加强对数据进行监控 (讲非监督学习时说过, 有一个使用场景就是进行异常数据检测), 通常在在线学习的过程中, 就需要对异常数据进行检测

其他: 也适用于数据量巨大, 完全无法批量学习的环境 (在线学习算法也是对批量学习算法的一个改进)


从另外一个维度, 我们还可以把机器学习分为:

  • 参数学习 Parametric Learning
  • 非参数学习 Nonparametric Learning

参数学习

这就是一个二维的特征空间, 假设横坐标是房屋的面积, 纵坐标是房屋的价格

对应每一个房屋相应就在特征空间中代表一个点, 基于这些数据点我可以假设房屋的价格和房屋的面积是一个线性的关系 f(x) = ax + b

当有了这个假设之后, 机器学习的过程就是想办法找到一个好的a和b来描述房屋的面积和房屋的价格之间的关系. a 和 b 就是参数

基于这样的学习特点的学习算法就叫做参数学习

参数学习最大的特点就是: 喂给机器很多数据, 这些数据的作用是希望学到我们想要的参数. 一旦我们学到了参数, 那么这些数据集就不再有用了

例如上面的 f(x) = ax+b, 一旦我们知道 a 和 b, 那么对于一个新的房屋, 只要将它的面积代入公式, 就可以预测出房屋的价格. 此时预测的结果和我们喂给机器的训练数据都是没有关系的

事实上可以做出非常多的假设, 统计学领域有非常多的统计模型. 我们可以假设特征和要预测的结果之间是某种统计模型可以涵盖的关系, 之后我们的任务就是学习统计模型中的参数. 一旦学到了这些参数, 后面的预测是非常简单的, 不再需要原有数据集的参与


非参数学习

非参数学习就是不对模型进行过多假设

非参数不等于没有参数

非参数学习中的参数其实并不少, 区别只是在于我们不对整个问题进行一个建模. 不把整个问题理解成是学习一些参数, 但是在学习的过程中依然是要有参数来参与的



和机器学习相关的”哲学”思考

机器学习这类算法主要处理的是不确定世界中相关的问题

这和传统算法解决的问题有很大不同, 经典算法所解决的问题通常都是有固定, 唯一, 标准的答案的

但机器学习不同, 它面对的是高度不确定的现实世界中的问题, 面对这些问题, 机器学习给我们的答案其实也是不确定的, 概率性的, 有统计意义的答案

面对这样的答案, 我们难免会思考某些问题: 这些答案真的可靠吗, 我们到底能多大程度地相信这些答案, 机器学习学习到的本质是什么….

从算法的角度它们是有优劣的, 但微软的这篇文章揭示出来, 如果我们喂给这些算法足够多的数据 (横坐标是数据规模, 纵坐标是预测的准确率) 所有的算法的准确率都在稳定上升

数据量越大, 准确率越高. 当数据量大到一定程度时, 其实算法的准确率之间的差别并不大

机器学习领域, 算法本身似乎不是那么重要, 只要我们喂给算法的数据足够多, 质量足够好, 甚至有过”数据计算法”这样一个说法

也正是因为如此, 大数据时代才拉开帷幕


数据本事确实非常重要, 而且现阶段机器学习解决的大多数问题都是数据驱动的(高度依赖数据本身的质量).

这也使得很多机器学习的研究者花费大量时间处理数据

  • 收集更多的数据

  • 提高数据质量

  • 提高数据的代表性

  • 研究更重要的特征(特征工程)

机器学习领域内有非常多的算法, 但是其实它们解决的问题大抵上是差不多的, 以监督学习为例, 它解决的问题就是两个问题: 分类问题和回归问题

在这种情况下, 针对要研究的问题到底要选择怎样的算法?

和选择相关的问题, 最简单也是最深刻的原则: 奥卡姆的剃刀

奥卡姆的剃刀

简单的就是好的, 对于一个问题不要有过多的假设, 不要对它进行过多地复杂化

然而一个容易引发争议的问题就是: 到底在机器学习领域, 什么叫”简单”. 不同的情况下有不同的答案


没有免费的午餐定理

另一个有趣的定理叫做没有免费的午餐定理

即: 可以用严格地数学推导出: 任意两个算法, 他们的期望性能是相同的

这个定理揭示出一个非常重要的问题, 如果任意两个算法, 他们的期望性能是相同的, 那么说明其实没有任何一个算法比另外一个算法好, 相当于说所有的算法是等价的

而如果是这样的话又为什么要学习这么多的算法呢? 期望可以简单地理解成是平均值

换句话说对于任意两个算法, 把它们作用在所有问题中, 对于某些问题a算法比b算法好, 对于另一些问题b算法比a算法好, 但是平均来讲a, b两个算法是一样的

具体到某个特定的问题, 有些算法可能更好. 但没有一种算法, 绝对比另外一种算法好

即: 脱离具体问题, 谈哪个算法好是没有意义的

同时它也告诉我们在机器学习领域时间的过程中必须要做的一件事: 在面对一个具体问题的时候, 尝试使用多种算法进行对比试验, 是必要的

最基础的分类算法-k近邻算法 (kNN)

k近邻算法(k-Nearest Neighbors) 非常适合入门

kNN算法特点:

  1. 思想极度简单
  2. 应用数学知识少 (几乎为零)
  3. 效果好 (缺点以后说)
  4. 可以解释机器学习算法使用过程中的很多细节问题
  5. 更完整的刻画机器学习应用的流程

之前说过, 已知的数据点其实是分布在一个特征空间中的 (可视化方便通常使用二维屏幕)

可以看出对于每一个病源, 它的肿瘤大小和发现的时间就构成了在这个特征平面中的一个点

对于这个点, 我们又已经知道了它是一个恶性的肿瘤(蓝色)还是良性的肿瘤(红色), 这样就获得了8个数据点的初始信息

如果这时新来了一个病人(绿色的点)

我们怎样判断它可能是良性肿瘤还是恶性肿瘤?


k近邻算法是这样一个算法:

首先要取一个k值, 这里让k=3, 具体k值要怎么取在后续有详细的介绍 (暂时可以理解机器学习的使用者根据经验取得了一个经验上的最好值)

如果k=3的话, 对于每一个新的数据点, k近邻算法做的就是在所有的点中寻找与这个点最近的3个点

这三个点以自己的结果进行投票

现在对于这个点最近的3个点都是代表恶性肿瘤的蓝色的点, 所以投票结果蓝色对红色是(3:0)

因此k近邻算法就说这个新的点有很大的概率也是一个蓝色的点


核心思想

k近邻的算法本质是认为: 如果两个样本足够的相似, 那么它就有更高的概率属于同一个类别

当然很有可能只看离他最近的一个样本是不准确的, 所以多看几个样本, 看和它最相似k个样本中那个类别最多, 我们就认为这个新的点属于那个类别

这里我们描述两个样本是否相似(相似性)就是靠两个样本在特征空间中的距离来进行描述的

最为常用的就是欧拉距离

由于连加表示不方便, 所以有时也用Σ符号表示
$$
\sqrt{\sum^n_{i=1}(X^{(a)}_i - X^{(b)}_i)^2}
$$

def kNN_classify(k, X_train, y_train, x):

    assert 1 <= k <= X_train.shape[0], "k must be valid"
    assert X_train.shape[0] == y_train.shape[0], \
        "the size of X_train must equal to the size of y_train"
    assert X_train.shape[1] == x.shape[0], \
        "the feature number of x must be equal to X_train"

    # 计算所有点到预测点的距离, 即对向量取模
    distance = [np.linalg.norm(x_train - x) for x_train in X_train]
    # 排序, 返回索引
    nearest = np.argsort(distance)

    # 从索引中取前k个元素进行投票
    topK_y = [y_train[i] for i in nearest[:k]]
    votes = Counter(topK_y)

    return votes.most_common(1)[0][0]

然而对于kNN算法来说, 我们并没有得到什么模型

class KNNClassifier:

    def __init__(self, k):
        """初始化kNN分类器"""
        assert k >= 1, "k must be valid"
        self.k = k
        self._X_train = None
        self._y_train = None

    def fit(self, X_train, y_train):
        """根据训练数据集X_train和y_train训练kNN分类器"""
        assert self.k <= X_train.shape[0], \
            "the size of X_train must be at least k."
        assert X_train.shape[0] == y_train.shape[0], \
            "the size of X_train must equal to the size of y_train"

        self._X_train = X_train
        self._y_train = y_train
        return self

    def predict(self, X_predict):
        """给定待预测数据集X_predict, 返回表示X_predict的结果向量"""
        assert self._X_train is not None and self._y_train is not None, \
            "must fit before predict!"
        assert X_predict.shape[1] == self._X_train.shape[1], \
            "the feature number of X_predict must be equal to X_train"
        y_predict = [self._predict(x) for x in X_predict]
        return np.array(y_predict)

    def _predict(self, x):
        """给定单个待预测数据x, 返回x的预测结果值"""
        assert self._X_train.shape[1] == x.shape[0], \
            "the feature number of x must be equal to X_train"

        distance = [np.linalg.norm(x_train - x) for x_train in self._X_train]
        nearest = np.argsort(distance)

        topK_y = [self._y_train[i] for i in nearest[:self.k]]
        votes = Counter(topK_y)

        return votes.most_common(1)[0][0]

    def __repr__(self):
        return "KNN(k=%d)" % self.k

判断机器学习算法的性能

首先我们把所有的原始数据都当作训练数据

然后用训练数据训练出模型, 具体再kNN算法中, 就表现在每当来了一个新的数据, 新的数据要和训练中的所有的数据都来求出相应的距离, 得出前k小的距离, 投票看新的数据属于那种类别

换句话说: 用全部的训练数据得到的模型用来预测新的数据的所属类型.

然而, 得到这个模型的意义在于要在真实的环境中使用这个模型

可是我们现在这么做是有很大的问题的

  1. 我们拿所有的训练数据得到的模型, 我们只能把这个模型放到真实的环境中使用了, 可是如果我们得到的模型很差怎么办? 我们根本没有机会取真实地调整它就去使用它了
  2. 真实环境难以拿到真实的label, 这种情况下我们根本无从知道模型是好是坏, 只能听天由命的让它运行在真实环境中

这些问题都在告诉我们: 用所有的原始数据来当训练集去直接训练模型, 投入到生产环境中这样的做法是不恰当的


改进这个问题最简单的一个方法就是训练和测试数据集的分离

从原始数据中抽取一大部分(80%)数据作为训练数据, 剩余的数据(20%)作为测试数据

这样我们只用蓝色的训练数据训练模型, 拿到模型之后把测试数据扔进模型来让模型预测

由于测试数据有真实的label值, 就可以很容易地看到训练数据训练出的模型的性能, 并且可以改进算法得到更好的模型

这种方式就叫做train test split

然而这种方式其实在判断机器学习算法的过程中也存在它的问题, 这个问题也是以后再说.

根据上面的思想, 写出一个函数由于拆分训练数据和测试数据

def train_test_split(X, y, test_ratio=0.2, seed=None):
    """将数据 X 和 y 按照 test_ratio分割成X_train, X_test, y_train, y_test"""
    assert X.shape[0] == y.shape[0], \
        "the size of X must be equal to the size of y"
    assert 0.0 <= test_ratio <= 1.0, \
        "test_ratio must be valid"

    # 方便debug
    if seed:
        np.random.seed(seed)

    # 获得索引的随机排列以及测试数据集的大小
    shuffle_indexes = np.random.permutation(len(X))
    test_size = int(test_ratio * len(X))

    # 获取训练数据集和测试数据集的indexes
    test_indexes = shuffle_indexes[:test_size]
    train_indexes = shuffle_indexes[test_size:]

    # 获取训练数据集和测试数据集
    X_train = X[train_indexes]
    y_train = y[train_indexes]
    X_test = X[test_indexes]
    y_test = y[test_indexes]

    return X_train, X_test, y_train, y_test

我们之前调用kNN算法全都是直接为k值传一个默认的值, 那么在具体实际使用时应该传什么值合适? 这就涉及到机器学习算法中一类非常重要的问题: 超参数


超参数

什么叫超参数? 简单的说就是在运行机器学习算法之前需要指定的参数

kNN算法中的k就是最为典型的一个超参数

和超参数对应的一个概念叫做模型参数

  • 超参数: 在算法运行前需要决定的参数
  • 模型参数: 算法过程中学习的参数

kNN没有模型参数, k是典型的超参数

机器学习算法工程师的一个重要工作就是调参, 调的就是超参数


如何寻找好的超参数

由于机器学习算法应用在不同的领域中领域知识通常是有意义的, 不同领域中面对不同问题最好的超参数是不一样的

某些问题有比较好的经验数值, 有些算法框架中的一些默认数值就是经验数值, 例如scikit-learn中默认的k的值为5

然而即使如此也有可能具体的问题的超参数是和经验数值不一样的, 那就只能使用实验搜索了, 即尝试几组不同的超参数, 取效果最好的参数值

调参的例子:

from sklearn import datasets
from sklearn.model_selection import train_test_split
from sklearn.neighbors import KNeighborsClassifier

如果我们找到最好k的值是10的话, 此时我们有必要对10以上的数在进行一下搜索

这是因为通常来讲, 不同的参数决定了不同的分类的准确率, 它们之间是呈现一个连续的变化的

如果我们找到最好的参数值在边界上时, 就意味着有可能有更好的值在边界外面

kNN算法并不只有k这一个超参数, 其实这个算法中还潜藏着一个非常重要的超参数, 与此同时这个超参数也对应着kNN算法的一个不太一样的用法


kNN算法的另一个超参数

现在的kNN算法, 当k=3时, 我们就找到离我们当前要判断的节点(绿色节点)最近的三个节点

由于离绿色节点中最近的有2个蓝色节点, 一个红色节点所以蓝色的节点获胜

这里虽然我们考虑了离绿色节点最近的三个节点, 但我们却忽略了最近的三个节点相应的距离是多少

这个绿色的节点其实离红色的节点是最近的, 虽然蓝色节点是离绿色节点次近的两个节点, 可是还是离绿色的节点比较远

所以此时进行投票的话, 红色节点的票是否应该比 两个蓝色节点的票权重要重一些?

这就是k近邻算法的另外一个用法, 就是考虑权重

通常而言, 考虑权重时是将倒数作为权重, 这样离得越近权重就越大

因此当我们考虑距离的时候, 红色获胜


而且当我们考虑距离之后还有另外一个优点

我们之前一直没有谈一个问题就是使用普通的k近邻算法的话, 比如k=3的话, 分类的样本数最终也有3类的话, 就很有可能产生平票的情况

比如说上图, 最近的3个节点分别是红蓝紫, 如果是之前的算法就只能随机选出一个结果—这其实是不合理的

而当我考虑距离之后就可以非常好地解决平票的问题, 此时实际比较的是离它最近的节点, 依然是红色节点获胜

关于距离的更多定义

考虑到了距离就不得不提另外一个话题: 什么是距离

至今位置我们在kNN算法中说道的距离都是指欧拉距离
$$
\sqrt{\sum^n_{i=1}(X^{(a)}_i - X^{(b)}_i)^2}
$$
说道距离还有一个非常著名的曼哈顿距离

曼哈顿距离就是指两个点在每个维度上的距离和

在上图的二维平面中, 两个点的曼哈顿距离就是它们在x方向上的差值加上在y方向上的差值

上图中红蓝黄三条线都是曼哈顿距离, 而绿色的线是曼哈顿距离


同样是距离, 我们可以使用曼哈顿距离, 还可以使用欧拉距离

我们对欧拉距离的这个式子再进行一下推导

小括号再平方就等同于是绝对值再平方, 我们可以把开根号理解成是1/2次幂

欧拉距离是每一项为2次幂, 式子总体取1/2次幂

而曼哈顿距离可以理解为式子总体是1次幂, 并且每一项的和是1次幂

这样看来曼哈顿距离和欧拉距离在数学形式上是具有一定的一致性的

我们把这个一致性进行一下推广

每一项求p次方, 最终求整体和的1/p次方, 这个就是明可夫斯基距离(Minkowski Distance)
$$
{(\sum^n_{i=1}|X^{(a)}_i - X^{(b)}_i|^p)}^\frac 1 p
$$

  • 当p=1时, 明可夫斯基距离就相当于是曼哈顿距离

  • 当p=2时, 明可夫斯基距离就相当于是欧拉距离

  • … …

在这种情况下, 我们又获得了一个新的超参数, p

当p>=3时, 明可夫斯基距离具体表达的数学上的意义就不进行深究了

我们只是得到了一个超参数p, 进而在算法中对于这个超参数相应的进行搜索, 来看对于我们这个问题p取谁更好

scikit-learn中KNeighborsClassifier构造函数就有参数p, 默认为2, 取得就是欧拉距离

    print("best_p = ", best_p)         # best_p =  2

这种搜索策略有一个名字网格搜索, 对于k, p两个参数, 其实就形成了 k * p 这样的网格, 对其中每一个点所代表的数据进行了一下搜索来找其中的最好值

不过在这里在具体搜索过程中有的时候会有一些麻烦, 比如weight这个参数, 当我们使用uniform时就不牵扯p这个参数, 但当使用distance时又会牵扯p这个参数

换句话说, 超参数之间可能存在相互依赖的关系

那么如何能一次性地把超参数全都列出来, 运行一遍程序就能得到最好的超参数的组合?

scikit-learn专门为网格搜索封装了一种专门的方式, 学习这种方式可以更加方便地实现寻找最好的超参数的过程

数据归一化 Feature Scaling

之前使用kNN算法来完成分类任务时, 其实少了非常重要的一步

那就是数据归一化(Feature Scaling)


为什么要进行数据归一化?

对于样本1来说肿瘤大小为1cm, 发现200天了. 而对于样本2来说肿瘤大小是5cm, 发现100天

那么这两个样本之间的距离是怎样的呢? 如果使用欧拉距离就是
$$
\sqrt {(1-5)^2-(200-100)^2}
$$
显然, 这种情况下距离的大小被发现的时间所主导了, 因为间隔了100天, 而肿瘤的大小只相差4cm

虽然在样本数据中5和1相差5倍之多, 而200和100只相差了2倍

但是由于我们的量纲不同, 导致了最终的距离主要衡量的是发现时间的天数之间的差值, 这是因为肿瘤大小之间的差值太小了

而如果我们把发现时间的单位换算为年, 此时这两个样本之间的距离由被肿瘤大小的距离所主导, 0.55-0.27这个数组变得特别地小

而在这个量纲下5-1由变得非常大

如果我们不进行数据的基本处理的话, 这样直接计算两个样本之间的距离很有可能是有偏差的, 不能非常好的同时反映出样本中每一个特征的重要程度, 正是因为如此, 我们要对数据进行归一化处理


解决方案: 将所有数据都映射到同一尺度


最值归一化(normalization / normalize)

通常最简单的方式就是最值归一化

最值归一化: 把所有数据映射到 0-1 之间
$$
x_{scale} = \frac {x-x_{min}} {x_{max}-x_{min}}
$$
对于每一个特征, 求出这个特征对应的最大值和最小值的差, 然后对于每一个特征点到最小值的距离比上这个差就得到一个0-1之间的数

这相当于是先把数据映射到 [0, xmax-xmin] 这个范围中, 然后来找对于x而言相比于整个范围所占的比例是多少

这种方法是一种简单的做法, 它适用于分布有明显边界的情况; 受outlier影响较大

有明显边界: 例如考试分数的边界为[0, 100], 颜色的边界为[0, 255]

无明显边界: 收入, 有的人的收入是极其高的(outlier), 如果一个人的月收入是100万元, 而其余所有人的月收入是1万元, 那么映射到0-1之间大部分数都聚集在0.01附近, 这个数据的映射结果是不够好的

相应的一个改进方式就是使用均值方差归一化


均值方差归一化 (standardization / standardize)

均值方差归一化: 把所有数据归一到均值为0方差为1的分布中(统计学中求z值)

这样做的结果是: 数据并不保证在0-1之间, 但是所有数据的均值是在0的位置, 并且整体数据的方差为1

这种归一化的方式适用于数据分布没有明显的边界; 有可能存在极端数据值, 其实即使数据有明显边界, 使用这种方式也是非常好的

所以除非像考试分数这样有非常明确的边界, 一般情况都使用均值方差归一化
$$
x_{scale}= \frac {x-x_{mean}} s
$$
s为标准差, 就算是方差的开根

统计学中的z值
$$
z = \frac {x-E(x)} {\sqrt {V(x)}}
$$

如何对测试数据集归一化?

对于原始数据集我们将其分为训练数据集和测试数据集, 如果我们要用归一化后的数据来训练模型的话显然首先我们要对训练数据集进行归一化处理

比如说进行均值方差这样的归一化, 就要求出训练数据集的均值(mean_train)和标准差(std_train)

当我们这样进行归一化之后, 将这样的训练数据集用于训练模型, 最终我们使用训练的模型来预测数据的话, 对于测试相应的也要进行归一化处理

那么对于测试数据集如何进行归一化处理? 难道也是求出测试数据集的均值(mean_test)和标准差(std_test) 来对它进行标准化?

但这么做其实是不对的

正确的做法是将测试数据集使用训练数据集的均值(mean_train)和标准差(std_train)来进行归一化

这是因为测试数据是模拟真实环境

  • 真实环境很有可能无法得到所有测试数据的均值和方差
  • 对数据的归一化也是算法的一部分

虽然可以得到测试数据集中的均值和方差, 但在实际使用中每次只来了一个样本, 那么这一个样本的均值和方差是什么?

因此为了让它归一化, 我们只能让它使用训练数据集的均值和方差进行归一化

因此是要保留训练数据集得到的均值和方差


scikit-learn中的数据归一化

为了方便地进行操作, 在scikit-learn中专门封装了一个类Scaler

封装就是想办法让Scaler这个类和机器学习算法这个类的整体的使用流程是一致的

Scaler的fit函数就是求出训练数据集的一些统计指标, 比如说对于均值方差归一化来说, 就是求出均值和标准差

之后Scaler中保存了关键信息, 再来其他的样例之后, Scalar就可以对输入样例进行一步transform来得到归一化后的输出结果

模仿scikit-learn中的Scaler编写自己的Scaler

import numpy as np

class StandardScaler:

    def __init__(self):
        self.mean_ = None
        self.scale_ = None

    def fit(self, X):
        """根据训练数据集X获得数据的均值和标准差"""
        assert X.ndim == 2, "The dimension of X must be 2"

        self.mean_ = np.array([np.mean(X[:, i]) for i in range(X.shape[1])])
        self.scale_ = np.array([np.std(X[:, i]) for i in range(X.shape[1])])

        return self

    def transform(self, X):
        """将X根据这个StandardScaler进行均值方差归一化处理"""
        assert X.ndim == 2, "The dimension of X must be 2"
        assert self.mean_ is not None and self.scale_ is not None, \
            "must fit before transform!"
        assert X.shape[1] == len(self.mean_), \
            "the feature number of X must be equal to mean_ and std_"

        resX = np.empty(shape=X.shape, dtype=float)
        for col in range(X.shape[1]):
            resX[:, col] = (X[:, col] - self.mean_[col]) / self.scale_[col]
        return resX

总结

k近邻算法是解决分类问题的算法, 不仅如此, 它天然地可以解决多分类问题, 它的思想简单, 效果强大

不仅如此, 使用k近邻算法还可以解决回归问题

回归问题就是我们具体要预测的是一个数值, 对于要预测的节点, 找到离它最近的k个节点, 由于是监督学习所以都有真实的值

上图的绿色节点的值大概可以预测为与它最近的3个节点的均值

当然也可以考虑上离它最近的3个节点对应的距离, 它们可以代表权值, 100离得更近, 所以权值更高, 使用加权平均的算法预测.

不管怎么说, k近邻算法是可以解决回归问题的

scikit-learn中封装了KNeighborsRegressor这个类, 它就是使用k近邻算法解决回归问题的类


缺点1: k近邻算法的缺点是效率低下

如果训练集有m个样本, n个特征, 则预测每一个新的数据需要O(m*n)

k近邻算法也有一些优化方式, 可以使用树结构: KD-Tree, Ball-Tree

但即使如此k近邻算法依然是效率非常低的算法


缺点2 : 高度数据相关

当然, 机器学习算法就是喂给它数据来进行预测, 所以理论上来说, 所有机器学习算法都是高度数据相关的

不过k近邻算法相对而言, 对outlier更加敏感, 假设我们用3近邻算法, 但在样本中有2个错误的值的话就足矣让预测出现错误了, 哪怕在样本空间中是有大量正确样本的


缺点3 : 预测结果不具有可解释性

我们只是找到了和要预测样本距离比较近的样本我们就说要预测的样本属于这个类别, 但是我们要预测的样本为什么属于这个类别我们根本无从知晓

很多研究上, 只是拿到结果是远远不够的, 我们希望对预测结果有一定的解释性, 进而能通过这些解释去推广或去制作新的工具


缺点4 : 维数灾难

维数灾难的概念: 随着维度的增加, “看似相近”的两个点之间的距离越来越大

而k近邻算法非常依赖两个点之间的距离的计算, 这使得一旦我们使用k近邻算法处理高维数据的时候就很有可能遭受维数灾难

10000维看似恨到, 实际就只能表达一个100*100像素的灰色图片

实际生产环境中, 很多领域处理成千上万, 乃至几十万维的数据都是非常正常的

解决方案就是降维

什么是线性回归算法

现在在坐标平面上每一个点都表示一个数据, 线性回归算法说: 我们认为房屋的面积(X) 和房屋的价格(Y) 成线性关系

即随着房屋面积的增大, 价格也跟着增大

再这样的假设下, 我们可不可以找到一条直线, 最大程度”拟合”样本特征和样本输出之间的关系

注意: 同样是画出一个二维平面图, 可是现在的二维平面图和kNN分类问题的二维平面图有很大的区别

对于分类问题来说, 横轴和纵轴都是样本的特征, 就是说每一个点有2个样本特征: 肿瘤的大小, 和肿瘤的发现时间

而这个样本的输出标记是什么? 是被这个点是蓝色的点还是红色的点所表示的

但是在线性回归算法例子中, 之后横轴是样本特征, 纵轴就已经是样本的输出标记了

这是因为在回归问题中, 真正要预测的是一个具体的数值, 这个具体的数值实在一个连续的空间里的, 而不是简单的可以用不同颜色代表不同类别来区分的, 所以它需要占用一个坐标轴的位置

如果想看有两个样本特征的回归问题的话, 就需要在3维空间中进行观测了

对于样本特征只有一个, 称为简单线性回归

顾名思义, 简单线性回归是更加简单的, 通过对简单线性回归的学习, 其实可以学习到线性回归算法的很多内容, 之后再将它推广到样本特征有多个的情况下(多元线性回归)


推导

要找到一个回归线在最大程度拟合这些点, 回归线的形式就是y=ax+b, a是斜率, b是截距

相应对于第i个点来说就对应一个样本特征x(i), 对应的输出标记就是y(i)

如果我们找到y=ax+b这条直线的话, 换句话说找到 a 和 b 的话, 就可以把 x(i) 代入式子中

得到的值就是使用简单线性回归法预测出来的对于x(i)这个值对应y的预测, 通常用 y_hat(i) 表示, 相应的, 预测值和真值之间就会有一个差距.

假设我们找到了最佳拟合的直线方程, 那么我们就希望 y(i) 和 y_hat(i) 的差距尽可能小

表达 y(i) 和 y_hat(i) 的差距:
$$
y^{(i)} - \hat y^{(i)}
$$
但是有可能值为负数, 这是不合理的, 因此还有一种办法就是使用绝对值
$$
|y^{(i)} - \hat y^{(i)}|
$$
使用绝对值表示是可以的, 但是在后面计算a, b的时候使用绝对值是非常不方便的, 因为这个函数不是处处可导的函数, 但是为了计算 a 和 b 的值需要找到方程的极值, 需要整个函数可导. 不过, 在衡量线性回归法的性能时还会用到绝对值这个思想

如果不使用绝对值, 那么一个非常简单的思路就是求平方
$$
(y^{(i)} - \hat y^{(i)})^2
$$
既抵消了正负号, 这个函数又是一个处处可导的函数. 用它来衡量y(i) 和 y_hat(i) 的差距, 相应的考虑到所有的样本的话, 就是这样的一个式子
$$
\sum^m_{i=1} (y^{(i)} - \hat y^{(i)})^2
$$
现在的目标就是使得上面这个式子尽可能小, 而我们知道
$$
\hat y^{(i)} = ax^{(i)}+b
$$
代入上面的式子中就是
$$
\sum^m_{i=1} (y^{(i)} - ax^{(i)}-b)^2
$$
现在目标就是找到 a 和 b 的值, 使得这个式子尽可能小

注意: 对于这个式子来说只有a和b两个值是未知数, 我们已知所有的 y(i)和x(i), 因为是监督学习, 我们要首先提供一组数据, 这组数据中的每一个样本都有它对应的x和y


抽象出参数学习的一个核心思想

先不急着求具体的数值, 先看一下目前为止推导出来的目标

找到某一些参数, 使得某一个函数尽可能小, 这是典型的一种机器学习算法的推导思路

换句话说我们所谓的建模的过程, 就是找到一个模型, 最大程度拟合数据, 在线性回归算法中, 这个模型就是一个直线方程

所谓的最大拟合数据, 其实本质就是找到这样一个函数, 我们称这个函数为损失函数(loss function), 也就是说度量出模型没有拟合住样本的这一部分

有的算法中度量拟合的部分, 这种情况下, 称这个函数为效用函数(utility function)

不管是损失函数还是效用函数, 机器学习算法都是通过分析问题, 确定问题的损失函数或者效用函数(有时候统称为目标函数), 通过最优化损失函数或着效用函数, 获得机器学习的模型

这里所说的不是具体某个算法, 而是一种求解机器学习算法的思路

近乎可以说所有的参数学习算法都是这样一个套路: 线性回归, 多项式回归, 逻辑回归, SVM, 神经网络… …

正是因为机器学习中大部分的算法都拥有这样一个思路, 有一个学科就显得非常重要: 最优化原理

实际上最优化原理绝对不仅仅是机器学习算法中所使用的思路, 在传统算法领域中, 最优化原理也发挥着重要的作用

因为计算机解决的大部分问题其实本质都是最优化问题: 最短路径, 最小生成树……

在最优化原理领域还有一个分支是凸优化, 它解决的是特殊的优化问题


a和b的数学推导

回到我们的目标

这个问题是一个典型的最小二乘法问题: 最小化误差的平方

通过最小二乘法我们可以求出具体的a和b的表达式


$$
\sum^m_{i=1} (y^{(i)} - ax^{(i)}-b)^2=J(a,b)
$$
把这个式子叫做 J(a, b), 一定要注意, 对于这个式子a和b才是未知数, 而不是x, y. x和y是监督学习提供给我们的已知的信息

数学上很多时候用 J 表示损失函数, 有一些教材使用 cost 的方式所谓损失函数

现在要求损失函数的最小值, 就是求这个函数的极值

求函数极值的一个最基本的方法就是对这个函数的各个未知分量求导(求偏导) 让它的导数等于0, 在导数等于 0 的地方, 就是函数极值的地方


推导b

$$
\sum^m_{i=1} (y^{(i)} - ax^{(i)}-b)^2
$$

对b求偏导, 把其余未知量都当作常量, 对b求导

我们可以将它拆开
$$
\sum^m_{i=1}y^{(i)}-a\sum^m_{i=1}x^{(i)}-\sum^m_{i=1}b=0
$$
由于b是一个常数, 所以最后一项相当于m个b相乘, 因此式子被化成这个样子
$$
\sum^m_{i=1}y^{(i)}-a\sum^m_{i=1}x^{(i)}-m*b=0
$$

$$
mb=\sum^m_{i=1}y^{(i)}-a\sum^m_{i=1}x^{(i)}
$$

此时等式两边同时除以m, 所有 y(i) 的和除以m就是y的均值, 同理所有 x(i) 的和除以m就是x的均值
$$
b=\bar y-a\bar x
$$

推导a

$$
\sum^m_{i=1} (y^{(i)} - ax^{(i)}-b)^2
$$

对a求偏导, 把其余未知量都当作常量, 对a求导

由于我们已经求出 b 的表达式, 我们就可以把 b 带入式子中
$$
\sum^m_{i=1} (y^{(i)} - ax^{(i)}-\bar y+a\bar x)x^{(i)}=0
$$
现在对于这个式子来说只有a这一个未知数, 所以整理出来以后就能得到 a 的表达式

首先把外面的 x(i) 乘进括号里
$$
\sum^m_{i=1} (x^{(i)}y^{(i)} - a(x^{(i)})^2-x^{(i)}\bar y+a\bar xx^{(i)})
$$

$$
\sum^m_{i=1} (x^{(i)}y^{(i)} - x^{(i)}\bar y-a(x^{(i)})^2+a\bar xx^{(i)})
$$

$$
\sum^m_{i=1} (x^{(i)}y^{(i)} - x^{(i)}\bar y)-\sum^m_{i=1} (a(x^{(i)})^2-a\bar xx^{(i)})
$$

此时未知数a都在后面, 作为所有项的一个归并项
$$
\sum^m_{i=1} (x^{(i)}y^{(i)} - x^{(i)}\bar y)-a\sum^m_{i=1} ((x^{(i)})^2-\bar xx^{(i)})=0
$$

$$
a=\frac{\sum^m_{i=1} (x^{(i)}y^{(i)} - x^{(i)}\bar y)} {\sum^m_{i=1} ((x^{(i)})^2-\bar xx^{(i)})}
$$

其实这个a已经是最后的结果了, 如果编程的话, 使用按照这个式子去计算 a 是没有问题的

不过我们在对这个式子进行一下整理, 会得到一个更为简洁的表达式, 而对于这个更加简洁的表达式, 我们可以在编程时使用一些技巧加速它的过程

看看对于a这个式子还能怎么变型
$$
a=\frac{\sum^m_{i=1} (x^{(i)}y^{(i)} - x^{(i)}\bar y)} {\sum^m_{i=1} ((x^{(i)})^2-\bar xx^{(i)})}
$$
由于
$$
\sum^m_{i=1} x^{(i)}=m\bar x
$$

相当于是将x和y的位置对调了, 同时还能推出这个, 因为x_bar, y_bar都是常数

这两项是一样的

有了这个结论, 就可以对a这个式子进行下变换
$$
a=\frac{\sum^m_{i=1} (x^{(i)}y^{(i)} - x^{(i)}\bar y)} {\sum^m_{i=1} ((x^{(i)})^2-\bar xx^{(i)})}
$$

因为
$$
\sum^m_{i=1}\bar xy^{(i)} =\sum^m_{i=1} \bar x\bar y
$$

$$
\sum^m_{i=1}\bar xx^{(i)} =\sum^m_{i=1} \bar x\bar x
$$

用因式分解整理一下

此时a就变成了更为干净的一个式子,虽然和原来的复杂度差不多. 在具体实现是其实我们会看到, 对于这个式子在实现的时候会简单很多

import numpy as np

# 第一个版本
class SimpleLinearRegression1:

    def __init__(self):
        """初始化 Simple Linear Regression 模型"""
        self.a_ = None
        self.b_ = None

    def fit(self, x_train, y_train):
        """ 根据训练数据集x_train, y_train训练Simple Linear Regression 模型 """
        assert x_train.ndim == 1, \
            "Simple Linear Regressor can only solve single feature training data"
        assert len(x_train) == len(y_train), \
            "the size of x_train must be equal to the size of y_train"
        x_mean = np.mean(x_train)
        y_mean = np.mean(y_train)

        num = 0.0
        d = 0.0
        for x_i, y_i in zip(x_train, y_train):
            num += (x_i - x_mean) * (y_i - y_mean)
            d += (x_i - x_mean) ** 2
        self.a_ = num / d
        self.b_ = y_mean - self.a_ * x_mean

        return self

    def predict(self, x_predict):
        """给定待预测数据集x_predict, 返回表示x_predict的结果向量"""
        assert x_predict.ndim == 1, \
            "Simple Linear Regressor can only solve single feature training data."
        assert self.a_ is not None and self.b_ is not None, \
            "must fit before predict!"

        return np.array([self._predict(x) for x in x_predict])

    def _predict(self, x_single):
        """给定单个待预测数据x_single, 返回x_signle的预测结果值"""
        return self.a_ * x_single + self.b_

    def __repr__(self):
        return "SimpleLinearRegression1()"

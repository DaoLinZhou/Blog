优点 : 开发速度快

缺点 : 不适合大型爬虫项目, 没有设置请求头, 没有动态ip 等等

需要的包: requests, lxml

在真正编程以前, 要先安装一下依赖的包

pip install requests
pip install lxml

具体用法


一. 引入包

import requests
from lxml import html

二. 获取相应数据

通过requests来get或post到数据, 可能是json格式, 也可能是html, 下文默认以html来解析

url = 'https://www.bilibili.com/'
html_context = requests.get(url).text

除了.text, 还有.status_code(返回状态码), .encoding(编码格式) 等等内容, 可以根据需要获取


三. 分析

lxml下的html支持通过xpath的方式对html文本进行解析

# 进行加载
selector = html.fromstring(html_context)    
# 通过xpath的方式进行解析
ul = selector.xpath('//div[@id="search_nature_rg"]/ul')

如果懒得写xpath, 也可以在对应元素右击, 找到复制xpath, 但这个结果只供参考


简单说一下xpath的语法

开头// 代表在任意位置匹配, / 代表在调用对象的路径下匹配(最开始是以html为路径), 匹配的就是ul下的li, 例如:

li_list = ul.xpath('/li')

如果我们要确定匹配对象的属性时可以通过[@属性名=”属性值”]来约束, 就如上面ul的xpath一样


如果我们要获取文本可以通过text(), 例如:

store = li.xpath('p[@class="search_shangjia"]/a/text()')

如果我们要获取属性值可以用 @属性名, 例如:

link = li.xpath('a/@href')
title = li.xpath('a/@title')

其他函数:

last() 获取body下最后一个p标签的内容

p = selector.xpath('/html/body/p[last()]/text()')

在其他的用法就直接查xpath的语法吧.

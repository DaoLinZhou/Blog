事先声明:

以下都是个人的经验以及从各处学来的知识的总结, 并不是万能钥匙. 一道题可以有多种解法, 甚至多种最优解, 我接下来要写的只是为了帮助自己在看到一道算法题时不至于一点思路没有. 但不一定会是最优的解题思路.

解和特殊情况

对于所有的算法问题, 我们都一定要考虑到它的特殊情况

  1. 给定的string是否可能为空
  2. 给定的int值是否可能非法
  3. 给定的vector的size是否可能为0
  4. 这道题是否可能没有解
  5. 这道题是否可能有多个解
  6. 如果有多个解, 这写解的排列顺序应该是什么
  7. 如果要删除一个元素, 如果这个元素不存在怎么办
  8. 如果要删除一个元素, 如果存在多个相同的元素怎么办
  9. 删除的元素是否需要返回

思路很重要

当面对一个问题, 写了大量代码, 但结果却不正确时, 大概就是思路错了

此时最好重新审题, 寻找自己忽略的信息


牢记自己对变量的定义

算法最容易出错的就是边界.

如何保证自己的算法没有边界问题? 那就是牢记自己对变量的定义, 时刻保证程序满足这些定义.

举个栗子:

在三路快排的那部分, 我画了一张草图, 解释我对各个变量的定义. 而在接下来的算法中我则是时刻保持它们不越界, while循环内判断的是 q <= k,

那是因为在我对q, k 定义时, 让它们可以重合, 如果q == k 那么q (或是k) 这个索引就是最后一个元素待分组的元素的位置. 当我们把这个元素分好组后, 就完成了一次partition操作.

如果我改变自己的定义, 就要对代码进行相应的更改, 使得在程序运行的任何时候, 都满足我们的定义

对撞指针

分别在数组头部和尾部创建指针, 判定某一条件, 根据情况移动两个指针, 直到两个指针碰撞到一起.

通常对撞指针适合用于有序数组, 因为有序性使得指针移动更有意义, 例如: 如果数组是从小到大排列, 如果l(头部指针) + r(尾部指针) 的和小于期望值, 便可以让l右移来增加总和; 反之则让r左移来减少总和.

有些题目要求求出所有组合, 此时结束循环的条件就必须是 l >= r, 即 l 与 r 碰撞,

但有些题目只要求一个解, 那么我们就可以找到一个解就进行返回, 只有最坏的情况下(没有解)指针才会碰撞

答题时应该根据具体内容来具体分析.


滑动窗口

TCP协议中也有相应的技术, 简单地说就是通过向前移动两个指针, 来保证指针之间的区域内的数据符合要求

一张TCP的图, 但原理大致是一样的

在头部建立两个指针(l, r), 先滑动r指针, 让窗口内有足够元素, 之后根据需求右移l或r, 如果右移l, 窗口内元素会减少, 右移r, 窗口内元素会增多.

根据题目的不同结束条件也不大相同, 有的题目l到达最右边结束, 有的题目r达到最右边结束, 有的题目会固定窗口大小, 有的题目不会.


查找一个元素

Set 和 Map 是两个用于查找的数据结构. 而且map可以存储额外的信息.

相比于通过O(n^2)对两个数组进行遍历查找, 不如用O(n)的时间先把一个数组放入map中, 之后再用O(n)的时间遍历另外一个数组, 看map中有没有与之对应的元素. 后者的时间复杂度总体是O(n).

使用链表的一些技巧

递归天然地适合链表或者和链表相似的数据结构, 所以有些问题使用递归会更加方便


画图是很有效的整理思路的方法, 如果我们经常弄错应该把哪个节点指向哪个节点, 不妨画画图, 之后照着图进行编程.


有时我们需要对链表头进行特殊操作, 此时不妨设立虚拟头节点, 这样就可以统一操作, 返回时只要返回这个节点的next就好. 如果虚拟头节点是再堆中创建, 不要忘记delete掉, 这里更建议通过栈创建.


如果想把一个链表分成等长的两段, 可以设立两个节点, 一个slow 初始化再第一个节点, 一个fast 初始化再第二个节点, 每一次slow走一步, fast走两步, 直到fast无法再走, slow+1的位置就是第二段链表的头节点. 注意此时从0-slow 的节点个数是大于等于slow+1到最后的节点个数的. 具体为什么可以用笔画一画.


栈, 队列 以及 优先队列

栈通常用于深度优先遍历, 或者模拟递归

队列通常用于广度优先遍历, 或是求最短路径

优先队列通常用于动态选择数据


三者都是图和树常用的辅助结构.

顺带一提, 有些和图相关的算法不需要我们真的建立一张图, 只要心中有图即可

例如: leetcode # 279 题, 和每个数相邻的节点是有规律的, 只要把符合这些规律的元素放入队列中就可以进行广度优先遍历


再顺带一提, 我们可以通过维护一个vector or List 来保存是从哪个节点到达当前节点的, 由此在遍历完成后可以通过反向查找来找到来时的路径.


递归和二叉树

二叉树天然适合递归, 但是要注意递归的终止条件, 有的时候有多个终止条件, 有的时候终止条件和我们想的不太一样. 二叉树问题大部分情况下使用递归会使得题目的解法更简单, 如果没有循环的思路, 不妨先想想递归的解法

二分搜索树是二叉树的一种特殊情况(特殊结构), 解题时要充分利用它的特性


递归不仅仅可以适用与树形结构, 对于树形问题也可以使用递归.

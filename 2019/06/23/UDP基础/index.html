由源端口, 目标端口, 长度, 奇偶校验值 和 用户数据组成


UDP简介

报文结构简单, 说明支持的功能较少

UDP 不同于 TCP, 它不支持 错误重传, 滑动窗口等精细控制

UDP 特点

  1. 面向非连接, 传输数据之前, 源端和终端不建立连接. 当他想传送时就抓取应用程序的数据, 并尽可能快地扔到网络上. 在发送端, UDP 传输数据的速度仅仅受到应用程序生成数据速度, 计算机的能力 和 传输带宽影响

  2. 不维护连接状态, 支持同时向多个客户端传输相同的消息

  3. 数据包报头只有8个字节, 额外开销较小

  4. 吞吐量只受限于数据生成速率, 传输速率以及机器性能

  5. 尽最大努力交付, 不保证可靠交付, 不需要维持复杂的链接状态表

  6. 面向报文, 不对应用程序提交的报文信息进行拆分或者合并


TCP 和 UDP 的区别

面向连接 vs 无连接

可靠性:

​ TCP 可靠, 利用握手与重传机制提供了可靠性保证

​ UDP 可能会丢失, 不知道到底有没有被接受

有序性:

​ TCP 有序列号, 到达可能无序, 但tcp会排序

​ UDP 不具有有序性

速度:

​ TCP 速度慢, 要创建连接, 保证消息的可靠性和有序性

​ UDP 速度快, 适合对速度敏感的应用: 在线视频, 电视广播, 多人在线游戏…

量级:

​ TCP 重量级, 源数据头占20字节

​ UDP 轻量级, 源数据头占8字节

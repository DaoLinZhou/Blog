函数返回引用

int& foo(int* p){
    *p = 7;
    return *p;            // 返回的是容器, 而不是存在容器里的值
}

int main(){
    int a = 3;
    int b = foo(&a);     //foo(&a) 返回*p, *p 是 a, 这里可以看做是把a这个容器的值赋值给b
    foo(&a) = 6;        //foo(&a) 返回a这个容器, 这里把这个容器赋值成6, 就是把a赋值成6
                        //可以单纯地把foo(&a)看做是一个变量
    return 0;
}
int& foo(int* p){
    *p = 4;
    int q = *p;
    return q;        // 这里会返回q这个容器, 但是当退出时这块作用域内定义的变量会被消除, 所以会报错
}
int& foo(int* p){
    *p = 4;
    int* q = new int(9);
    return *q;        // 这里会返回q所指向的内存块, 而这个内存块是在heap中, 不会因为退出函数而被删除
                    // 所以这么写是正确的, 没有error
}

为什么不把所有的函数都设置成内联函数?

  • 内联编译是建议性的, 由编译器决定, 即使使用inline关键字, 也不一定进行内联编译

  • 内联函数逻辑必须简单, 不能包含for while循环等复杂结构, 调用频繁的函数建议使用内联

  • 递归函数无法使用内联方式

int main(){
    Coordinate coord[3];        // 在栈上实例化
    coord[1].m_iX = 10;

p指向第一个元素

p++ 后 指针就指向了第二个元素

此时p[0]访问的也是第二个元素

p– 后 指针会回到第一个元素

int main() {

    Coordinate *p = new Coordinate[3];    //执行所有对象的构造函数
    p->m_iY = 12;
    p->m_iX = 7;
    p[1].m_iY = 9;
    (++p)->m_iX = 10;    //p指向第二个元素
    p += 1;                //p指向第三个元素
    p->m_iX = 15;
    p->m_iY = 20;

    for(int i = 0; i < 3; i++){
        cout << p->m_iX << ", " << p->m_iY << endl;
        p--;            //p回退
    }
    p++;                //p指向第一个元素
    delete []p;            //释放p[], 执行所有对象的析构函数
    p = NULL;            //指针指空
    return 0;
}

深拷贝与浅拷贝

拷贝构造函数

浅拷贝

单纯的把所有属性赋值给新的对象


浅拷贝的缺点

如果对象有指针的话, 使用浅拷贝会使得两个指针指向同一个内存空间


缺点:

  1. 如果分别给两个指针赋值, 后者会覆盖掉原来的内容
  2. 销毁对象时会释放两个对象的内存, 如果已经释放一次, 再次释放系统会崩溃

深拷贝

我们希望拷贝时不是简单的把内存地址拷贝过来,

而是把指针指向内存的每一个元素依次地拷贝过来, 这就是深拷贝



上面首先新建了一片内存空间, 之后再逐一拷贝

virtual 的限制

前面讲过虚析构函数, 这里又讲虚函数, virtual有没有限制?

  1. virtual 只能修饰成员函数. 普通函数不能使虚函数

  2. virtual 不能修饰静态的成员函数

  3. 内联函数 (inline) 不能是虚函数

  4. 构造函数不能是虚函数


虚函数与虚析构函数的实现原理

函数指针

指针指向对象 –> 对象指针

指针指向函数 –> 函数指针


函数的本质就是一段二进制代码, 写在内存当中.

我们可以用指针指向代码的开头

其实函数指针和普通的对象指针没有区别

都是存储一段内存地址

实现原理的例子

class Shape{
public :
    virtual double calcArea(){
        return 0;
    }
protect:
    int edge;
};

class Circle: public Shape{
public:
    Circle(double r);
private:
    double r;
}

此时circle没有实现clacArea, 所以circle应该调用父类方法计算面积


当实例化Shape时, 出来实例化数据成员, 还实例化了一个虚函数表指针

虚函数表指针同样是指针, 占有4个内存单元

假设虚函数表的起始位置是0xccff, 那么虚函数表指针也就指向0xccff

如果实例化多个Shape对象, 它的虚函数表指针也指向0xccff, 确保每个对象的虚函数表指针都指向自己的虚函数表

虚函数表中一定有一个指针指向 calcArea 函数


调用时 先找到虚函数表指针 -> 找到虚函数表 -> 通过位置的偏移找到虚函数的入口地址 -> 找到虚函数



如果实例化Circle

circle中没有定义虚函数, 但它从父类中继承了虚函数

所以实例化Circle时也会产生虚函数表 ( circle的虚函数表 0x6688, 和上面Shape的虚函数表 0xccff 不同)

但是在这个虚函数表中 指向 calcArea 的指针却是一样的 0x3355

这就能保证通过circle 运行calcArea 能通过 找到自己的虚函数表 -> 找到calcArea的地址 并且执行

如果定义了 calcArea 那么会怎么样?

class Circle: public Shape{
public:
    Circle(double r);
    virtual double calcArea();
private:
    double r;
}

此时 circle 的虚函数表中 关于计算面积的指针 覆盖掉了父类中原有指针的值

此时调用的是Circle自己定义的虚函数

这就是多态的原理


函数的覆盖与隐藏

隐藏 当父类和子类出现同名函数, 子类函数把父类函数隐藏

覆盖 当子类没定义同名虚函数, 子函数的虚函数表中就会写上父类虚函数的入口地址

​ 当子类定义同名虚函数, 则会把父类虚函数的入口地址覆盖掉, 指向自己的虚函数入口地址


虚析构函数的实现原理

回顾 虚析构函数的作用:

当用父类的指针指向子类的对象时,

使用delete只能执行父类的虚构函数, 子类的虚构函数没有执行

而如果父类使用虚析构函数 则会同样执行子类的虚析构函数


理论前提:

执行完子类的析构函数就会执行父类的析构函数


所以如果我们delete执行子类的析构函数, 那么就会实现目标

class Shape{
public :
    virtual double calcArea(){
        return 0;
    }
    virtual ~Shape(){
        //...
    }
protect:
    int edge;
};

class Circle: public Shape{
public:
    Circle(double r);
    virtual ~Circle();
private:
    double r;
}

如果为父类析构函数添加virtual关键字 (子类会自动添加virtual, 但为了规范还是写上virtual关键字)

那么父类的虚析构表里 就会有一个父类析构函数的函数指针


子类也同样在析构表中有一个指向析构函数的函数指针

此时如果用父类指针指向子类对象, 那么delete时就可以

找到虚函数表指针 -> 找到虚函数表 -> 找到子类的析构函数 -> 从而指向子类的析构函数 -> 系统自动执行父类的析构函数

++运算符的重载

回想一下int如何进行++操作的

x++++; // 语法错误
因为x++ 返回的是一个值, 例如4, 而我们不能对一个值进行++操作

++++x++; // 正确 因为 ++ x 返回的是一个引用, 因此可以继续对它进行++操作
class Coordinate{
public:
    Coordinate(int x, int y);

    //区别: 前置返回引用,无参数
    //        后置返回对象, int 参数
    Coordinate& operator++();    //前置++    返回引用
    Coordinate operator++(int);    //后置++,    返回对象, int 是标识, 标识当前为后置重载
private:
    int x;
    int y;
};

Coordinate& Coordinate::operator++(){    //前置++
    x++;
    y++;

    return *this;
}

Coordinate operator++(int){        // int没有实际用处, 只是一个标志
    Coordinate old(*this);        // old 记录当前值,不改变本身的值, 如果接收后置++的值, 也是以前的值
    x++;                        // 但当下一行代码再去使用就会是新值
    y++;

    return old;                    // 返回对象    
}

上面的0是系统自动编译的, 没有任何意义, 只是标识

比较运算符重载

<, >, == … 比较运算符重载时, 甚至+-*/ 等等运算时, 尤其当我们要把类传入set, map等模版类库时,

一定要为函数添加const修饰, 因为他们只是比较, 而不改变内容. 如果不用const修饰, set, map会找不到重载的运算符, 因为他们内部生成的是const对象, 只能调用const函数

<<号运算符

返回值必须是 ostream&, 第一个参数必须是ostream 的引用

class Coordinate{
friend ostream& operator<<(ostream &out, const Coordinate &c);
public:
    Coordinate(int x, int y);

private:
    int x;
    int y;
};

ostream& operator<<(ostream &out, const Coordinate &coor){
    out << coor.x << "," << coor.y;
    return out;
}

可以看出 cout 就是一个ostream对象

输出运算符可以采用成员函数重载吗?

由于<<重载的必要条件是 第一个参数必须是ostream引用,

但成员函数的第一个对象却是隐性的this指针, 所有输出运算符不能进行重载


<< 只能通过友元运算符进行重载


[] 索引运算符
class Coordinate{
public:
    Coordinate(int x, int y);
    int operator[](int index);    //索引需要index
private:
    int x;
    int y;
};

int Coordinate::operator [](int index){
    if(0 == index)
        return x;
    if(1 == index)
        return y;
}

索引运算符可以使用友元函数重载吗?

不能, 因为索引运算符的第一个参数必须是this指针,

而友元函数却可以是其他值

索引运算符只能用成员函数重载


函数模板

有时候会有这种情况

函数大体一致, 但是参数不同,导致需要重复写同一份逻辑

解决方案是 把类型当做参数

关键字: template typename class (这里的class不是定义类的, 而是表明数据类型的, 和typename作用一样)

使用class定义:

template <class T>
T max(T a, T b){        // 函数模板, 如果没有使用, 不会产生任何代码数据
    return (a > b) ? a : b;
}

//....
//模板函数
int ival = max(100, 99);            //自动识别数据类型
char cval = max<char>("A", "B");    //指定数据类型

使用typename定义:

template <typename T>
void swap(T& a, T& b){
    T tmp = a; a=b; b=tmp;
}
//...
int x=20, y=30;
void swap<int>(x, y);

变量作为模板参数

template <int size>
void display(){
    cout << size << endl;
}

display<10>();

多参数函数模板

template <typename T, typename C>    //typename 不能省略
void display(T& a, C& b){
    cout << a << "," << b << endl;
}

int a = 1024; 
string b = "hello";
display<int, string>(a,b);

类模板

template <class T>
class MyArray{

public:
    void display(){...};    //类内定义
    void test();
private:
    T *arr;
};

template <class T>        //类外定义
void MyArray<T>::test(){    
    //...
}

template <class T, int size>
class Container{
public:
    void display();
private:
    T obj;
};

template <class T, int size>            //每一个成员函数前面都要加上
void Container<T, size>::display(){
    for(int i=0; i < size; i++){
        cout << obj << endl;
    }
}

int main(){
    Container<int 10> c;
    c.display();
    return 0;
}

重点: 模板代码不能分离编译, 必须把模板的声明和定义写在 .h 文件中, 不能一个写在.h 一个写在 .cpp中

vector 向量

向量就是对数组的封装

可以根据存储元素的个数自动变长

特点: 读取能在常数时间内完成

int main(){
    vector<int> vec;
    vec.push_back(10);    //在尾部插入元素10
    vec.push_pop();        //把10 抹掉
    cout << vec.size() << endl;

    for(int i=0; i < vec.size; i++)
        cout << vec[i] << endl;        //像数组一样使用

    return 0;
}

C语言操作文件

文件类型 FILE* file

打开文件 FILE* fopen(path, mode);

关闭文件 fcolse(FILE*);

#include <stdio.h>

int main(int argc, char* argv[]){

    FILE* file;
    char buf[1024] = {0,};
    // a append的意思,往文件里追加内容会添加到文件后面, 游标(可以理解为光标)在文件末尾
    // +    如果文件不存在 则创建, 有则直接打开
    file = fopen("1.txt","a+");

    // 每个元素大小为1, 有13个元素
    fwrite("hello, World!\n",1, 13, file);

    //将游标放到文件开头
    rewind(file);

    //读取文件到 buf 中, 每个元素大小为1, 读取10个
    fread(buf, 1, 10, file);

    //关闭文件
    fclose(file);

    printf("buf:%s\n", buf);

    return 0;
}

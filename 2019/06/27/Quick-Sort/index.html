快速排序

快速排序是21世纪最伟大的算法之一

许多系统内部就是使用快速排序来进行排序的(有点绕口??)

本文从最简单的快速排序开始, 演进写出性能较好的快速排序

普通快排

代码复杂程度: 简单

时间复杂度: O(nlogn)

空间复杂度: O(1)

稳定性: 不稳定

缺点:

  1. 对近乎有序的数组会退化成 n^2 级别的算法
  2. 基准元素的选择有规律(不随机)
  3. 没有考虑相等元素的情况

思想: 随机选一个数, 对数组进行整理, 左边都比这个数小, 右边都比他大, 这个数就在他排好序后应该在的位置


优化版普通快速排序

随机算法, 有极大的概率效率非常高, 极低的概率退化为 n^2 级别(近乎为0)


双路快排

代码复杂程度: 中等

时间复杂度: O(nlogn)

空间复杂度: O(1)

稳定性: 不稳定

思想: 和普通快排大致一样, 但要把等于标定点的元素均匀分布在大于和小于的部分

swap i 和 j


三路快排

代码复杂程度: 中等

时间复杂度: O(nlogn)

空间复杂度: O(1)

稳定性: 不稳定

特点: 处理有重复值的数组优势非常大, 系统级别一般用它. 而没有处理重复值数组的速度也可以保证

思想: 三条路线, 小于,等于,大于, 进行排序

p : less后下一个元素位置

q : equal后下一个元素位置

k : more 前一个元素


非递归三路快排

如果想把一个递归算法, 改写成非递归算法, 最好的辅助数据结构就是栈(stack)

大体思路没变, 只是把递归操作改为入栈操作

但由于使用了栈, 所以会消耗额外的数据空间


总结:

mergeSort 与 quickSort 都使用了分治算法, 而这两者又代表了分治算法的两种情况

mergeSort : 分简单, 归并难

quickSort : 分难,归并简单(不用归并, 只要分就可以)


获取数组中第N大的元素

通过快速排序, 每一次会把一个元素放大它排好序应该在的位置, 通过与这个位置比较我们能直到应该是向前处理还是向后处理. 直到找到我们想要的元素

最小生成树( Minimum Span Trees )是关于有权图的一个重要的算法

什么是生成树 : 对于一个有v个节点的图, 有v-1条边, 连接v个节点

一个图可以有多个生成树, 而 最小生成树 , 则是边的权重之和最小的生成树

最小生成树的应用:

  1. 电缆的布线设计 (保证所有节点都有供电, 而电缆的布置费用最低)
  2. 网络设计
  3. 电路设计
  4. … …

通常最小生成树是针对带权无向图, 并且是连通图.

如果不是连通图, 我们可以分别从图的每个连通分量上求得所有最小生成树, 而这个结果通常也被成为最小连通森林

一下算法默认图是连通图, 就像二分搜索算法默认数组有序一样.

目标 : 找 v-1 条边, 连接 v 个节点, 总权值最小


切分定理 Cut Property

切分定理 : 给定任意切分, 横切边中权值最小的边必然属于最小生成树

什么是切分?

定义 : 把图中节点分成两部分, 就是一个切分 (Cut)


什么是横切边?

定义 : 如果一个边的两个端点, 属于 切分(Cut) 不同的两边, 这个边称为 横切边(Crossing Edge)

例如上图中, 7-2 之间的边 0.34 就是一个横切边, 它连接蓝色节点和红色节点


简易证明:

切分把一个图分成了两个部分, 这两个部分一定有横切边 (如果没有横切边图就不连通了).

我们要证明: 权值最小的横切边一定属于最小生成树

假设:

​ 有一个横切边a, 它不是最小的横切边, 但它属于最小生成树. 同时, 蓝色部分和红色部分都已经求出了它们的最小生成树

​ 由于a不是最小横斜边, 那么就有横切边b小于a,

​ 因为b<a, 所以, b+蓝色部分最小生成树权值+红色部分最小生成树权值 < a+蓝色部分最小生成树权值+红色部分最小生成树权值 = 图的最小生成树

​ 因此有一个生成树的权值比图的最小生成树权值更小, 矛盾.


切分定理的应用

注意: 对于给定任意切分, 切分定理都生效.

如果我们对一个图进行切分, 让它分成这样两个部分:

  1. 只有一个节点
  2. 其余所有节点

那么这个节点的相邻边中最小的边就属于最小生成树


Lazy Prim 算法

选取一个节点, 通过切分定理不断扩散.

当前切分的最小横切边 (从优先队列中拿出, 并判断是否满足横切边的要求) 连接的节点划分到以访问的切分, 并且把这个节点的相邻边加入到所有横切边的优先队列中.


分析: 使用优先队列, 而且是求最小横切边, 那么一定要用最小堆. MinHeap 和 MaxHeap 相似, 所以就不放代码了

#ifndef ALGORITHM_LAZYPRIMMST_H
#define ALGORITHM_LAZYPRIMMST_H

#include <iostream>
#include <vector>
#include "Edge.h"
#include "../heap/MinHeap.h"

template <typename Graph, typename Weight>
class LazyPrimMST{

private:
    Graph &G;
    MinHeap<Edge<Weight>> pq;   // 优先队列 Priority queue
    bool* marked;               // 记录每个节点是否被访问过
    vector<Edge<Weight>> mst;   // 存储最小生成树的所有边
    Weight mstWeight;           // 记录最小生成树的总权值

    void visit(int v){
        assert(!marked[v]);
        marked[v] = true;

        // 遍历相邻边
        typename Graph::adjIterator adj(G, v);
        for(Edge<Weight>* e = adj.begin(); !adj.isEnd(); e = adj.next()){
            /* 首先当前节点已经访问过了, 如果这个边的另一个节点也被访问过了,
             * 这就不是横切边, 只有当没被访问过时才把这条边加入优先队列   */
            if( !marked[e->other(v)] )
                pq.insert(*e);
        }
    }

public:
    LazyPrimMST(Graph &graph):G(graph), pq(MinHeap<Edge<Weight>>(graph.E())){
        // 初始化
        marked = new bool[G.V()];
        for(int i = 0; i < G.V(); i++)
            marked[i] = false;
        mst.clear();

        // Lazy Prim
        visit(0);
        while(!pq.isEmpty()){
            Edge<Weight> e = pq.extractMin();
            /* 当边加入时两边的节点是不同的, 但有可能通过其他边访问过另一个节点
             * 导致两边节点都被访问过了, 所以在取出时要检查一下, 这里体现出Lazy */
            if(marked[e.v()] == marked[e.w()])
                continue;
            // 通过检查就说明属于最小生成树, 保存进向量
            mst.push_back(e);
            /* 这条边的两头一边被访问过, 另一边没有,
             * 继续要访问没有被访问过的那个点*/
            if(!marked[e.v()])  // 如果v点没被访问, 就访问v
                visit(e.v());
            else                // 反之访问w, 因为必定有一点没被访问
                visit(e.w());
        }

        // 计算mstWeight
        mstWeight = mst[0].wt();
        for(int i=1; i<mst.size(); i++)
            mstWeight += mst[i].wt();
    }

    ~LazyPrimMST(){
        delete [] marked;
    }

    vector<Edge<Weight>> mstEdges(){
        return mst;
    }

    Weight result(){
        return mstWeight;
    }

};

#endif //ALGORITHM_LAZYPRIMMST_H

时间复杂度分析: V(节点数) E(边数)

所有边都会进入最小堆, while循环执行 E(边数) 次, 每次执行有两个主要操作

  1. pq.extractMin log(E) 级别
  2. visit 遍历节点的所有邻边, 对于邻接表就是E级别, 对于邻接矩阵就是V^2级别. 但由于一般用邻接矩阵表示稠密图, 此时 V^2 约等于 E
    1. 而在每次遍历中要进行 insert 操作, 也是 log(E) 级别

综合来看 时间复杂度为 O(ElogE)


Prim 算法

Lazy Prim有两个缺点

  1. 就是所有的边都要进入最小堆, 但是随着切分的改变, 被访问过的节点越来越多, 很多在堆中的边已经不再是横切边了

  2. 虽然有很多横切边, 但我们只关注最短的横切边. 尤其是对于一个点, 我们只考虑它相邻边中最短的横切边就够了

所以我们需要一个数据结构, 满足以下条件

  • 存储和每个节点相连的最短的横切边
  • 在我们改变切分时, 更新和每个节点相连的最短的横切边

所以应该使用 IndexMinHeap

而这个 IndexMinHeap 只需要开节点个数的空间就足够了, 每个空间保存的就是和这个节点相连的最短的横切边

有V个节点, 而最小生成树只有V-1条边, 所以在 IndexMinHeap 中初始节点就不存东西了

#ifndef ALGORITHM_PRIMMST_H
#define ALGORITHM_PRIMMST_H

#include <iostream>
#include <cassert>
#include <vector>
#include "Edge.h"
#include "../heap/IndexMinHeap.h"

using namespace std;

template <typename Graph, typename Weight>
class PrimMST{
private:
    Graph &G;
    IndexMinHeap<Weight> ipq;       // 只保存权值
    vector<Edge<Weight>*> edgeTo;   // 存储和每个节点相邻的最短的横切边
    bool* marked;
    vector<Edge<Weight>> mst;
    Weight mstWeight;

    void visit(int v){
        assert(!marked[v]);
        marked[v] = true;

        // 遍历点v的相邻边
        typename Graph::adjIterator adj(G, v);
        for(Edge<Weight>* e = adj.begin(); !adj.isEnd(); e = adj.next()){
            // 从相邻边中取出相邻节点
            int w = e->other(v);
            // 如果没访问过
            if( !marked[w] ){
                if( !edgeTo[w] ){   // 当没有边和w连接时
                    ipq.insert(w, e->wt());     // 插入这条边
                    edgeTo[w] = e;  // 连接w的边是e
                }    // 当有边和w连接, 并且旧的边比这条边大时
                else if( e->wt() < edgeTo[w]->wt() ){  
                    edgeTo[w] = e;  // 连接w的边是e
                    // 更新index w在索引堆中的值
                    ipq.change(w, e->wt());
                }
                // 当有边和w连接, 并且旧的边比这条边小时, 什么都不做
            }
        }
    }

public:
    PrimMST(Graph &graph):G(graph), ipq(IndexMinHeap<double>(graph.V())){
        marked = new bool[G.V()];
        for(int i=0; i < G.V(); i++){
            marked[i] = false;
            edgeTo.push_back(NULL);
        }
        mst.clear();

        // Prim
        visit(0);
        while( !ipq.isEmpty() ){
            int v = ipq.extractMinIndex();  // 获取的是最小边的Index
            assert( edgeTo[v] );            // 确保有边连接v节点
            mst.push_back( *edgeTo[v] );    // 存入边
            visit(v);
        }

        // 计算mstWeight
        mstWeight = mst[0].wt();
        for(int i = 1; i < mst.size(); i++)
            mstWeight += mst[i].wt();
    }

    ~PrimMST(){
        delete[] marked;
    }

    vector<Edge<Weight>> mstEdges(){
        return mst;
    }

    Weight result(){
        return mstWeight;
    }

};

#endif //ALGORITHM_PRIMMST_H

时间复杂度分析: 堆只有v个元素, 而其余的时间复杂度和 Lazy Prim 一样, 所以是 Elog(V) 级别


Kruskal 算法

思想 : 如果每次都选择所有边中最小的一条边, 这条边是否就属于最小生成树呢?

答案是yes, 因为我们总能找到一个切分, 使得这条边是横切边中最小的边

我们可以不断的找这些最短边, 只要这些最短边不构成环

难点就是变成如何判断新加一条边会产生环?

解决方案就是使用并查集, 如果两个节点是连接的, 在添加一条边就会构成环

#ifndef ALGORITHM_KRUSKALMST_H
#define ALGORITHM_KRUSKALMST_H

#include <iostream>
#include <vector>
#include "../heap/MinHeap.h"
#include "../unionfind/UnionFind5.h"
#include "Edge.h"

using namespace std;

template <typename Graph, typename Weight>
class KruskalMST{

private:
    // 因为不需要辅助函数, 所以也不需要存一份graph的引用
    vector<Edge<Weight>> mst;
    Weight mstWeight;

public:
    KruskalMST(Graph &graph){
        MinHeap<Edge<Weight>> pq( graph.E() );
        // 遍历所有节点
        for(int i = 0; i < graph.V(); i++){
            // 把所有节点的相邻边都添加到优先队列中
            typename Graph::adjIterator adj(graph, i);
            for(Edge<Weight>* e = adj.begin(); !adj.isEnd(); e = adj.next()){
                /* 由于图是无向的, 所以要防止边的重复添加
                 * 例如 1<->2 这条边, 只添加1->2, 不添加2->1 */
                if( e->v() < e->w() )   
                    pq.insert(*e);
            }
        }
        // 自己实现的并查集
        UF5::UnionFind uf(graph.V());
        // 我们只需要v-1条边
        while( !pq.isEmpty() && mst.size() < graph.V() - 1 ){
            Edge<Weight> e = pq.extractMin();
            // 如果两点已经连接, 再连接就会成为环
            if(uf.isConnected(e.v(), e.w())) 
                continue;
            mst.push_back(e);
            uf.unionElements(e.v(), e.w());
        }

        mstWeight = mst[0].wt();
        for(int i = 1; i < mst.size(); i++)
            mstWeight += mst[i].wt();
    }

    ~KruskalMST(){}

    vector<Edge<Weight>> mstEdges(){
        return mst;
    }

    Weight result(){
        return mstWeight;
    }

};

#endif //ALGORITHM_KRUSKALMST_H

三种算法对比

算法 时间复杂度 算法实现
Lazy Prim O(ElogE) 一般
Prim O(ElogV) 复杂
Kruskal O(ElogE) 简单

如果横切边有相等的边会导致什么情况?

算法依然正确, 但会有多个最小生成树

而如果所有边都各不相同的话, 只有可能有一个最小生成树

同时也衍生出一个问题: 给定一个图, 它有多少个最小生成树


另一种算法思路

Vyssotsky’s Algorithm

按照任意的顺序, 将边都添加到生成树中

一旦形成环, 删除环中权值最大的边


这个算法在许多教材中都没有, 因为这个算法要删除边, 还要保证删除的是环上最大的边, 我们不仅要探测环的存在, 还要找到环上最大的边. 暂时没有数据结构可以有效地实现这个功能


而在Kruskal中使用了并查集, Prim和Lazy Prim中用了最小索引堆和最小堆这些高级数据结构. 而在很长时间里, Kruskal 和 Prim算法都没有很好的实现方法, 直到这些数据结构被发现,

并查集主要支持两种操作

Union(并) 和 Find(查)

实现一 quickFind

数组index代表节点, 存储的值代表类型.

quickFind 版本, find的复杂度为O(1), 但union复杂度为O(n), 是性能最差的实现.

    class UnionFind {
    private:
        int *id;
        int count;

    public:
        UnionFind(int n) {
            count = n;
            id = new int[n];
            for (int i = 0; i < n; i++)
                id[i] = i;
        }

        ~UnionFind() {
            delete[] id;
        }

        int find(int p) {
            assert(p >= 0 && p < count);
            return id[p];
        }

        int isConnected(int p, int q) {
            return find(p) == find(q);
        }

        void unionElements(int p, int q) {
            int pID = find(p);
            int qID = find(q);
            if (pID == qID)
                return;
            for (int i = 0; i < count; i++)
                if (id[i] == qID)
                    id[i] = pID;
        }
    };

实现二 quickUnion

quickUnion 版本的并查集union速度更快

数组index为节点, 数组内存储的值为父亲节点

但是union操作时我们只是随便把 p 的根节点指向 q 的根节点, 这里依然可以进行优化

class UnionFind {
private:
    int count;
    int* parent;

public:
    UnionFind(int n) {
        parent = new int[n];
        for(int i=0;i < n; i++)
            parent[i] = i;
    }

    ~UnionFind() {
        delete [] parent;
    }

    int find(int p){
        assert(p >= 0 && p < count);
        while (p != parent[p])
            p = parent[p];
        return p;
    }

    bool isConnected(int p, int q){
        return find(p) == find(q);
    }

    void unionElements(int p, int q){
        int pRoot = find(p);
        int qRoot = find(q);
        if(pRoot == qRoot)
            return;
        parent[pRoot] = qRoot;
    }
};

基于size的优化

在声明一个sz数组, 和parent等长, 初始化为1, 存储以index为根的节点下有多少节点

private:
        int count;
        int* parent;
        int* sz;

把size少的指向size多的

void unionElements(int p, int q){
    int pRoot = find(p);
    int qRoot = find(q);
    if(pRoot == qRoot)
        return;
    if(sz[pRoot] < sz[qRoot]){
        parent[pRoot] = qRoot;
        sz[qRoot] += sz[pRoot];
    }else{
        parent[qRoot] = pRoot;
        sz[pRoot] += sz[qRoot];
    }
}

但是有一种极端情况, 如果size多的高度反而比size少的低, 那么这部操作依然不能很好地优化并查集

所以, 更好的方法是基于rank进行优化, 注意rank不是高度


基于rank的优化

private:
        int count;
        int* parent;
        int* rank;
void unionElements(int p, int q){
    int pRoot = find(p);
    int qRoot = find(q);
    if(pRoot == qRoot)
        return;
    if(rank[pRoot] < rank[qRoot]){
        parent[pRoot] = qRoot;
    }else if(rank[pRoot] > rank[qRoot]){
        parent[qRoot] = pRoot;
    } else {    // rank[pRoot] == rank[qRoot]
        parent[pRoot] = qRoot;
        rank[qRoot] += 1;
    }
}

把rank小的指向rank大的, 如果一样那就随便其中一个指向另一个

注意union操作时, 如果 rank[pRoot] < rank[qRoot], 那么这两个节点的rank至少差1, 所以union后不用自增.

但如果两个rank相等, union之后要把rank加一


路径压缩

名词听起来高大上, 但实际实现很简单

第一种方案就是把当前节点指向父亲节点的父亲节点,

如果父亲节点是第一个节点, 它的父亲节点是自身, 逻辑依然没有问题

int find(int p){
    assert(p >= 0 && p < count);
    while (p != parent[p]) {
        parent[p] = parent[parent[p]];
        p = parent[p];
    }
    return p;        
}

第二种方案则是把每个节点直接指向父亲节点的根节点

int find(int p){
    if(p != parent[p])
        parent[p] = find(parent[p]);
    return parent[p];
}

理论上, 第二种方式的效率应该比第一种更高, 但实际两者差距不大, 有时候第一种速度反而更快

具体选择时那种都可以

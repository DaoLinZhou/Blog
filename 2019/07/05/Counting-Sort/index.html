非基于比较的排序算法, 快于任何比较排序算法

时间复杂度: Ο(n+k)

空间复杂度: O(k+k)

这是一种牺牲空间换取时间的做法，而且当O(k)>O(n*log(n))的时候其效率反而不如基于比较的排序

从一道练习题说起

我的计算机老师在讲解数组之前, 出了一个刁难的小问题.

不用数组的情况下, 随机生成100 个 [-3, 3] 之间的整数, 顺序地打印它们?

我当时第一个想法就是申明7个变量, 分别对应[-3, 3]. 每当随机生成一个值, 就依次对比[-3, 3], 之后再对相应的变量进行自增.

假设 A 代表 -2, A 的值为 5, 就证明生成了 5 个 -2

这就是计数排序的大致思想.


实现代码

void countingSort(int arr[], int n, int min, int max){
    int *aux = new int[(max-min+1)];
    //初始化
    for(int i=0; i<(max-min+1); i++)
        aux[i] = 0;
    //开始计数
    for(int i=0; i<n; i++)
        aux[arr[i]-min]++;
    //填充,i指向aux, index指向arr
    for(int i=0, index=0; i<(max-min+1); i++)
        for(int j=0; j<aux[i]; j++, index++)
            arr[index] = i+min;
    delete[](aux);
}

基数排序 Radix sort

假设我们有一组数

329, 557, 457, 226, 720, 657, 449, 355, 839, 510, 719, 845.

要对他们进行排序


基数排序的思想就是针对每一位进行排序.

第一轮, 针对个位数进行排序:


第二轮, 针对十位数排序, 当10位数相同时, 根据个位数的大小进行排序

(实现思路: 根据第一轮排序的结果, 从前到后(从小到大)进行第二轮的排序, 这样个位数小的就会在上面)


同理, 第三轮排序是针对百位数的.


由于最高位就是百位, 所以这就排好序的.

这个算法的时间复杂度为O(n*k), n为数据个数, 数据最大为k位.

当n非常大, 而k非常小时, 是比O(nlogn)快的.

但是空间复杂度非常高. O(n * k). 但是, 也有一些方法可以降低空间复杂度


example:

使用radix sort 对等长的string进行排序

void radixsort(vector<string>& vec, int len){

    vector<vector<string>> record(26);

    for(int i = len-1; i >= 0; i--){
        for(string& s : vec){
            record[s[i]-'a'].push_back(s);
        }

        // copy back
        int j = 0;
        for(vector<string>& v : record){
            for(string& s : v){
                vec[j] = s;
                j++;
            }
            v.clear();
        }
    }
}

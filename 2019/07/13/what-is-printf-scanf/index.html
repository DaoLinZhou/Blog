Linux 的主要思想是 : “一切皆文件”

外接一个打印机, 它会把打印机当做文件

插入U盘就会把U盘当做文件

外接摄像头, 也会把摄像头当做文件

所有的外部设备和磁盘中的文件是一样一样的

三个文件

当我们要启动程序的时候, 就会有三个文件 stdin, stdout, stderr

这三个文件分别是标准输入流, 标准输出流, 标准错误流.


默认stdin的输入设备是键盘, 但其实可以通过流的重定向来进行更改, 让不同设备给应用程序输入数据

默认stdout的输出设备是显示器终端, 如果切换到网卡上, 就可以向网卡写入数据. 如果切换到打印机上, 就可以让打印机打印数据


Linux 认为, 任何一个应用程序, 都会有可能会和文件或设备进行交互

所以每当启动c语言程序时, 都会创建三个文件

#include <stdio.h>

int main(){
    //fprintf: 向一个文件里写入数据, 这里让它把数据写到输出文件(流)上, 就是输入一段话, 等价于下面语句
    //如果使用打印机的资源句柄的话, 就是让打印机把这句话打出来
    //printf("please input the value of a:\n")
    fprintf(stdout,"please input the value of a:\n");

       int a;
    //fscanf: 从一个文件中读取数据, 同理
    //scanf("%d", &a);
    fscanf(stdin, "%d", &a);

    if(a<0){
        fprintf(stderr, "the value must > 0");
        return 1;
    }

    return 0;
}

重定向输入输出流

Linux通道的设计方式非常重要

这个小程序如下

当 j 不等于 0 时正常输出

当 j 等于 0 时对输入数据到标准错误流

#include <stdio.h>

int main(){
    printf("input the int value i:\n");
    int i, j;
    scanf("%d", &i);
    printf("input the int value j:\n");
    scanf("%d", &j);
    if(0!=j){
        printf("%d/%d=%d\n",i,j,i/j);
    }else{
        fprintf(stderr, "j != 0\n");
        return 1;
    }
    return 0;
}

假设这个程序编译后为a.exe

./a.exe        //运行这个文件
./a.exe 1 >> out.txt    // 1代表标准输出流, 默认>>就是1
                        // 即把标准输出流指向out.txt文件
                        // 即把输出结果写入out.txt文件
                        // 不覆盖原本内容, 在末尾追加

./a.exe 1 > out.txt        // 效果同上, 但覆盖原本内容之后写入文件

./a.exe 2 >> err.txt     // 错误流重定向, return值不为0
./a.exe 2 > err.txt        // 效果同上, 但覆盖原本内容之后写入文件

./a.exe 0 < info.txt    // 0代表标准输入流, <默认是0    
                        // 即把标准输入流指向info.txt文件
                        // 即读取info.txt中的值作为输入

//同时使用
./a.exe 1>out.txt 2>err.txt 0<info.txt

注意: 输出文件不存在会自动创建, 但输入文件不能不存在

堆 heap (优先队列)

特点: 堆 非常适合作为 优先队列 的实现, 新添加的元素可以通过shiftup快速找到自己对应的位置

优先队列: 出队顺序和入队顺序无关, 和优先级相关

二叉堆 Binary Heap

特点:

  1. 任何一个节点都不大于它的父亲节点 (最大堆, 树顶的元素总是最大的元素)
  2. 二叉堆 是一颗 完全二叉数
    1. 完全二叉树: 除了最后一层, 其余每次节点个数都必须是最大值
    2. 最后一层的所有节点都集中在左侧

原地堆排序

思想: heapify, 把最大的元素(arr[0]) 和 最后一个数互换, 再次heapify, 堆边界自减, 直到等于0(等于0时不用再heapify, 一个元素不用排序)

注意: 原地堆排序是不用MaxHeap那个类的


索引堆(Index Heap)

为什么要使用索引堆?

二叉堆的局限性:

  1. 当存储的元素十分复杂的时候, 进行元素交换消耗是巨大的(例如:字符串)

  2. 由于堆操作改变了元素的位置, 所以索引对应的元素会变得分常困难(只有遍历一边才能找到相应的元素, 效率低下), 而当我们很难索引到它的时候就很难改变它

索引堆的例子

堆的扩展与应用

ex) 在N个元素中选出前M名

在1000000个元素中选出前100名

更普遍的: 在N个元素中选出前M名

使用最小堆, 每次添加元素后如果大于M名则移除最小的元素(维持堆只有M个元素, 但由于要添加元素所以堆至少要M+1的空间, 如果堆的起始位置是从1开始则要M+2的空间, 这里要注意边界问题), 等N个元素都添加进堆中时, 堆中的M个元素答案


ex) 多路归并排序

原本的归并排序是把数组分成两个有序部分(两路), 但我们可以用堆来进行多路归并排序

假设我们要归并4个有序的数组, 把所有数组最小元素放入最小堆, 每次从堆中退出一个最小元素. 被推出的元素属于哪个子数组就从哪个子数组中再添加一个新元素进堆

这样我们就可以进行k路归并排序, k为变量. k越大, 归并的层数就越小. 而归并时需要比较的元素就越多.

如果数组中有n个元素, 而我们要做n路归并排序, 只需要递归一层, 而堆中要存储n个元素.

此时归并排序退化成了堆排序


d叉堆

不同与二叉堆, 堆可以有d叉, 只要满足堆的性质就可以


最大最小队列

最大堆只能获取最大的元素, 而最小堆只能获取最小的元素, 有没有一种数据结构能轻易获取最大元素或者最小元素?

思路1: 同时维护最大索引堆和最小索引堆,

移除最小元素时通过index从最大索引堆中移除

移除最大元素时通过index从最小索引堆中移除


思路2: 使用平衡二叉搜索树


双端队列的底层就是最大最小堆

堆 heap (优先队列)

特点: 堆 非常适合作为 优先队列 的实现, 新添加的元素可以通过shiftup快速找到自己对应的位置

优先队列: 出队顺序和入队顺序无关, 和优先级相关

二叉堆 Binary Heap

特点:

  1. 任何一个节点都不大于它的父亲节点 (最大堆, 树顶的元素总是最大的元素)
  2. 二叉堆 是一颗 完全二叉数
    1. 完全二叉树: 除了最后一层, 其余每次节点个数都必须是最大值
    2. 最后一层的所有节点都集中在左侧 
#ifndef ALGORITHM_HEAP_H
#define ALGORITHM_HEAP_H

#include <iostream>
#include <ctime>
#include <cassert>

using namespace std;

template <typename T>
class MaxHeap{
public:
    MaxHeap(int capacity) : capacity(capacity+1) {
        size = 0;
        data = new T[capacity+1];
    }

    int getSize(){
        return size;
    }

    bool isEmpty(){
        return size==0;
    }

    void insert(T item){
        assert(size < capacity);
        data[++size] = item;
        shiftUp(size);
    }

    T extractMax(){
        assert(size > 0);
        T ret = data[1];
        swap(data[1], data[size--]);
        shiftDown(1);
        return ret;
    }


    virtual ~MaxHeap() {
        delete[] data;
    }

private:
    int capacity;
    T* data;
    int size;

    void shiftUp(int index){
        while(data[index/2] < data[index] && index > 1){
            swap(data[index/2], data[index]);
            index /= 2;
        }
    }

    //待优化: swap 可以改为赋值, 插入排序的优化
    void shiftDown(int index){
        while(index*2 <= size){
            int j = index*2;
            if(j+1 <= size && data[j] < data[j+1])
                j++;
            if(data[index] >= data[j])
                break;
            swap(data[index], data[j]);
            index = j;
        }
    }

};

#endif

堆排序

template <typename T>
void heapSort(T arr[], int n){
    MaxHeap<int> *maxHeap = new MaxHeap<int>(n+1);
    //添加元素
    for(int i=0; i<n; i++)
        maxHeap->insert(arr[i]);
    //取出最大的元素
    for(int i = n-1; i >= 0; i--)
        arr[i] = maxHeap->extractMax();
}

Heapify

将n个元素逐一插入到堆中的时间复杂度是O(nlogn)

heapify的过程时间复度是O(n)

思想: 把每个非叶子节点从后向前进行shiftDown从而建立堆

构造函数:

    MaxHeap(const int arr[], int n){
        data = new T[n+1];
        capacity = n;
        for(int i=0; i < n; i++)
            data[i+1] = arr[i];
        size = n;
        for(int i = n/2; i >= 1; i--)
            shiftDown(i);
    }

原地堆排序

思想: heapify, 把最大的元素(arr[0]) 和 最后一个数互换, 再次heapify, 堆边界自减, 直到等于0(等于0时不用再heapify, 一个元素不用排序)

注意: 原地堆排序是不用MaxHeap那个类的

template <typename T>
void __shiftDown(T arr[], int n, int i) {
    while(i*2+1 < n){
        int j = i*2+1;
        if(j+1 < n && arr[j+1] > arr[j])
            j += 1;
        if(arr[i] >= arr[j])
            break;
        swap(arr[i], arr[j]);
        i = j;
    }

}

template <typename T>
void heapSort(T arr[], int n){
    //heapify
    for(int i = (n-1) / 2; i >= 0; i--)
        __shiftDown(arr, n, i);
    for(int i = n-1; i > 0; i--){
        swap(arr[0], arr[i]);
        __shiftDown(arr, i, 0);
    }
}

索引堆(Index Heap)

为什么要使用索引堆?

二叉堆的局限性:

  1. 当存储的元素十分复杂的时候, 进行元素交换消耗是巨大的(例如:字符串)

  2. 由于堆操作改变了元素的位置, 所以索引对应的元素会变得分常困难(只有遍历一边才能找到相应的元素, 效率低下), 而当我们很难索引到它的时候就很难改变它

索引堆的例子

把索引和数据分开存储, 而真正表征堆的数组, 是由索引构建成的

实现的时候只要清楚每个变量所代表的意义, 就能自然的写出代码

#include <iostream>
#include <ctime>
#include <cassert>
#include <algorithm>

using namespace std;

template <typename T>
class IndexMaxHeap{
private:
    int count;
    T* data;
    int* indexes;
    int capacity;

public:
    // 由于是从1开始存储元素
    IndexMaxHeap(int capacity) : capacity(capacity) {
        data = new T[capacity+1];
        indexes = new int[capacity+1];
        count = 0;
    }

    virtual ~IndexMaxHeap() {
        delete[] data;
        delete[] indexes;
    }

    int getCount(){
        return count;
    }

    bool isEmpty(){
        return count == 0;
    }

    //对于用户来说, 是从0开始
    void insert(int i, T item){
        assert(count+1 <= capacity);
        assert(i+1 >= 1 && i+1 <= capacity);
        i += 1;
        /* 数据是添加在 index的位置, 如果extractMaxIndex后data[i]的元素依然留着,
           再次为它添加不会造成空间浪费, 而是覆盖原有的值 */
        data[i] = item;
        // index 是添加在最末尾
        indexes[++count] = i;
        shiftUp(count);
    }

    T extractMax(){
        assert(count > 0);
        T ret = data[indexes[1]];
        swap(indexes[1], indexes[count--]);
        shiftDown(1);
        return ret;
    }

    // 索引堆支持的特殊操作, 返回索引
    int extractMaxIndex(){
        assert(count > 0);
        int ret = indexes[1] - 1;
        swap(indexes[1], indexes[count--]);
        // 不用考虑data中的元素, 因为indexes中没有的话就无法被访问到
        shiftDown(1);
        return ret;
    }

    T getMax(){
        assert(count > 0);
        return data[indexes[1]];
    }

    // 索引堆支持的特殊操作, 通过索引获取元素
    T getItem(int index){
        assert(index+1 >= 1 && index+1 <= capacity);
        return data[index+1];
    }

    // 索引堆支持的特殊操作, 通过索引改变元素
    void change(int i, T newItem){
        i+=1;
        data[i] = newItem;

        // 由于改变了值, 可能会导致堆的规则被破坏, 所以要尝试向上移,向下移
        // 找到indexes[j] = i, j表示data[i]在堆中的位置
        // 之后shiftUp(j), 再shiftDown(j)
        for(int j = 1; i <= count; j++){
            if(indexes[j] == i){
                shiftUp(j);
                shiftDown(j);
                return;
            }
        }
    }

private:
    void shiftUp(int i){
        while(i > 1 && data[indexes[i]] > data[indexes[i/2]]){
            swap(indexes[i], indexes[i/2]);
            i /= 2;
        }
    }
    void shiftDown(int i){
        while((i*2) <= count){
            int j = i*2;
            if(j+1 <= count && data[indexes[j+1]] > data[indexes[j]])
                j += 1;
            if(data[indexes[i]] >= data[indexes[j]])
                break;
            swap(indexes[j], indexes[i]);
            i = j;
        }
    }
};


// 索引堆的用途并不是排序, 但是这里要用排序来证明代码的正确性
template <typename T>
void IndexMaxHeapSort(T arr[], int n){
    IndexMaxHeap<T> *indexMaxHeap = new IndexMaxHeap<T>(n);
    for(int i=0; i<n; i++)
        indexMaxHeap->insert(i, arr[i]);
    assert(indexMaxHeap->testIndexes());
    for(int i=n-1; i>=0; i--)
        arr[i] = indexMaxHeap->extractMax();
}

优化索引堆

为普通的索引堆添加反向查找表

将数据和索引分离开来 or 加入反向查找表 都是算法设计中经常使用的方法

具体实现:

实现没什么难度, 就是注意从用户角度是从0开始. 以及时刻维护 index 表和 rev 表

template <typename T>
class IndexMaxHeap{
private:
    int count;
    T* data;
    int* reverse;   //反向查找
    int* indexes;
    int capacity;

public:
    // 由于是从1开始存储元素
    IndexMaxHeap(int capacity) : capacity(capacity) {
        data = new T[capacity+1];
        indexes = new int[capacity+1];
        reverse = new int[capacity+1];
        for(int i=0; i<=capacity; i++)
            reverse[i] = 0;
        count = 0;
    }

    virtual ~IndexMaxHeap() {
        delete[] data;
        delete[] indexes;
        delete[] reverse;
    }

    int getCount(){
        return count;
    }

    bool isEmpty(){
        return count == 0;
    }

    //对于用户来说, 是从0开始
    void insert(int i, T item){
        assert(count+1 <= capacity);
        assert(i+1 >= 1 && i+1 <= capacity);
        i += 1;
        /* 数据是添加在 index的位置, 如果extractMaxIndex后data[i]的元素依然留着,
           再次为它添加不会造成空间浪费, 而是覆盖原有的值 */
        data[i] = item;
        // index 是添加在最末尾
        indexes[count+1] = i;
        reverse[i] = count+1;
        count ++;
        shiftUp(count);
    }

    T extractMax(){
        assert(count > 0);
        T ret = data[indexes[1]];
        swap(indexes[1], indexes[count]);
        reverse[indexes[1]] = 1;
        reverse[indexes[count]] = 0;    //不存在的指向0
        count --;
        shiftDown(1);
        return ret;
    }

    // 索引堆支持的特殊操作, 返回索引
    int extractMaxIndex(){
        assert(count > 0);
        int ret = indexes[1] - 1;
        swap(indexes[1], indexes[count]);
        reverse[indexes[1]] = 1;
        reverse[indexes[count]] = 0;    //不存在的指向0
        count--;
        // 不用考虑data中的元素, 因为indexes中没有的话就无法被访问到
        shiftDown(1);
        return ret;
    }

    T getMax(){
        assert(count > 0);
        return data[indexes[1]];
    }

    // 新增函数, O(1)复杂度内判断是否包含索引i
    bool contain(int i){
        assert(i+1 >= 1 && i+1 <= capacity);
        return reverse[i+1] != 0;
    }

    // 索引堆支持的特殊操作, 通过索引获取元素
    T getItem(int index){
        assert( contain(index) );
        return data[index+1];
    }

    // 索引堆支持的特殊操作, 通过索引改变元素
    void change(int i, T newItem){
        assert( contain(i) );
        i+=1;
        data[i] = newItem;

        // 由于改变了值, 可能会导致堆的规则被破坏, 所以要尝试向上移,向下移
        // 找到indexes[j] = i, j表示data[i]在堆中的位置
        // 之后shiftUp(j), 再shiftDown(j)
//        for(int j = 1; i <= count; j++){
//            if(indexes[j] == i){
//                shiftUp(j);
//                shiftDown(j);
//                return;
//            }
//        }
        int j = reverse[i];
        shiftUp(j);
        shiftDown(j);
    }

private:
    void shiftUp(int i){
        while(i > 1 && data[indexes[i]] > data[indexes[i/2]]){
            swap(indexes[i], indexes[i/2]);
            reverse[indexes[i/2]] = i/2;
            reverse[indexes[i]] = i;
            i /= 2;
        }
    }
    void shiftDown(int i){
        while((i*2) <= count){
            int j = i*2;
            if(j+1 <= count && data[indexes[j+1]] > data[indexes[j]])
                j += 1;
            if(data[indexes[i]] >= data[indexes[j]])
                break;
            swap(indexes[j], indexes[i]);
            reverse[indexes[i]] = i;
            reverse[indexes[j]] = j;
            i = j;
        }
    }
};

堆的扩展与应用

ex) 在N个元素中选出前M名

在1000000个元素中选出前100名

更普遍的: 在N个元素中选出前M名

使用最小堆, 每次添加元素后如果大于M名则移除最小的元素(维持堆只有M个元素, 但由于要添加元素所以堆至少要M+1的空间, 如果堆的起始位置是从1开始则要M+2的空间, 这里要注意边界问题), 等N个元素都添加进堆中时, 堆中的M个元素答案


ex) 多路归并排序

原本的归并排序是把数组分成两个有序部分(两路), 但我们可以用堆来进行多路归并排序

假设我们要归并4个有序的数组, 把所有数组最小元素放入最小堆, 每次从堆中退出一个最小元素. 被推出的元素属于哪个子数组就从哪个子数组中再添加一个新元素进堆

这样我们就可以进行k路归并排序, k为变量. k越大, 归并的层数就越小. 而归并时需要比较的元素就越多.

如果数组中有n个元素, 而我们要做n路归并排序, 只需要递归一层, 而堆中要存储n个元素.

此时归并排序退化成了堆排序

内存由操作系统统一管理

一个字节Byte有8个二进制位

计算机插两根2G内存条 等同于 插一根4G内存条

无论插多少根内存条, 计算机会把内存看成是一个整体来计算内存大小

32位操作系统

32 位操作系统最大使用4G内存


32 位操作系统上, 地址总线是32位, 即寻址空间是32位

32 位指的是: 给内存编号只能编到32个二进制位


两根地址总线有4种状态

32根地址总线就有2^32个状态, 2^32字节
$$
2^{32} = 2^{10} * 2^{10} * 2^{10} * 2^2
$$

所以32位操作系统最多只能管理4G内存


64位操作系统

64位主机可以管理 2^64 字节

2^30 => GB

2^40 => TB

… …

64位系统下的内存足够使用了


内存地址

操作系统会对所有的内存进行编号, 编号 = 唯一的内存字节的地址, 1个字节存放8个二进制位的数据

64 位操作系统内存地址编号, 二进制表示

0000 0000 … … 0000 0000 (64 个 0)

1111 1111 … … 1111 1111 (64 个 1)


右边的每一个格子都代表计算机中的一个字节, 每个字节可以保存8个二进制位的数据, 假设他们都是0

最高位 0xffffffffffffffff 16个f, 因为一个f 对应 1111 四个一, 16 * 4 = 64, 相当于64 个 1


为什么要交给操作系统管理

计算机中要同时运行多个程序

如果由程序员对内存直接进行管理, 可能会出现多个程序对同一个地址来操作

那么该内存应该分给那个程序

我们不知道用户会在什么时候启用哪些程序, 也不知道哪个程序会占用哪个内存, 所以内存不应由程序员直接管理


当我们买计算机时, 第一个应用程序就是操作系统. 其他的应用程序都是安装到操作系统上的. 所以实际上 应用程序是由操作系统来调用的


操作系统 不仅能给内存编号, 还可以给内存进行一定的规划

在64位操作系统中: 系统认为程序员使用的内存只要有前面48位就可以了

即上图红框区域 0x0 到 0x7fffffffffffffff

而0x7fffffffffffffff 之上的内存空间是给操作系统内核使用的


用户内存和操作系统内存隔离的好处

  • 操作系统的内存不会被大量占用
  • 避免机器卡住, 卡死, 死机等状态 (哪怕用户内存全部沾满, 操作系统内存也不会受到影响)
  • 可以通过操作系统把应用程序关闭
  • 使操作系统更安全

假设我们写了一个c语言代码, 编译后存到磁盘

一旦我们运行这个程序, 代码编译后的二进制数据就会加载到内存中, 即代码段

我们可能会声明一些全局变量, 或者常量.这些放在数据段

绿色区域为自由可分配内存

栈: 第一个执行的函数, 如果这个函数调用其他函数会依次往下压栈.

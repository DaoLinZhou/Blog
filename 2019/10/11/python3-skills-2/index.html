处理二进制文件(wav举例)

这里以处理wav音频文件为例

import struct
import numpy as np

# 获取subchunk == name的位置和subchunk的size
def find_subchunk(f, name):
    # 如图所示, 前12位是我们不需要的
    f.seek(12)
    while True:
        # 读取subchunk的id和size
        chunk_name = f.read(4)

        # 使用struct的unpack把二进制转换为int
        # 因为int是4个字节, 而如图chunk的size也是4个字节
        chunk_size, = struct.unpack('i', f.read(4))

        if chunk_name == name:
            # 返回当前位置, 和size
            return f.tell(), chunk_size

        # 0 代表从头, 1 代表当前点, 2 代表文件结尾
        # 从当前位置开始定位, 定位到下一个subchunk的位置
        f.seek(chunk_size, 1)


if __name__ == '__main__':

    # b 表示二进制
    f = open('test.wav', 'rb')

    # 获取data的信息的位置和大小
    offset, size = find_subchunk(f, b'data')

    # 使用numpy创建缓冲
    buf = np.zeros(size//2, dtype=np.short)

    # 把所有信息读到buffer
    f.readinto(buf)

    # 对buf中, 所有元素进行/=16
    # 具体的表现则是声音变小
    buf //= 16

    # 新建文件
    f2 = open('out.wav', 'wb')

    # 重新定位到开头
    f.seek(0)

    # 先把前面的信息副职过去
    info = f.read(offset)
    f2.write(info)
    # 把处理好的data写入f2
    buf.tofile(f2)

    f2.close()
    f.close()

    # 二进制模式下返回bytes, 而不是str
    # info = f.read(100)
    # print(info)

得到的新的wav文件的声音就比原本的声音小


如何设置文件缓冲

将文件写入到硬件设备时, 使用系统调用, 这类I/O操作的时间很长, 为了减少IO操作的次数, 文件通常使用缓冲区(就是等有足够多的数据之后才进行系统调用). 文件的缓冲行为分为: 全缓冲, 行缓冲, 无缓冲


全缓冲:

缓冲区的大小是固定的, 当缓冲区满了后会进行系统调用(system call), 通过系统调用把内存中的内容写到硬盘中

# 查看a.bin尾部的内容
tail -f a.bin

监听a.bin这个文件的变化

f = open('a.bin', 'wb')
f.write(b'abcefg')

然而此时tail并没有显示任何信息, 说明abcefg并没有真实的写入磁盘, 而是进入缓冲区当中, 且没有填满缓冲区

缓冲区的大小是多大呢? 这个和磁盘相关.

如果在驱动信息中找不到, 则会使用 io.DEFAULT_BUFFER_SIZE, 即8192

这里假设缓冲区为4096

f.write(b'+'*4090) # 之前已经写了6个字节, 在写4090个

依然没有任何信息, 但是如果

f.write(b'-')

此时就会写入到文件中, 也就是说如果写入的字节超过缓冲区范围就会往文件中进行写入

而无缓冲就是把缓冲区设为0, 让缓冲区无法容纳任何东西, 使得每个字节都要写入文件

注意上面的操作都是使用二进制进行读取的


如果我们使用字符流进行读取却有些不一样

f2 = open('a.txt', 'w')
f2.write('abc'*5000)

文件依然没有被写入

原因如下:

print(type(f))  # <class '_io.BufferedWriter'>
print(type(f2)) # <class '_io.TextIOWrapper'>

print(f2.buffer) 
# <_io.BufferedWriter name='E:\\py_WorkSpace\\PySkills\\read_write_file\\a.txt'>

# 就变成以二进制打开文件了, 这说明文本模式打开是基于二进制打开的

三层模型

如图所示, TextIOWrapper是基于BufferedWriter实现的,

BufferedWriter是处理二进制用的, 而TextIO是处理文本文件用的, 可以看出TextIO的主要作用是进行编解码.

在BufferedWriter之下是raw, raw是无缓冲直接写入

# 层次在上面的总是可以调用底层的方法
# 这么做就是跳过缓冲区, 直接写入
f.buffer.raw.write(b'b')

而TextIO也有缓冲, 说明是两级缓冲, 而我们之前写5000 个依然没有反应说明是没有填满缓冲, 根据测试, 这一级的缓冲区的大小为8192


行缓冲

行缓冲就是遇到换行(\n)就进行写入, 它也有size, 如果没有遇到换行但是size满了也进行写入

linux的终端就是行缓冲文件

tty        # 查看终端对应的文件

会返回一个文件的path

这个文件不是普通的文件, 它在linux中叫做char设备

而我们直接对这个文件进行写入, 默认情况下这种tty设备就是行缓冲

注意 行缓冲只能对文本模式生效, 而不是二进制


手动指定缓冲区大小

f = open('test.txt', 'wb', buffering=8192)

注意: 文本模式有两级缓冲, 文本模式下指定的缓冲是buffered那级, 而不是最上面那一层的

无缓冲就是把buffering设为0

行缓冲则是在文本模式下(因为行缓冲只对文本模式有用), 把buffering设为1

如果大于1则将其设置为缓冲区的大小


如何将文件映射到内存


实际用途:

  1. 在访问某些二进制文件时, 希望能把文件映射到内存中, 可以实现随机访问 (例如linux中的 framebuffer 设备文件)

  2. 某些嵌入式设备, 寄存器被编址到内存地址空间, 我们可以映射 /dev/mem 某范围, 去访问这些寄存器

  3. 如果多个进程映射同一个文件, 还能实现进程通信的目的


linux的 framebuffer 是对整个屏幕显示的抽象, 屏幕的显示是由像素点构成的, 每个像素点可能由4个字节表示颜色rgba, 而这个framebuffer就是1920 * 1080 * 4 这么大的文件, 对这个文件进行读写就可以修改屏幕的显示. 对于这种文件通过数组进行访问会比read/write方便很多


解决方案:

使用标准库中mmap.mmap() 函数, 将文件映射到进程的内存地址


创建一个大小为1兆, 内容全0的二进制文件

# dd命令用于读取、转换并输出数据
# /dev/zero 是一个特殊文件, 它里面内容全0
# 指定每块的大小为1024(bs), 一共有1024个块(count)
dd if=/dev/zero of=demo.bin bs=1024 count=1024

# -x 代表以16进制
# 以16进制的方式查看二进制文件demo.bin
od -x demo.bin

使用例子

import mmap

f = open('demo.bin', 'r+b')
# 0 代表把整个文件都映射进内存
m = mmap.mmap(f.fileno(), 0)

# 普通的write方式写入
m.write(b'abc')

# 用数组的随机访问进行写入
m[5] = 78
m[8:16] = b'\xff' * 8

centos中 /dev/fb0 就是对屏幕的抽象, 而我们要通过映射更改屏幕的显示

size = 8294400

m = mmap.mmap(f.fileno(), size)

# 给一半的屏幕写入白色, \xff就是255, \x00就是0, rgba中, rgb都为255, a为0
m[:size//2] = b'\xff\xff\xff\x00' * (size // 4 // 2)

m.close()
f.close()

如何使用临时文件

例如: 某项目中, 我们每采集数据1G数据就对它做分析, 最终只保留分析结果. 而这样大量的临时数据如果常驻内存会消耗大量资源. 所以我们可以使用临时文件存储这些临时数据 ( 外部存储 ).

临时文件不用命名, 且关闭后会自动删除


解决方案 : 使用标准库中的 TemporaryFile 以及 NamedTemporaryFile

from tempfile import TemporaryFile, NamedTemporaryFile

# 指定目录创建临时文件, 因为默认目录的大小可能会不够用
# 操作系统级别的临时文件, 系统调用的open打开的
tf = TemporaryFile(dir='/')
tf.write(b'*' * 1024 * 1024)
tf.write(b'*' * 1024 * 1024)
tf.write(b'*' * 1024 * 1024)
tf.write(b'*' * 1024 * 1024)

# 用法和普通文件一样,
# 但我们无法在文件系统中访问到它 例如:ls
tf.seek(0)
tf.read(512)

# 自动删除这个文件
tf.close()

#######################################

# 能创建出带名字的临时文件
# 不是系统级别,
# 无论创建临时文件的名字还是自动帮你删除这个文件
# 都是由python库实现的, 而不是由操作系统实现
# 多进程都想访问统一个临时文件时可以使用
ntf = NamedTemporaryFile()

# 存储的位置
ntf.name()

# 当构造器delete为True时(默认)会自动删除文件, 可以修改为False来不删除
ntf.close()

例如: 自定义一种新类型的元组, 对于传入的可迭代对象, 我们只保留其中int类型且值大于0的元素

先尝试这样实现

class IntTuple(tuple):
    def __init__(self, iterable):
        # 过滤iterable
        f_it = (e for e in iterable if isinstance(e, int) and e > 0)
        super(IntTuple, self).__init__(f_it)

但是运行时会发现出现问题, 因为父类object的init方法没有参数

也就是说很有可能tuple没有实现init方法


如果打印self则会发现元组已经存在, 那么这个self到底是谁来创建的?


通常我们把一个类的init方法成为这个类的构造器, 但这实际上是不准确的, 我们在创建一个类的实例时, 实际上会调用两个方法

class A:
    def __new__(cls, *args):
        print("In A.__new__", cls, args)
        return object.__new__(cls)

    def __init__(self, *args):
        print("In A.__init__", args)

if __name__ == '__main__':
    # a1 = A(1, 2)
    a = A.__new__(A, 1, 2)
    A.__init__(a, 1, 2)

其实在python中所有类的实例都是由object的new方法创建的, 如果不实现new, 调用的依然是父类object的new方法

# list的初始化
l = list.__new__(list)
list.__init__(l, 'abc')

# tuple的初始化
t = tuple.__new__(tuple, "abc")
# new 之后元组就初始化完成了, tuple根本就没实现init

print(tuple.__init is object.__init__)  # True

对于特殊情况(例如: tuple) 我们可以在new的时候直接初始化好

因此我们可以这么实现

class IntTuple(tuple):
    def __new__(cls, iterable):
        # 过滤iterable
        f_it = (e for e in iterable if isinstance(e, int) and e > 0)
        return super().__new__(cls, f_it)

if __name__ == '__main__':
    t = IntTuple([1, 2, 'e', -1, 9, ['x', 'y'], 3])
    print(t)

如何为创建大量实例节省内存

定义类的slots内置属性, 声明实例有哪些属性 (关闭动态绑定)

class Player1:
    def __init__(self, uid, name, level):
        self.uid = uid
        self.name = name
        self.level = level


class Player2:
    __slots__ = ['uid', 'name', 'level']

    def __init__(self, uid, name, level):
        self.uid = uid
        self.name = name
        self.level = level

上面两个类中, Player2实例使用的空间会更小

if __name__ == '__main__':
    p1 = Player1('0001', 'Jim', 20)
    p2 = Player2('0001', 'Jim', 20)

    # 查看一下 p1 比 p2 多出哪些东西
    # dir(p1) 会以列表的形式打印p1的所有属性
    # 对两个集合作差就可以得出我们要的结果
    print(set(dir(p1)) - set(dir(p2)))
    # {'__weakref__', '__dict__'}
    # p1 比 p2 多了两个属性, __weakref__ 和 __dict__

weakref 是弱引用, 在这里它的开销并不大. 主要占用内存的是dict

p1中的三个属性都是动态创建的

而实例内部就是通过dict来维护动态绑定的属性

# 等价语句
p1.z = 300
p1.__dict__['z'] = 300

# 删除属性
p1.__dict__.pop('x')

import sys

# 这个函数可以查看内置类型占用多少空间
res = sys.getsizeof(p1.__dict__)
print(res)

res2 = sys.getsizeof(p1.name) + sys.getsizeof(p1.level) + sys.getsizeof(p1.uid)
print(res2)

# res1 和 res2 分别为 112, 133

可以看出存储的数据是133个字节, 而维护他们的字典是112个字节, 近乎浪费一半的空间.

如果实例个数较少, 牺牲些空间换取动态绑定属性的特性还是值得的. 但如果实例的个数非常多, 就会造成非常大的浪费

而关闭动态添加数据就是用 slots 提前声明号实例中能有哪些属性, 提前分配好空间

if __name__ == '__main__':
    # tracemalloc 用于跟踪内存使用

如何让类支持with的自动开启和关闭 / 支持上下文管理

这里是一个简易的Telnet客户端

telnet是一种网络协议, 默认使用23端口, 和ssh类似但是是明文的, 不安全 所以默认不能使用root权限账号

import getpass
import telnetlib
from collections import deque
from sys import stdout, stdin


class TelnetClient:
    def __init__(self, host, port=23):
        self.host = host
        self.port = port

    def connect(self):
        self.tn = telnetlib.Telnet(self.host, self.port)
        self.history = deque([])

    def cleanup(self):
        self.tn.close()
        self.tn = None

        with open('history.txt', 'a') as f:
            f.writelines(self.history)

    def login(self):
        self.tn.read_until(b'login: ')
        user = input("Enter your remote account: ")
        self.tn.write(user.encode('utf8') + b"\n")

        self.tn.read_until(b'Password: ')
        password = getpass.getpass()
        self.tn.write(password.encode('utf8') + b"\n")
        out = self.tn.read_until(b'$ ')
        stdout.write(out.decode('utf8'))

    def interact(self):
        while True:
            cmd = stdin.readline()
            if not cmd:
                break
            self.history.append(cmd)
            self.tn.write(cmd.encode('utf8'))
            out = self.tn.read_until(b'$ ').decode('utf8')

            stdout.write(out[len(cmd)+1:])
            stdout.flush()


if __name__ == '__main__':
    client = TelnetClient('服务器ip')
    client.connect()
    client.login()
    client.interact()
    client.cleanup()

这里可以看到从创建client到使用它, 再到关闭一共要五行代码(main中)

而我们的目标是让这个类支持with自动打开和关闭(即支持上下文管理)


举个with的简单例子

with open("test.txt", 'w') as f:

注意: 这里并不是把open的返回值直接给f, 这里等同于下面语句

F = open("test.txt", 'w')
with F as f:

而 f 则是通过 F 的内置enter方法得到的

由于 F 的 enter 方法返回的是 self 所以 f 就是 F, 但这取决于enter的实现

而退出则会自动调用 F 的 exit 内置方法执行清理工作

因此我们只要把 connect 和 cleanup 分别改为 enter 和 exit

    def __enter__(self):
        self.tn = telnetlib.Telnet(self.host, self.port)
        self.history = deque([])

        # 返回的值赋值给as后的变量
        return self

    # 如果执行出现异常, 同样会调用此方法, 三个参数都是异常的描述
    # 如果正常退出, 则这三个参数都是None
    def __exit__(self, exc_type, exc_val, exc_tb):
        self.tn.close()
        self.tn = None

        with open('history.txt', 'a') as f:
            f.writelines(self.history)

        # 返回真说明异常已经被处理完毕, 不再抛给上一层
        return True
if __name__ == '__main__':
    with TelnetClient('服务器ip') as client:
        client.login()
        client.interact()

创建可管理的对象属性

通常我们在面向对象的编程中, 不建议直接对属性进行操作, 而是应该实用Getter和Setter来操作

import math

class Circle:
    def __init__(self, radius):
        self.radius = radius

    def get_radius(self):
        return self.radius

    def set_radius(self, radius):
        self.radius = radius

    def get_area(self):
        return self.radius ** 2 * math.pi

    def set_area(self, S):
        self.radius = math.sqrt(S / math.pi)

c = Circle(1111)
c.set_radius(32.95)

而在python中可以使用property函数为类创建可管理属性, 传入的参数

class Circle:
    def __init__(self, radius):
        self.radius = radius

    def get_radius(self):
        print("get R")
        return self.radius

    def set_radius(self, radius):
        print("set R")
        self.radius = radius

    # 使用方法2
    # 可以通过c.S来读取, 必须要和方法名一样
    @property
    def S(self):
        return self.radius ** 2 * math.pi

    # 可以通过c.S = xxx 来访问
    # S指的就是上面那个方法的名字
    @S.setter
    def S(self, s):
        self.radius = math.sqrt(s / math.pi)

    # 使用方法1
    # 虽然这里R是一个类属性, 但我们对R进行操作时
    # 调用的是操作radius的方法, 因此一个类属性就可以
    R = property(get_radius, set_radius)


if __name__ == '__main__':
    c = Circle(5.712)
    c.R = 31.89
    print("this is print ", c.R)
    # 设置S后R的值改变了
    c.S = 99.88
    print(c.S)
    print(c.R)

from functools import total_ordering

# 使用total_ordering, 运算符只要重载两个就行
# 一个是__eq__, 之后再随便重载一个就行 lt, le, gt, ge
# 其余的它会自动帮你实现
@total_ordering
class Rect:
    def __init__(self, w, h):
        self.w = w
        self.h = h

    def area(self):
        return self.w * self.h

    # 相当于java中的toString
    def __str__(self):
        return 'Rect:(%s, %s)' % (self.w, self.h)

    def __eq__(self, other):
        return self.area() == other.area()

    def __le__(self, other):
        return self.area() <= other.area()


rect1 = Rect(6, 9)
rect2 = Rect(7, 8)
print(rect1 < rect2)
print(rect1 > rect2)  # 由于我们没有重载对于方法, 会被翻译成 rect2 < rect1

使用描述符对实例属性作类型检查

对属性进行类型检查的方法有很多种, 例如在getter&setter中进行检查, 这里主要讲述什么是描述符, 以及如何用它检查类型


当一个类包含set, get, delete内置函数其中之一, 那它就是描述符

class Attr:

    def __init__(self, key, _type):
        self.key = key
        self._type = _type

    def __set__(self, instance, value):
        print('in __set__')
        # set时判断属性类型
        if not isinstance(value, self._type):
            raise TypeError('must be %s' % self._type)
        instance.__dict__[self.key] = value

    def __get__(self, instance, cls):
        print('in __get__', instance, cls)
        return instance.__dict__[self.key]

    def __delete__(self, instance):
        print('in __del__', instance)
        del instance.__dict__[self.key]


class Person:
    # 对这些属性的访问都会被描述符所捕获
    name = Attr('name', str)
    age = Attr('age', int)


if __name__ == '__main__':
    p = Person()
    p.name = 'daolin'
    p.age = 33

然而当出现循环引用时

# 双向链表
class Node:
    def __init__(self, data):
        self.data = data
        self.left = None
        self.right = None

    def add_right(self, node):
        self.right = node
        node.left = self

    def __str__(self):
        return 'Node:<%s>' % self.data

    def __del__(self):
        print('in __del__: delete %s' % self)


def create_linklist(n):
    head = current = Node(1)
    for i in range(2, n+1):
        node = Node(i)
        current.add_right(node)
        current = node
    return head

if __name__ == '__main__':
    head = create_linklist(1000)
    # 引用计数依然大于0, 对于不会执行del
    head = None

    for _ in range(1000):
        time.sleep(1)
        print('run...')
    input('wait...')

而解决方法就是使用弱引用, weakref.ref(), 弱引用不会计算在引用计数里. 

import weakref


class Node:
    def __init__(self, data):
        self.data = data
        self._left: Node = None
        self.right: Node = None

    def add_right(self, node):
        self.right = node
        # 把左节点改为弱引用, 所有的元素引用计数就都为1了
        node._left = weakref.ref(self)

    # 左节点通过self._left()获取引用
    # 右节点通过self.right获取引用
    # 为了对称, 根据上面学过的知识, 通过property让他们使用方法一样
    @property
    def left(self):
        # 弱引用通过()返回真正的引用
        return self._left()

    def __str__(self):
        return 'Node:<%s>' % self.data

    def __del__(self):
        print('in __del__: delete %s' % self)


def create_linklist(n):
    head = current = Node(1)
    for i in range(2, n+1):
        node = Node(i)
        current.add_right(node)
        current = node
    return head

if __name__ == '__main__':
    head = create_linklist(1000)
    print(head.right, head.right.left)
    head = None

如何通过方法名调用类的方法

有两种方式:

  1. 通过getattr获取方法对象, 之后调用
  2. 通过标准库operator下的methodcaller调用

这两种方法有什么不同


getattr:

s = '123abc987'
# 获取s的find方法对象, 如果没有这个方法则赋值为None(第三个参数)
method = getattr(s, 'find', None)
# 传参调用, 从index为3的位置寻找abc
method('abc', 3)

methodcaller

s = '123abc987'
method = 'find'
# 先告诉它要调用那个方法, 以及传参
caller = methodcaller(method, 3)
# 之后才传入调用的对象, 通常要判断一下这个对象中是否有这个方法
if hasattr(s, method):
    caller(s)

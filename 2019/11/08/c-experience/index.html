使用fork方式

步骤:

  1. fork创建一个子进程, 父进程退出, 子进程成为孤儿进程, 被init进程接管
  2. 虽然父进程退出, 但相关信息还保留着, sessionId依然标记原来的父进程是多少在那个session中, 所以要调用 setsid 建立新的进程会话
  3. 将当前工作目录切换到根目录
  4. 后台程序没有标准输出和标准出错的, 将标准输入, 输出, 出错重定向到 /dev/null

网络编程基础

七大步骤

  1. 创建socket, 指定使用tcp协议 (相当于打开了一个文件 / 创建了一个文件句柄)
  2. 将socket 与 地址和端口绑定 ( 在操作系统底层就会把事件抛给fd(文件描述符), 即这个socket)
  3. 监听端口, 不然虽然事件抛过去了, 但如果不见听上层应用依然不知道, 捕获不到时间
  4. 创建新的socket, 作数据的收发, 而不再继续占用监听的socket
  5. 使用recv接收数据
  6. 使用send发生数据
  7. 使用close发送数据

TCP常见套接字选项

  • SO_REUSEADDR 端口处于 WAIT_TIME 依然可以启动 (关闭socket后会有一段time wait时间, 直到收到finish, 设置它时候就不用等待一段时间了, 即可以启动另外一个程序依然使用这个地址和端口, 对于服务端来说非常有用)
  • SO_RCVBUF 接收的buffer的大小
  • SO_SNDBUF 发送的buffer的大小, 设置太大会有延迟

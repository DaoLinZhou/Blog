寄存器


汇编语言的两大知识点:

  1. 汇编指令

  2. 寄存器

寄存器和内存很像, 都是用来放东西的.

但寄存器在cpu里面, 距离cpu更近, 所以cpu操作起来更快


通常, CPU会先将内存中的数据存到内存中, 然后再对寄存器中的数据进行运算, 再赋值回去.

# 伪代码 b = a+1

# 1.把a的值读取到寄存器
mov eax, [a]

# 2.对寄存器内的数据进行+1
add eax, 1

# 3.赋值回去
mov [b], eax

x64 汇编 - 寄存器

不同汇编(AT&T)的寄存器也可能不一样

主要关注最左边的那一列

RAX \ RBX \ RCX \ RDX 一般被称为通用寄存器

而剩余的那部分一般习惯用于特殊用途, 严格来说这些也是通用寄存器


既然寄存器是用来存东西的, 那么就有一个问题: 它能存多少 (它多大)?

一个寄存器能存多大数据? 和指针一样, 而指针的大小取决于cpu的架构

x64 能存 8 byte, x86 能存 4 byte


RAX 和 EAX

然而在通过VS对c++进行反编译时, 我们却可以看到还有eax, 那么什么是eax? rax和eax之间有什么关系?


64bit (64位环境下), RAX \ RBX \ RCX \ RDX 是通用寄存器

但以前的cpu没有这么高级, 有32位, 16位的, 在当时肯定也有汇编


而在32位环境下, 这些通用寄存器分别叫做

EAX \ EBX \ ECX \ EDX


而在16位环境下, 这些通用寄存器分别叫做

AX \ BX \ CX \ DX


但是为了向后兼容, 它被设计成这样

它把64位寄存器的一半拿出来, 当作32位的寄存器用, 正如最开始那张图上所表示, 这些寄存器是分为2半的, 左边和右边各4个字节, 其中最低的4个字节就是eax寄存器.

rax 里面包含 eax

同理, 32位为了兼容16位, 也分出一半 (最低的两个字节) 给ax来用

ax 由被分为 AH 和 AL, 各占一个字节

AH H High 高字节

AL L Low 低字节

因此, 虽然我们使用x64环境, 但我们依然可以看到x86环境的汇编


一般R开头的寄存器是64bit, 占8字节

一般E开头的寄存器是32bit, 占4字节

ABCD这四个通用寄存器在汇编的发展过程中是一直存在的


在c++中嵌入汇编 (内联汇编)

int main() {
    int a = 10;
    // 内联汇编
    __asm {
        mov eax, a
        add eax, 1
        mov a, eax
    }
    cout << a << endl;
    return 0;
}

通过 __asm包裹的代码就是汇编代码, 但它只是表面上的汇编, 真实的汇编不可能有a这个东西


__asm {
    mov ax, 10
    mov eax, 11223344H
}

考虑这段代码, 当对eax进行赋值时会覆盖掉ax, 同样对ax的更改也会影响到eax的低2位


intel 汇编指令学习

mov dest src

将 src 的内容赋值给 dest

mov dest src

[地址值]

中括号[ ] 里面放的都是内存地址

mov dword ptr [ebp-8], 3

# 假设 ebp-8的值为1122h, 同时省略前面一大串
# 将3这个值放到1122h这个地址所在的存储空间
mov [1122h], 3

# 然而赋值是要提前知道容器所占空间的大小
# 例如: int a = 5, int是4个字节, 就是说用4个字节存储5
# 所以当我们把数值放入某块内存空间时要指定单位
# word    是 2个字节
# dword   是 4个字节(double word)
# qword   是 8个字节(quad word)
# ptr 是固定写法
mov dword ptr [1122h], 3

# 但1122h 这个地址只占一个字节, 怎么才会占4个字节?
# 要么向上延伸, 要么向下延伸
# 一般来说都是往高地址走, 即占用1122h, 1123h, 1124h, 1125h

# 从1122h这个存储空间读取4个字节到eax
# 同样往高地址走, 读取 1122h, 1123h, 1124h, 1125h
mov eax, dword ptr [1122h]

# 同样我们可以在vs的内存中找到地址, 计算里面的值是不是我们要的结果, 注意读取的顺序, 因为cpu有大小端模式, 通常我们使用的都是小端模式, 即从低字节开始存储,
# 一个变量的地址值, 是它所有字节地址中的最小值

call

调用函数, 后面接函数的地址

call    00081000

lea dest, [地址值]

与mov的差别

load effect address 装在一个有效的地址值

# 直接将1122h这个地址值赋值给eax
# eax == 1122h
lea eax, [1122h]

# 把1122h这个地址内存储的数据赋值给eax
mov eax, dword ptr [1122h]

ret

函数返回


xor op1, op2

将op1和op2异或的结果赋值给op1, 类似于 op1 = op1 ^ op2


add op1, op2

类似与 op1 = op1 - op2


sub op1, op2

类似于 op1 = op1 - op2


inc op

increase 自增, 类似于 op = op + 1


dec op

decrease 自减, 类似于 op = op - 1


jmp 内存地址

跳转到这个内存地址所对应的汇编代码, 执行代码,

寄存器


汇编语言的两大知识点:

  1. 汇编指令

  2. 寄存器

寄存器和内存很像, 都是用来放东西的.

但寄存器在cpu里面, 距离cpu更近, 所以cpu操作起来更快


通常, CPU会先将内存中的数据存到内存中, 然后再对寄存器中的数据进行运算, 再赋值回去.


x64 汇编 - 寄存器

不同汇编(AT&T)的寄存器也可能不一样

主要关注最左边的那一列

RAX \ RBX \ RCX \ RDX 一般被称为通用寄存器

而剩余的那部分一般习惯用于特殊用途, 严格来说这些也是通用寄存器


既然寄存器是用来存东西的, 那么就有一个问题: 它能存多少 (它多大)?

一个寄存器能存多大数据? 和指针一样, 而指针的大小取决于cpu的架构

x64 能存 8 byte, x86 能存 4 byte


RAX 和 EAX

然而在通过VS对c++进行反编译时, 我们却可以看到还有eax, 那么什么是eax? rax和eax之间有什么关系?


64bit (64位环境下), RAX \ RBX \ RCX \ RDX 是通用寄存器

但以前的cpu没有这么高级, 有32位, 16位的, 在当时肯定也有汇编


而在32位环境下, 这些通用寄存器分别叫做

EAX \ EBX \ ECX \ EDX


而在16位环境下, 这些通用寄存器分别叫做

AX \ BX \ CX \ DX


但是为了向后兼容, 它被设计成这样

它把64位寄存器的一半拿出来, 当作32位的寄存器用, 正如最开始那张图上所表示, 这些寄存器是分为2半的, 左边和右边各4个字节, 其中最低的4个字节就是eax寄存器.

rax 里面包含 eax

同理, 32位为了兼容16位, 也分出一半 (最低的两个字节) 给ax来用

ax 由被分为 AH 和 AL, 各占一个字节

AH H High 高字节

AL L Low 低字节

因此, 虽然我们使用x64环境, 但我们依然可以看到x86环境的汇编


一般R开头的寄存器是64bit, 占8字节

一般E开头的寄存器是32bit, 占4字节

ABCD这四个通用寄存器在汇编的发展过程中是一直存在的


在c++中嵌入汇编 (内联汇编)

通过 __asm包裹的代码就是汇编代码, 但它只是表面上的汇编, 真实的汇编不可能有a这个东西


考虑这段代码, 当对eax进行赋值时会覆盖掉ax, 同样对ax的更改也会影响到eax的低2位


intel 汇编指令学习

mov dest src

将 src 的内容赋值给 dest

[地址值]

中括号[ ] 里面放的都是内存地址


call

调用函数, 后面接函数的地址


lea dest, [地址值]

与mov的差别

load effect address 装在一个有效的地址值


ret

函数返回


xor op1, op2

将op1和op2异或的结果赋值给op1, 类似于 op1 = op1 ^ op2


add op1, op2

类似与 op1 = op1 - op2


sub op1, op2

类似于 op1 = op1 - op2


inc op

increase 自增, 类似于 op = op + 1


dec op

decrease 自减, 类似于 op = op - 1


jmp 内存地址

跳转到这个内存地址所对应的汇编代码, 执行代码,
